# 13. Генерация кода и build_runner — обзор

## О чём этот раздел

Dart не поддерживает runtime-рефлексию в AOT-режиме, поэтому многие задачи, которые в других языках решаются через рефлексию (сериализация, DI, маршрутизация), решаются **генерацией кода** на этапе сборки. `build_runner` — стандартный механизм запуска генераторов.

## Ключевые понятия

```
┌─────────── Процесс генерации кода ────────────────┐
│                                                    │
│  Исходный код        →  Генератор      → .g.dart  │
│  (аннотации, AST)       (Builder)        файлы    │
│                                                    │
│  user.dart           →  json_serializable          │
│  @JsonSerializable       → user.g.dart             │
│                                                    │
│  state.dart          →  freezed                    │
│  @freezed                → state.freezed.dart      │
│                                                    │
└────────────────────────────────────────────────────┘
```

## Когда используется генерация кода

| Задача               | Пакет-генератор                   |
| -------------------- | --------------------------------- |
| JSON-сериализация    | `json_serializable`               |
| Неизменяемые классы  | `freezed`                         |
| DI-контейнер         | `injectable`                      |
| Маршрутизация        | `auto_route`, `go_router_builder` |
| Моки для тестов      | `mockito` (build_runner)          |
| GraphQL-типы         | `graphql_codegen`                 |
| Копирование объектов | `copy_with_extension`             |

## Основные команды

```bash
# Одноразовая генерация
dart run build_runner build

# Генерация с удалением старых файлов
dart run build_runner build --delete-conflicting-outputs

# Watch-режим (автогенерация при изменениях)
dart run build_runner watch

# Очистка сгенерированных файлов
dart run build_runner clean
```
