# 14.1 Dart FFI

## 1. Формальное определение

**FFI (Foreign Function Interface)** — механизм вызова функций из нативных библиотек (C, C++, Rust и др.) напрямую из Dart-кода. Реализован через библиотеку `dart:ffi`.

FFI работает **только** на нативных платформах (iOS, Android, macOS, Windows, Linux). Не поддерживается в Web-компиляции.

## 2. Зачем это нужно

- **Переиспользование** — доступ к огромной экосистеме C/C++ библиотек (SQLite, OpenSSL, zlib).
- **Производительность** — вычислительно тяжёлые операции выполняются в нативном коде.
- **Платформа** — вызов системных API, недоступных из чистого Dart.
- **Flutter-плагины** — FFI замещает платформенные каналы для нативного кода.

## 3. Как это работает

### Загрузка динамической библиотеки

```dart
import 'dart:ffi';
import 'dart:io' show Platform;

// Загрузка библиотеки в зависимости от платформы
DynamicLibrary _loadLibrary() {
  if (Platform.isWindows) return DynamicLibrary.open('mylib.dll');
  if (Platform.isMacOS) return DynamicLibrary.open('libmylib.dylib');
  if (Platform.isLinux) return DynamicLibrary.open('libmylib.so');
  if (Platform.isAndroid) return DynamicLibrary.open('libmylib.so');
  throw UnsupportedError('Unsupported platform');
}

// Для стандартных системных библиотек
final stdlib = DynamicLibrary.process(); // macOS/Linux
// или DynamicLibrary.executable() для символов из самого приложения
```

### Вызов простой C-функции

```c
// native/math_utils.c
#include <math.h>

int add(int a, int b) {
    return a + b;
}

double square_root(double x) {
    return sqrt(x);
}
```

```dart
import 'dart:ffi';

// 1. Определяем typedef для нативной функции (C-сигнатура)
typedef AddNative = Int32 Function(Int32 a, Int32 b);

// 2. Определяем typedef для Dart-функции
typedef AddDart = int Function(int a, int b);

// 3. Загружаем и привязываем
final dylib = DynamicLibrary.open('libmath_utils.so');

final add = dylib.lookupFunction<AddNative, AddDart>('add');

// 4. Вызываем
final result = add(3, 5);
print(result); // 8
```

### Типы данных FFI

```
┌─────────── Соответствие типов ──────────────────────┐
│  C тип          │  dart:ffi тип    │  Dart тип      │
├─────────────────┼──────────────────┼────────────────┤
│  int8_t         │  Int8            │  int           │
│  int16_t        │  Int16           │  int           │
│  int32_t        │  Int32           │  int           │
│  int64_t        │  Int64           │  int           │
│  uint8_t        │  Uint8           │  int           │
│  uint16_t       │  Uint16          │  int           │
│  uint32_t       │  Uint32          │  int           │
│  uint64_t       │  Uint64          │  int           │
│  float          │  Float           │  double        │
│  double         │  Double          │  double        │
│  void           │  Void            │  void          │
│  void*          │  Pointer<Void>   │  Pointer<Void> │
│  char*          │  Pointer<Utf8>   │  Pointer<Utf8> │
│  bool           │  Bool            │  bool          │
│  size_t         │  Size            │  int           │
└─────────────────┴──────────────────┴────────────────┘
```

### Работа со строками

```dart
import 'dart:ffi';
import 'package:ffi/ffi.dart'; // Для Utf8, Utf16

// C: char* greet(const char* name)
typedef GreetNative = Pointer<Utf8> Function(Pointer<Utf8> name);
typedef GreetDart = Pointer<Utf8> Function(Pointer<Utf8> name);

final greet = dylib.lookupFunction<GreetNative, GreetDart>('greet');

// Dart → C строка
final namePtr = 'Alice'.toNativeUtf8();

// Вызов
final resultPtr = greet(namePtr);

// C → Dart строка
final result = resultPtr.toDartString();
print(result); // "Hello, Alice!"

// Освобождение памяти
calloc.free(namePtr);
// resultPtr тоже нужно освободить, если C не делает это сам
```

### Структуры

```c
// C struct
typedef struct {
    int x;
    int y;
    double radius;
} Circle;

Circle create_circle(int x, int y, double radius);
double circle_area(Circle* c);
```

```dart
// Dart binding для struct
final class Circle extends Struct {
  @Int32()
  external int x;

  @Int32()
  external int y;

  @Double()
  external double radius;
}

// Использование
typedef CreateCircleNative = Circle Function(Int32 x, Int32 y, Double r);
typedef CreateCircleDart = Circle Function(int x, int y, double r);

typedef CircleAreaNative = Double Function(Pointer<Circle> c);
typedef CircleAreaDart = double Function(Pointer<Circle> c);

final createCircle =
    dylib.lookupFunction<CreateCircleNative, CreateCircleDart>('create_circle');
final circleArea =
    dylib.lookupFunction<CircleAreaNative, CircleAreaDart>('circle_area');

// Создание структуры в Dart
final circle = calloc<Circle>();
circle.ref.x = 10;
circle.ref.y = 20;
circle.ref.radius = 5.0;

final area = circleArea(circle);
print('Area: $area');

calloc.free(circle);
```

## 4. Управление памятью

```dart
import 'dart:ffi';
import 'package:ffi/ffi.dart';

// Выделение памяти
final ptr = calloc<Int32>();      // Один int32
final arr = calloc<Int32>(10);     // Массив из 10 int32

// Запись значений
ptr.value = 42;
arr[0] = 1;
arr[1] = 2;

// Чтение
print(ptr.value);  // 42
print(arr[3]);     // 0 (calloc обнуляет)

// Освобождение — ОБЯЗАТЕЛЬНО!
calloc.free(ptr);
calloc.free(arr);
```

### NativeFinalizer — автоматическое освобождение

```dart
// Dart 2.17+ — привязка освобождения к сборщику мусора
class NativeResource implements Finalizable {
  static final _Release =
      dylib.lookup<NativeFunction<Void Function(Pointer<Void>)>>('release');
  static final _finalizer = NativeFinalizer(_Release);

  final Pointer<Void> _handle;

  NativeResource(this._handle) {
    // Когда объект NativeResource соберётся GC,
    // автоматически вызовется release(_handle)
    _finalizer.attach(this, _handle, detach: this);
  }

  void dispose() {
    _finalizer.detach(this);
    dylib.lookupFunction<Void Function(Pointer<Void>),
        void Function(Pointer<Void>)>('release')(_handle);
  }
}
```

## 5. ffigen — автоматическая генерация привязок

Вместо ручного написания typedef'ов используйте `ffigen`:

```yaml
# pubspec.yaml
dev_dependencies:
  ffigen: ^11.0.0
```

```yaml
# ffigen.yaml
name: MyBindings
description: Bindings for mylib
output: "lib/src/mylib_bindings.dart"
headers:
  entry-points:
    - "native/include/mylib.h"
  include-directives:
    - "native/include/mylib.h"
```

```bash
dart run ffigen
# Генерирует lib/src/mylib_bindings.dart со всеми привязками
```

## 6. Callback — вызов Dart из C

```c
// C: функция, принимающая callback
typedef int (*Comparator)(int a, int b);
void sort_array(int* arr, int len, Comparator cmp);
```

```dart
// Dart callback
int compareAsc(int a, int b) => a - b;

// Создание указателя на Dart-функцию
final cmpPtr = Pointer.fromFunction<Int32 Function(Int32, Int32)>(
  compareAsc,
  0,  // exceptional return value (если Dart бросит исключение)
);

// Передача в C
sortArray(arrPtr, length, cmpPtr);
```

## 7. Распространённые ошибки

### ❌ Утечка памяти

```dart
// Плохо — выделили, но не освободили
final ptr = calloc<Utf8>(100);
// ... используем ptr ...
// забыли calloc.free(ptr);

// Хорошо — используем try/finally или Arena
final arena = Arena();
try {
  final ptr = arena<Utf8>(100);
  // ... используем ptr ...
} finally {
  arena.releaseAll();  // Освобождает ВСЮ память арены
}

// Или ещё лучше — using
using((arena) {
  final ptr = arena<Utf8>(100);
  // ... автоматически освободится при выходе ...
});
```

### ❌ Использование указателя после освобождения

```dart
final ptr = calloc<Int32>();
ptr.value = 42;
calloc.free(ptr);
print(ptr.value);  // ← Undefined behavior! Crash!
```

### ❌ Неправильное соответствие типов

```dart
// C функция возвращает int64, но мы указали Int32
// → Потеря данных, некорректные значения
typedef BadNative = Int32 Function();  // должно быть Int64!
```
