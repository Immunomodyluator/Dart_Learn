## Зачем нужен CI/CD

CI/CD (Continuous Integration / Continuous Deployment) автоматизирует рутинные задачи:

- **CI** — при каждом коммите автоматически запускаются тесты, анализ и форматирование.
- **CD** — после проверки код автоматически публикуется или развёртывается.

Это снижает человеческий фактор: ошибки форматирования, пропущенные тесты и упущенные предупреждения анализатора ловятся до мержа.

## Типичный пайплайн для Dart

```
Push / PR
   │
   ├── dart format --set-exit-if-changed .
   ├── dart analyze --fatal-infos
   ├── dart test
   ├── dart test --coverage=coverage
   │
   └── (при мерже в main)
       ├── dart pub publish        ← пакет
       └── dart compile exe + deploy   ← сервер/CLI
```
