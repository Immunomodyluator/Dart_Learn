# 4.3 Map — карты

## 1. Формальное определение

**Map\<K, V\>** — ассоциативная коллекция пар «ключ→значение», где каждый ключ типа `K` уникален и отображается на значение типа `V`. Доступ по ключу — O(1) в среднем.

По умолчанию `Map` в Dart — `LinkedHashMap`, сохраняющий порядок вставки. Литеральный синтаксис: `{'key': value}`.

**Сигнатура:** `abstract class Map<K, V>`

## 2. Зачем это нужно

- **Ассоциативный доступ** — поиск значения по ключу за O(1).
- **JSON-данные** — `Map<String, dynamic>` — стандартное представление JSON в Dart.
- **Кеширование и мемоизация** — хранение вычисленных результатов.
- **Конфигурации** — словари настроек, маппинги enum→обработчик.
- **Группировка данных** — подсчёт частот, индексы.

## 3. Как это работает

### Создание

```dart
void main() {
  // Литерал
  var capitals = {
    'Россия': 'Москва',
    'Франция': 'Париж',
    'Япония': 'Токио',
  }; // Map<String, String>

  // Пустая карта
  var empty = <String, int>{};
  Map<String, List<int>> grouped = {};

  // Конструкторы
  var m1 = Map<String, int>.of({'a': 1, 'b': 2});
  var m2 = Map<String, int>.fromIterable(
    ['a', 'b', 'c'],
    key: (e) => e,
    value: (e) => e.codeUnitAt(0),
  );
  var m3 = Map<String, int>.fromEntries([
    MapEntry('x', 10),
    MapEntry('y', 20),
  ]);
  var m4 = Map<String, int>.unmodifiable({'a': 1});
}
```

### Основные операции

```dart
void main() {
  var scores = <String, int>{};

  // Добавление / обновление
  scores['Алиса'] = 95;
  scores['Борис'] = 87;
  scores['Алиса'] = 98; // Перезапись

  // Доступ
  print(scores['Алиса']);    // 98 (int?)
  print(scores['Карл']);     // null (нет ключа)

  // Безопасный доступ
  var value = scores['Карл'] ?? 0; // 0 — default

  // putIfAbsent — добавить, только если ключа нет
  scores.putIfAbsent('Борис', () => 100); // Не перезапишет: уже есть
  scores.putIfAbsent('Дарья', () => 91); // Добавит

  // update — обновить существующее значение
  scores.update('Алиса', (v) => v + 1); // 98 → 99
  scores.update('Егор', (v) => v, ifAbsent: () => 50); // Создаст с 50

  // Удаление
  scores.remove('Борис');
  scores.removeWhere((k, v) => v < 60);

  // Проверки
  print(scores.containsKey('Алиса')); // true
  print(scores.containsValue(99));     // true
  print(scores.length);
  print(scores.isEmpty);

  // Итерация
  scores.forEach((key, value) {
    print('$key: $value');
  });

  for (final entry in scores.entries) {
    print('${entry.key}: ${entry.value}');
  }

  // Ключи и значения
  print(scores.keys);    // Iterable<String>
  print(scores.values);  // Iterable<int>
}
```

### Map.fromIterables и groupBy

```dart
import 'package:collection/collection.dart'; // groupBy

void main() {
  // Два списка → Map
  var keys = ['a', 'b', 'c'];
  var values = [1, 2, 3];
  var map = Map.fromIterables(keys, values); // {a: 1, b: 2, c: 3}

  // Группировка (из package:collection)
  var words = ['cat', 'car', 'dog', 'dart'];
  var grouped = groupBy(words, (String w) => w[0]);
  // {c: [cat, car], d: [dog, dart]}
}
```

## 5. Практический пример

### Кеширующий HTTP-клиент (мемоизация)

```dart
class CachedFetcher {
  final Map<String, _CacheEntry> _cache = {};
  final Duration ttl;

  CachedFetcher({this.ttl = const Duration(minutes: 5)});

  Future<String> fetch(String url) async {
    final now = DateTime.now();

    // Проверить кеш
    final cached = _cache[url];
    if (cached != null && now.difference(cached.timestamp) < ttl) {
      print('Cache HIT: $url');
      return cached.data;
    }

    // Имитация HTTP-запроса
    print('Cache MISS: $url');
    final data = await _simulateRequest(url);

    // Сохранить в кеш
    _cache[url] = _CacheEntry(data, now);
    return data;
  }

  void invalidate(String url) => _cache.remove(url);
  void clearAll() => _cache.clear();
  int get cacheSize => _cache.length;

  Future<String> _simulateRequest(String url) async {
    await Future.delayed(Duration(milliseconds: 100));
    return '{"status": "ok", "url": "$url"}';
  }
}

class _CacheEntry {
  final String data;
  final DateTime timestamp;
  _CacheEntry(this.data, this.timestamp);
}

void main() async {
  final fetcher = CachedFetcher(ttl: Duration(seconds: 30));

  // Первый запрос — MISS
  await fetcher.fetch('https://api.example.com/users');

  // Повторный — HIT
  await fetcher.fetch('https://api.example.com/users');

  print('Размер кеша: ${fetcher.cacheSize}');
}
```

## 6. Что происходит под капотом

### LinkedHashMap (по умолчанию)

```
Map<String, int> m = {'a': 1, 'b': 2, 'c': 3};

Hash table + doubly linked list:

Buckets:
  [0] → null
  [1] → Entry('a', 1) ←→ Entry('b', 2) ←→ Entry('c', 3)
  [2] → null                    │
  [3] ─────────────────────────►│ (hash collision chain)

Каждый Entry:
  - key, value
  - hashCode
  - nextBucket (цепочка коллизий)
  - prevInsertion, nextInsertion (порядок вставки)
```

### Процесс доступа `map['key']`

```
1. Вычислить 'key'.hashCode
2. Bucket index = hashCode & (buckets.length - 1)   // mask, не modulo
3. Пройти цепочку bucket'а
4. Для каждого entry: entry.key == 'key' ?
5. Найдено → вернуть entry.value
6. Не найдено → вернуть null
```

### Rehashing

Когда `length / buckets.length > loadFactor` (~0.75):

1. Новый массив bucket'ов размером `buckets.length * 2`
2. Все entries перехешируются
3. O(n) единоразово

### Реализации Map

| Реализация            | Порядок           | Где               |
| --------------------- | ----------------- | ----------------- |
| `LinkedHashMap`       | Вставки           | По умолчанию      |
| `HashMap`             | Нет               | `dart:collection` |
| `SplayTreeMap`        | По ключу (sorted) | `dart:collection` |
| `UnmodifiableMapView` | Как обёрнутый     | `dart:collection` |

## 7. Производительность и ресурсы

| Операция          | LinkedHashMap         | SplayTreeMap |
| ----------------- | --------------------- | ------------ |
| `map[key]`        | O(1) avg              | O(log n)     |
| `map[key] = val`  | O(1) avg              | O(log n)     |
| `remove(key)`     | O(1) avg              | O(log n)     |
| `containsKey()`   | O(1) avg              | O(log n)     |
| `containsValue()` | O(n)                  | O(n)         |
| Итерация          | O(n), порядок вставки | O(n), sorted |

**Память:** ~3-4 указателя на entry + backing array buckets.

## 8. Частые ошибки и антипаттерны

### ❌ Забыть, что `map[key]` возвращает nullable

```dart
Map<String, int> scores = {'a': 10};

// int x = scores['a']; // Ошибка: тип int?, не int!
int x = scores['a']!;      // OK, если уверены
int y = scores['b'] ?? 0;  // Безопасный default
```

### ❌ Использование Map<String, dynamic> вместо класса

```dart
// Плохо: нет типобезопасности
var user = <String, dynamic>{
  'name': 'Алиса',
  'age': 30,
};
print(user['naem']); // Опечатка → null, без ошибки компилятора

// Хорошо: класс
class User {
  final String name;
  final int age;
  User({required this.name, required this.age});
}
```

### ❌ Мутация ключей

```dart
// Не используйте мутабельные объекты как ключи Map!
// hashCode/== изменятся → ключ "потеряется"
```

### ❌ containsKey + [] вместо ?? или putIfAbsent

```dart
// Плохо: два lookup
if (map.containsKey(key)) {
  return map[key]!;
}

// Хорошо: один lookup
return map[key] ?? defaultValue;
```

## 9. Сравнение с альтернативами

| Аспект           | Dart `Map`     | Java `HashMap`          | JS `Map`/`{}`     | Python `dict`     |
| ---------------- | -------------- | ----------------------- | ----------------- | ----------------- |
| Литерал          | `{'k': v}`     | ❌                      | `{k: v}` (Object) | `{'k': v}`        |
| Порядок          | Вставки        | Нет (LinkedHashMap: да) | Вставки           | Вставки (3.7+)    |
| Null-ключи       | ✅             | ✅                      | ✅                | ❌ (None OK)      |
| `putIfAbsent`    | ✅             | `computeIfAbsent`       | ❌                | `setdefault`      |
| Sorted variant   | `SplayTreeMap` | `TreeMap`               | ❌                | ❌                |
| Pattern matching | Dart 3 ✅      | ❌                      | ❌                | ❌ (3.10 partial) |

## 10. Когда НЕ стоит использовать

| Сценарий                        | Вместо Map используйте                 |
| ------------------------------- | -------------------------------------- |
| Фиксированная структура         | Класс или Record `(String, int)`       |
| Только проверка наличия         | `Set<K>`                               |
| Упорядоченные данные по индексу | `List<V>`                              |
| Иерархическая конфигурация      | Классы с вложенностью                  |
| JSON «на проброс»               | Чем раньше парсить в класс — тем лучше |

## 11. Краткое резюме

1. **`{'key': value}`** — литерал Map. Пустой `<String, int>{}`.
2. **`map[key]`** возвращает `V?` (nullable!). Используйте `?? default` или `!`.
3. **LinkedHashMap** по умолчанию — порядок вставки сохраняется.
4. **`putIfAbsent`** и **`update`** — атомарные операции «проверить и записать».
5. **O(1)** доступ, вставка, удаление — в среднем.
6. **`Map<String, dynamic>` — антипаттерн** для внутренних данных. Используйте классы.
7. **Ключи должны быть иммутабельны** — иначе `hashCode` ломается.
