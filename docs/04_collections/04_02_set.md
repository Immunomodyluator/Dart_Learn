# 4.2 Set — множества

## 1. Формальное определение

**Set\<E\>** — неупорядоченная коллекция уникальных элементов типа `E`. Каждый элемент встречается не более одного раза. Уникальность определяется через `operator ==` и `hashCode`.

По умолчанию `Set` в Dart — это `LinkedHashSet`, который **сохраняет порядок вставки** (в отличие от математического определения множества).

**Литеральный синтаксис:** `{1, 2, 3}` (не путать с пустым `Map`: `{}`).

## 2. Зачем это нужно

- **Уникальность** — автоматическое исключение дубликатов.
- **Быстрый поиск** — `contains()` за O(1) вместо O(n) для List.
- **Операции множеств** — объединение, пересечение, разность — встроены.
- **Дедупликация** — `list.toSet().toList()` — idiom для удаления дубликатов.
- **Flutter** — Set тегов, ролей, выбранных ID.

## 3. Как это работает

### Создание

```dart
void main() {
  // Литерал
  var fruits = {'яблоко', 'банан', 'вишня'}; // Set<String>

  // Пустой Set (НЕ {}, это Map!)
  var empty = <int>{};
  Set<double> prices = {};

  // Из List (дедупликация)
  var unique = {1, 2, 2, 3, 3, 3}; // {1, 2, 3}
  var fromList = [1, 2, 2, 3].toSet(); // {1, 2, 3}

  // Конструкторы
  var s1 = Set<int>.of([3, 1, 2]);              // LinkedHashSet
  var s2 = Set<int>.unmodifiable({1, 2, 3});     // Нельзя менять
  var s3 = Set<int>.identity();                  // Сравнение по identical()
}
```

### Основные операции

```dart
void main() {
  var set = {10, 20, 30};

  // Добавление
  set.add(40);             // true (добавлен)
  set.add(20);             // false (уже есть)
  set.addAll([50, 60]);

  // Удаление
  set.remove(10);          // true (удалён)
  set.removeWhere((e) => e > 50);

  // Проверки
  print(set.contains(20)); // true — O(1)!
  print(set.length);       // количество элементов
  print(set.isEmpty);

  // Итерация (порядок вставки сохраняется)
  for (final item in set) {
    print(item);
  }
}
```

### Операции множеств

```dart
void main() {
  var a = {1, 2, 3, 4};
  var b = {3, 4, 5, 6};

  // Объединение (union)
  print(a.union(b));           // {1, 2, 3, 4, 5, 6}

  // Пересечение (intersection)
  print(a.intersection(b));    // {3, 4}

  // Разность (difference)
  print(a.difference(b));      // {1, 2}
  print(b.difference(a));      // {5, 6}

  // Проверка подмножества
  print({1, 2}.containsAll({1, 2})); // true
  print(a.containsAll(b));           // false
}
```

## 4. Минимальный пример

```dart
void main() {
  var tags = <String>{};

  tags.add('dart');
  tags.add('flutter');
  tags.add('dart'); // Игнорируется

  print(tags);          // {dart, flutter}
  print(tags.length);   // 2
  print(tags.contains('dart')); // true
}
```

## 5. Практический пример

### Система ролей с проверкой разрешений

```dart
class User {
  final String name;
  final Set<String> roles;

  User(this.name, this.roles);

  bool hasRole(String role) => roles.contains(role);

  bool hasAnyRole(Set<String> required) =>
      roles.intersection(required).isNotEmpty;

  bool hasAllRoles(Set<String> required) =>
      roles.containsAll(required);
}

class PermissionChecker {
  static const _rolePermissions = <String, Set<String>>{
    'admin': {'read', 'write', 'delete', 'manage_users'},
    'editor': {'read', 'write'},
    'viewer': {'read'},
  };

  static Set<String> getPermissions(Set<String> roles) {
    return roles
        .expand((role) => _rolePermissions[role] ?? <String>{})
        .toSet();
  }

  static bool canPerform(User user, String action) {
    return getPermissions(user.roles).contains(action);
  }
}

void main() {
  final admin = User('Алиса', {'admin'});
  final editor = User('Борис', {'editor', 'viewer'});

  print(PermissionChecker.canPerform(admin, 'delete'));   // true
  print(PermissionChecker.canPerform(editor, 'delete'));  // false
  print(PermissionChecker.canPerform(editor, 'read'));    // true

  print(PermissionChecker.getPermissions(editor.roles));
  // {read, write}
}
```

## 6. Что происходит под капотом

### LinkedHashSet (по умолчанию)

```
Set<int> s = {42, 17, 99};

Внутри — hash table + linked list:

Hash table (buckets):
  [0] → null
  [1] → Entry(42) →→→→→→→┐
  [2] → Entry(17) ──┐    │ (linked list — порядок вставки)
  [3] → null        │    │
  [4] → Entry(99) ←─┘←───┘

Каждый Entry хранит:
  - value: 42
  - hashCode: 42.hashCode
  - next (bucket chain)
  - nextInsertion / prevInsertion (порядок вставки)
```

### Как работает contains

```
s.contains(17):
  1. Вычислить 17.hashCode
  2. Bucket index = hashCode % buckets.length
  3. Пройти по цепочке bucket'а
  4. Сравнить через == → найдено → true
  O(1) average, O(n) worst case (при коллизиях)
```

### Реализации Set в Dart

| Реализация            | Порядок       | Где                              |
| --------------------- | ------------- | -------------------------------- |
| `LinkedHashSet`       | Вставки       | `dart:collection` (по умолчанию) |
| `HashSet`             | Нет           | `dart:collection`                |
| `SplayTreeSet`        | Сортированный | `dart:collection`                |
| `UnmodifiableSetView` | Как обёрнутый | `dart:collection`                |

## 7. Производительность и ресурсы

| Операция          | LinkedHashSet         | SplayTreeSet |
| ----------------- | --------------------- | ------------ |
| `add(e)`          | O(1) avg              | O(log n)     |
| `contains(e)`     | O(1) avg              | O(log n)     |
| `remove(e)`       | O(1) avg              | O(log n)     |
| `union(b)`        | O(m)                  | O(m log n)   |
| `intersection(b)` | O(min(n,m))           | O(m log n)   |
| Итерация          | O(n), порядок вставки | O(n), sorted |

**Память:** LinkedHashSet ~2x-3x от размера данных (hash table + entries + links).

## 8. Частые ошибки и антипаттерны

### ❌ Пустой Set через `{}`

```dart
var wrong = {};       // Map<dynamic, dynamic>!
var right = <int>{};  // Set<int>
Set<int> also = {};   // OK — тип из аннотации
```

### ❌ Мутабельные объекты как элементы

```dart
class Point {
  int x, y;
  Point(this.x, this.y);

  @override
  int get hashCode => Object.hash(x, y);

  @override
  bool operator ==(Object other) =>
      other is Point && x == other.x && y == other.y;
}

var set = <Point>{};
var p = Point(1, 2);
set.add(p);
p.x = 10; // hashCode изменился! Set сломан!
print(set.contains(p)); // false или непредсказуемо!
```

### ❌ Использование Set когда важен индекс

```dart
// Set не поддерживает [] оператор
var set = {10, 20, 30};
// var x = set[0]; // Ошибка компиляции!

// Если нужен индексный доступ — используйте List
var x = set.elementAt(0); // Работает, но O(n)!
```

## 9. Сравнение с альтернативами

| Аспект            | Dart `Set`        | Java `HashSet`     | JS `Set`         | Python `set`       |
| ----------------- | ----------------- | ------------------ | ---------------- | ------------------ | ----------- |
| Порядок (default) | Вставки           | Нет                | Вставки          | Нет (3.7+ вставки) |
| Литерал           | `{1, 2}`          | ❌                 | `new Set([1,2])` | `{1, 2}`           |
| Типизация         | `Set<int>`        | `HashSet<Integer>` | Слабая           | Нет                |
| Операции          | `.union()` и т.д. | `addAll` и т.д.    | ❌ (ручные)      | `                  | `, `&`, `-` |
| Sorted variant    | `SplayTreeSet`    | `TreeSet`          | ❌               | `sorted()`         |

Dart выигрывает сохранением порядка по умолчанию и литеральным синтаксисом.

## 10. Когда НЕ стоит использовать

| Сценарий                   | Вместо Set используйте               |
| -------------------------- | ------------------------------------ |
| Нужен индексный доступ     | `List<E>`                            |
| Ключ→значение              | `Map<K, V>`                          |
| Порядок по критерию        | `SplayTreeSet` или `List` + `sort()` |
| Маленькие коллекции (< 10) | `List` — hash overhead не оправдан   |
| Часто нужен `toList()`     | Сразу используйте `List`             |

## 11. Краткое резюме

1. **`{1, 2, 3}`** — литерал Set. Пустой: `<int>{}`, НЕ `{}` (это Map).
2. **Уникальность** через `hashCode` + `==`. Мутабельные объекты опасны.
3. **O(1)** `contains`, `add`, `remove` — в среднем случае.
4. **Операции множеств** — `union`, `intersection`, `difference` — встроены.
5. **LinkedHashSet** по умолчанию — сохраняет порядок вставки.
6. **`SplayTreeSet`** — для сортированных множеств.
7. **Для дедупликации** — `list.toSet().toList()`.
