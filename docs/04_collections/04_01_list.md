# 4.1 List — списки

## 1. Формальное определение

**List\<E\>** — упорядоченная индексируемая коллекция элементов типа `E`. Элементы доступны по нулевому индексу. List может быть фиксированной длины (fixed-length) или растущей (growable, по умолчанию).

В Dart `List` — это то, что в других языках называется «массив» (array) или «вектор» (vector). Литеральный синтаксис — квадратные скобки: `[1, 2, 3]`.

**Сигнатура:** `abstract class List<E> implements EfficientLengthIterable<E>`

## 2. Зачем это нужно

- **Хранение упорядоченных данных** — последовательности элементов с гарантированным порядком.
- **Произвольный доступ** — O(1) доступ по индексу.
- **Основа коллекционного API** — большинство методов `Iterable` работают через List.
- **Flutter** — виджеты `children: [...]`, `ListView.builder` принимает список данных.
- **Универсальность** — от стека до буфера, List покрывает большинство сценариев.

## 3. Как это работает

### Создание

```dart
void main() {
  // Литерал (growable)
  var numbers = [1, 2, 3];                    // List<int>
  var mixed = <Object>[1, 'two', true];        // List<Object>

  // Пустой список с явной типизацией
  var empty = <String>[];
  List<double> prices = [];

  // Конструкторы
  var filled = List<int>.filled(5, 0);         // [0, 0, 0, 0, 0] (fixed-length)
  var generated = List<int>.generate(5, (i) => i * 2); // [0, 2, 4, 6, 8]
  var unmod = List<int>.unmodifiable([1, 2, 3]); // Неизменяемый

  // Из другого Iterable
  var fromSet = List<int>.of({3, 1, 2});       // [3, 1, 2]
}
```

### Основные операции

```dart
void main() {
  var list = [10, 20, 30, 40, 50];

  // Доступ
  print(list[0]);        // 10
  print(list.first);     // 10
  print(list.last);      // 50
  print(list.length);    // 5

  // Модификация
  list[1] = 25;          // Замена по индексу
  list.add(60);          // Добавить в конец
  list.addAll([70, 80]); // Добавить несколько
  list.insert(0, 5);     // Вставить по индексу
  list.removeAt(0);      // Удалить по индексу
  list.remove(30);       // Удалить по значению (первое вхождение)
  list.removeLast();     // Удалить последний
  list.removeWhere((e) => e > 60); // Удалить по условию

  // Поиск
  print(list.contains(25));      // true
  print(list.indexOf(25));       // индекс или -1
  print(list.any((e) => e > 40)); // true
  print(list.every((e) => e > 0)); // true

  // Подсписок
  var sub = list.sublist(1, 3);  // Новый List из [1..3)

  // Сортировка (in-place)
  list.sort();
  list.sort((a, b) => b.compareTo(a)); // По убыванию
}
```

### Growable vs Fixed-length

```dart
void main() {
  // Growable (по умолчанию)
  var grow = [1, 2, 3];
  grow.add(4); // OK

  // Fixed-length
  var fixed = List<int>.filled(3, 0);
  // fixed.add(4); // UnsupportedError!
  fixed[0] = 10; // OK — менять элементы можно

  // Unmodifiable — нельзя ни менять, ни добавлять
  var unmod = List<int>.unmodifiable([1, 2, 3]);
  // unmod[0] = 10; // UnsupportedError!
  // unmod.add(4);  // UnsupportedError!
}
```

## 4. Минимальный пример

```dart
void main() {
  var fruits = ['яблоко', 'банан', 'вишня'];

  fruits.add('дыня');
  fruits.removeAt(1);

  for (final fruit in fruits) {
    print(fruit);
  }
  // яблоко
  // вишня
  // дыня
}
```

## 5. Практический пример

### Менеджер задач с приоритетами

```dart
class Task implements Comparable<Task> {
  final String title;
  final int priority; // 1 = высший
  bool isDone;

  Task(this.title, {this.priority = 3, this.isDone = false});

  @override
  int compareTo(Task other) => priority.compareTo(other.priority);

  @override
  String toString() => '[$priority] $title${isDone ? " ✓" : ""}';
}

class TaskManager {
  final List<Task> _tasks = [];

  void add(String title, {int priority = 3}) {
    _tasks.add(Task(title, priority: priority));
  }

  void complete(int index) {
    if (index >= 0 && index < _tasks.length) {
      _tasks[index].isDone = true;
    }
  }

  List<Task> get pending =>
      _tasks.where((t) => !t.isDone).toList()..sort();

  List<Task> get completed =>
      _tasks.where((t) => t.isDone).toList();

  int get totalCount => _tasks.length;
  int get pendingCount => _tasks.where((t) => !t.isDone).length;
}

void main() {
  final manager = TaskManager();

  manager.add('Купить молоко', priority: 2);
  manager.add('Написать отчёт', priority: 1);
  manager.add('Полить цветы', priority: 3);
  manager.add('Ответить на письмо', priority: 1);

  manager.complete(2);

  print('Ожидают (${manager.pendingCount}):');
  for (final task in manager.pending) {
    print('  $task');
  }

  print('\nВыполнено (${manager.completed.length}):');
  for (final task in manager.completed) {
    print('  $task');
  }
}
```

## 6. Что происходит под капотом

### Внутренняя структура

```
Growable List:
  ┌──────────────┐
  │ _GrowableList │
  │   length: 3   │
  │   data ──────────► [ slot0 | slot1 | slot2 | ---- | ---- ]
  └──────────────┘      │                               capacity: 5
                         └─ backing array (Dart VM: _List / fixed array)

При add() когда length == capacity:
  1. Создаётся новый backing array размером capacity * 2 (или * 1.5)
  2. Элементы копируются
  3. Старый массив → GC
```

### Amortized O(1) для add

Стратегия удвоения гарантирует, что `add()` в среднем работает за O(1):

- Capacity 4 → 8 → 16 → 32 → 64 ...
- Копирование происходит редко, но при копировании O(n)
- Среднее: O(1) amortized

### Dart VM vs dart2js

| Аспект       | Dart VM                   | dart2js                        |
| ------------ | ------------------------- | ------------------------------ |
| List backing | `_GrowableList` → `_List` | JavaScript Array               |
| Fixed-length | `_List` (C++ backing)     | Array с заблокированным `push` |
| Unmodifiable | `_ImmutableList`          | Frozen JS Array                |
| `sort()`     | Dual-pivot Quicksort      | `Array.prototype.sort`         |

## 7. Производительность и ресурсы

| Операция             | Сложность      | Комментарий          |
| -------------------- | -------------- | -------------------- |
| `list[i]`            | O(1)           | Прямой доступ        |
| `list.add(e)`        | O(1) amortized | Иногда O(n) при grow |
| `list.insert(0, e)`  | O(n)           | Сдвиг всех элементов |
| `list.removeAt(0)`   | O(n)           | Сдвиг всех элементов |
| `list.removeLast()`  | O(1)           |                      |
| `list.contains(e)`   | O(n)           | Линейный поиск       |
| `list.indexOf(e)`    | O(n)           | Линейный поиск       |
| `list.sort()`        | O(n log n)     | Dual-pivot Quicksort |
| `list.sublist(s, e)` | O(e - s)       | Копирование          |

**Рекомендации:**

- Если нужны частые вставки/удаления в начало — используйте `Queue` из `dart:collection`.
- Для частого поиска по значению — `Set` эффективнее.
- `List.filled(n, e)` выделяет точный размер → нет лишних realloc.

## 8. Частые ошибки и антипаттерны

### ❌ Модификация списка при итерации

```dart
var list = [1, 2, 3, 4, 5];
// Concurrent modification error:
for (final item in list) {
  if (item.isEven) list.remove(item); // ConcurrentModificationError!
}

// ✅ Правильно: removeWhere
list.removeWhere((item) => item.isEven);

// ✅ Или создать новый список
var odd = list.where((item) => item.isOdd).toList();
```

### ❌ Создание списка через `List()` конструктор (Dart 2.x)

```dart
// Устарело и удалено:
// var list = List<int>(); // Error в Dart 3

// ✅ Правильно:
var list = <int>[];
```

### ❌ Повторное toList() без необходимости

```dart
// Плохо: двойная аллокация
var result = data.where((e) => e > 0).toList().map((e) => e * 2).toList();

// Лучше: ленивая цепочка
var result = data.where((e) => e > 0).map((e) => e * 2).toList();
```

### ❌ Ошибка с unmodifiable view

```dart
var original = [1, 2, 3];
var view = List<int>.unmodifiable(original);

original.add(4); // original изменился, но view НЕ обновился!
// view — это копия, не view. Название вводит в заблуждение.
```

## 9. Сравнение с альтернативами

| Аспект        | Dart `List`     | Java `ArrayList`     | JS `Array`      | Python `list`    |
| ------------- | --------------- | -------------------- | --------------- | ---------------- |
| Типизация     | `List<int>`     | `ArrayList<Integer>` | Слабая          | Нет              |
| Литерал       | `[1, 2]`        | ❌ (Arrays.asList)   | `[1, 2]`        | `[1, 2]`         |
| Фиксированный | `List.filled()` | ❌ (`int[]`)         | ❌              | ❌               |
| Сортировка    | Dual-pivot QS   | TimSort              | Engine-specific | TimSort          |
| Spread        | `[...a, ...b]`  | ❌                   | `[...a, ...b]`  | `[*a, *b]`       |
| collection if | `[if (c) x]`    | ❌                   | ❌              | `[x for ... if]` |

Dart уникален поддержкой collection if/for непосредственно в литералах — ни Java, ни JS этого не имеют.

## 10. Когда НЕ стоит использовать

| Сценарий                                | Вместо List используйте                         |
| --------------------------------------- | ----------------------------------------------- |
| Уникальные элементы                     | `Set<E>`                                        |
| Ключ→значение                           | `Map<K, V>`                                     |
| FIFO очередь                            | `Queue<E>` из `dart:collection`                 |
| Большие двоичные данные                 | `Uint8List`, `Float64List` из `dart:typed_data` |
| Linked list (частые вставки в середину) | `LinkedList` из `dart:collection`               |
| Неизменяемый список (контракт)          | `const [...]` или `List.unmodifiable()`         |

## 11. Краткое резюме

1. **`[1, 2, 3]`** — литерал growable-списка. `<int>[]` — пустой.
2. **Growable vs fixed-length vs unmodifiable** — три режима. Growable по умолчанию.
3. **O(1) доступ по индексу**, O(1) amortized `add()`, O(n) `insert(0, ...)`.
4. **`List.generate()`** — создание с вычисляемыми значениями.
5. **Нельзя модифицировать при итерации** — используйте `removeWhere` или создавайте новый список.
6. **Под капотом** — backing array с удвоением ёмкости. На JS — нативный `Array`.
7. **Dart 3** — `List()` конструктор без аргументов удалён, используйте `[]`.
