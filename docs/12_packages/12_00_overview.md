# 12. Публикация и управление пакетами — обзор

## О чём этот раздел

Dart-экосистема построена вокруг пакетов. Каждый проект — это пакет, описанный `pubspec.yaml`. Репозиторий пакетов — [pub.dev](https://pub.dev) — хранит тысячи библиотек с версионированием, документацией и рейтингом.

## Ключевые понятия

```
┌─────────────────────── Пакет ───────────────────────┐
│                                                      │
│  pubspec.yaml       ← идентичность пакета            │
│  lib/               ← публичный API                  │
│  lib/src/           ← приватная реализация            │
│  test/              ← тесты                          │
│  example/           ← примеры использования           │
│  README.md          ← описание                       │
│  CHANGELOG.md       ← история изменений              │
│  LICENSE            ← лицензия                       │
│  analysis_options.yaml ← настройки линтера           │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## Виды пакетов

| Вид | Содержит | Пример |
|-----|----------|--------|
| Library package | `lib/` с публичным API | `http`, `test`, `riverpod` |
| Application package | `bin/` или `web/` | Ваше приложение |
| Plugin package | Платформо-специфичный код | `path_provider`, `url_launcher` |

## Основные команды

```bash
# Создание нового пакета
dart create -t package my_package

# Получение зависимостей
dart pub get

# Обновление зависимостей
dart pub upgrade

# Проверка перед публикацией
dart pub publish --dry-run

# Публикация на pub.dev
dart pub publish

# Поиск пакетов
dart pub search http
```
