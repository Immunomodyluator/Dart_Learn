# 9. Асинхронность и конкурентность — обзор

## О чём этот раздел

Dart — однопоточный язык с **кооперативной многозадачностью** через event loop. Асинхронные операции (I/O, сеть, таймеры) не блокируют поток — результаты доставляются через `Future` и `Stream`. Для CPU-bound задач существуют **Isolates** — изолированные потоки со своей памятью.

## Как устроена асинхронность в Dart

```
┌──────────────────────────────────────────────────┐
│                   Main Isolate                    │
│                                                   │
│   ┌───────────┐   ┌───────────┐   ┌───────────┐  │
│   │ Microtask  │──▶│  Event    │──▶│  Event    │  │
│   │   Queue    │   │  Queue    │   │  (idle)   │  │
│   └───────────┘   └───────────┘   └───────────┘  │
│         ▲                ▲                        │
│         │                │                        │
│    scheduleMicro-   Future.then,                  │
│    task(), Future    Timer, I/O,                   │
│    .value, etc.     Stream events                 │
│                                                   │
├───────────────────────────────────────────────────┤
│          ┌─────────────────────────┐              │
│          │   Worker Isolate(s)     │              │
│          │   (отдельная память,    │              │
│          │    свой event loop)     │              │
│          └─────────────────────────┘              │
└───────────────────────────────────────────────────┘
```

### Ключевые принципы

1. **Один поток** — `main()` и весь Dart-код выполняются в одном потоке (main isolate).
2. **Non-blocking I/O** — файловые и сетевые операции делегируются ОС; поток не блокируется.
3. **Future** — «обещание» результата, который появится позже.
4. **Stream** — последовательность асинхронных событий (0..N значений).
5. **Event loop** — бесконечный цикл: сначала microtask queue, потом event queue.
6. **Isolate** — полностью изолированный поток со своей кучей (heap); общение через сообщения.

### Быстрый пример

```dart
Future<String> fetchData() async {
  // Имитация сетевого запроса (не блокирует поток!)
  await Future.delayed(Duration(seconds: 1));
  return 'Данные получены';
}

Stream<int> countDown(int from) async* {
  for (var i = from; i >= 0; i--) {
    await Future.delayed(Duration(milliseconds: 500));
    yield i; // Каждое значение — отдельное событие
  }
}

void main() async {
  // Future — одно значение
  final data = await fetchData();
  print(data); // Данные получены

  // Stream — несколько значений
  await for (final n in countDown(3)) {
    print(n); // 3, 2, 1, 0
  }
}
```

## Порядок изучения

1. **Futures** → базовая единица асинхронности.
2. **async/await** → удобный синтаксис поверх Future.
3. **Streams** → реактивные последовательности данных.
4. **Isolates** → настоящий параллелизм для CPU-задач.
5. **Event loop** → глубинное понимание порядка выполнения.
