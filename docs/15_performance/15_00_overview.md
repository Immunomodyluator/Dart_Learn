# 15. Производительность и профилирование — обзор

## О чём этот раздел

Инструменты и техники для измерения, анализа и оптимизации производительности Dart-приложений. Профилирование CPU и памяти, устранение узких мест, понимание сборщика мусора.

## Ключевые понятия

```
┌────────────── Производительность Dart ─────────────┐
│                                                    │
│  Профилирование                                   │
│  ├── CPU profiler — горячие точки кода             │
│  ├── Memory profiler — утечки, аллокации           │
│  └── Timeline — визуализация событий              │
│                                                    │
│  Оптимизация                                       │
│  ├── Аллокации — меньше временных объектов         │
│  ├── Алгоритмы — правильные структуры данных       │
│  └── Async — устранение блокировок                 │
│                                                    │
│  GC (Garbage Collector)                            │
│  ├── Young space — частые мелкие сборки            │
│  ├── Old space — редкие полные сборки              │
│  └── Финализаторы — weak references               │
│                                                    │
└────────────────────────────────────────────────────┘
```
