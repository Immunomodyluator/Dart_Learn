### Вопрос 1 (4.1): Числа и операции

Что выведет следующий код?

```dart
print(7 / 2);
print(7 ~/ 2);
```

- A) `3` и `3`
- B) `3.5` и `3.5`
- C) `3.5` и `3`
- D) `3` и `3.5`

<details>
<summary>Правильный ответ</summary>

**Ответ: C) `3.5` и `3`**

В Dart оператор `/` всегда возвращает `double` (даже при делении двух `int`), поэтому `7 / 2 = 3.5`. Оператор `~/` — целочисленное деление (truncate division), результат `int`: `7 ~/ 2 = 3`.
</details>

### Вопрос 2 (4.2): Строки и интерполяция

Что выведет этот код?

```dart
var name = 'Dart';
var version = 3;
print('Hello, $name ${version > 2 ? "3+" : "2"}!');
```

- A) Ошибка компиляции: нельзя использовать выражения внутри `${}`
- B) `Hello, Dart 3+!`
- C) `Hello, $name 3+!`
- D) `Hello, Dart ${version > 2 ? "3+" : "2"}!`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `Hello, Dart 3+!`**

В Dart строковая интерполяция поддерживает произвольные выражения внутри `${}`, включая тернарный оператор. `$name` вычисляется как `Dart`, а `${version > 2 ? "3+" : "2"}` — как `3+`.
</details>

### Вопрос 3 (4.3): Булевы значения

Что произойдёт при компиляции этого кода?

```dart
var x = 1;
if (x) {
  print('truthy');
}
```

- A) Выведет `truthy` — в Dart `1` считается truthy как в JavaScript
- B) Ошибка компиляции: условие `if` должно быть типа `bool`, а не `int`
- C) Ничего не выведет — `1` не равно `true`
- D) Ошибка времени выполнения

<details>
<summary>Правильный ответ</summary>

**Ответ: B) Ошибка компиляции: условие `if` должно быть типа `bool`, а не `int`**

Dart строго типизирован: в `if` допускается только значение типа `bool`. В отличие от JavaScript или C, в Dart нет понятия «truthy/falsy» — число `1` не является `true`. Это предотвращает целый класс багов совместимости типов.
</details>

### Вопрос 4 (4.4): Runes и Symbols

Что такое `Runes` в Dart и зачем они нужны?

- A) Способ создать символьный литерал типа `char`, которого нет в Dart
- B) Итерируемая последовательность Unicode code points строки; нужна, когда один символ занимает более одной кодовой единицы UTF-16
- C) Метаданные для рефлексии, аналог аннотаций Java
- D) Специальный тип для работы с байтами в `dart:io`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) Итерируемая последовательность Unicode code points строки; нужна, когда один символ занимает более одной кодовой единицы UTF-16**

Строки Dart хранятся как последовательность кодовых единиц UTF-16. Многие эмодзи и редкие символы занимают 2 кодовые единицы (суррогатные пары). `String.runes` возвращает правильные code points, а не сырые `codeUnits`, что важно для корректного подсчёта символов.
</details>

### Вопрос 5 (4.5): Records (Dart 3)

Что выведет этот код в Dart 3?

```dart
var point = (x: 10, y: 20);
var (x: a, y: b) = point;
print('$a, $b');
```

- A) Ошибка компиляции: нельзя деструктурировать Record таким образом
- B) `x: 10, y: 20`
- C) `10, 20`
- D) `(10, 20)`

<details>
<summary>Правильный ответ</summary>

**Ответ: C) `10, 20`**

В Dart 3 Record с именованными полями `(x: 10, y: 20)` деструктурируется паттерном `(x: a, y: b)`, где `a` и `b` — новые локальные переменные, получающие значения полей `x` и `y` соответственно. Результат — `print('$a, $b')` выводит `10, 20`.
</details>
