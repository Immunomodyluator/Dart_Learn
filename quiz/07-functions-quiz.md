### Вопрос 1 (7.1): Синтаксис функций

Какие из следующих объявлений функций эквивалентны?

```dart
// Вариант 1
int double1(int x) { return x * 2; }

// Вариант 2
int double2(int x) => x * 2;
```

- A) Не эквивалентны: стрелочная функция не поддерживает возвращаемый тип
- B) Эквивалентны: `=>` — сахар для `{ return expr; }` с одним выражением
- C) Не эквивалентны: `double2` вернёт `null` при вызове
- D) Эквивалентны, но `double2` работает только в JIT-режиме

<details>
<summary>Правильный ответ</summary>

**Ответ: B) Эквивалентны: `=>` — сахар для `{ return expr; }` с одним выражением**

Стрелочный синтаксис `=> expr` полностью эквивалентен `{ return expr; }` — это просто краткая форма записи. Оба варианта типизированы одинаково и работают идентично.
</details>

### Вопрос 2 (7.2): Параметры: позиционные, именованные, опциональные

Что выведет этот код?

```dart
void greet(String name, {String greeting = 'Hello'}) {
  print('$greeting, $name!');
}

greet('Alice');
greet('Bob', greeting: 'Hi');
```

- A) Ошибка компиляции: именованные параметры должны быть `required`
- B) `Hello, Alice!` и `Hi, Bob!`
- C) `Hello, Alice!` и `Hello, Bob!`
- D) `null, Alice!` и `Hi, Bob!`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `Hello, Alice!` и `Hi, Bob!`**

`{String greeting = 'Hello'}` — именованный параметр с дефолтным значением. При вызове `greet('Alice')` параметр `greeting` принимает значение по умолчанию `'Hello'`. При `greet('Bob', greeting: 'Hi')` передаётся явное значение `'Hi'`.
</details>

### Вопрос 3 (7.3): Замыкания и лексическая область

Что выведет код?

```dart
List<Function> funcs = [];
for (var i = 0; i < 3; i++) {
  funcs.add(() => print(i));
}
funcs[0]();
funcs[1]();
funcs[2]();
```

- A) `0`, `1`, `2`
- B) `3`, `3`, `3`
- C) `2`, `2`, `2`
- D) Ошибка времени выполнения

<details>
<summary>Правильный ответ</summary>

**Ответ: A) `0`, `1`, `2`**

В Dart `var` в `for`-цикле создаёт новую переменную на каждую итерацию (в отличие от JavaScript `var`). Каждое замыкание захватывает свою собственную копию `i`. Поэтому `funcs[0]()` выводит `0`, `funcs[1]()` — `1`, `funcs[2]()` — `2`.
</details>

### Вопрос 4 (7.4): Типы функций и typedef

Для чего используется `typedef` в следующем примере?

```dart
typedef Predicate<T> = bool Function(T value);

bool filter(List<int> list, Predicate<int> test) {
  return list.any(test);
}
```

- A) `typedef` создаёт новый тип, несовместимый с `bool Function(int)`
- B) `typedef` — псевдоним для сложного типа функции; улучшает читаемость, но не создаёт новый тип
- C) `typedef` нужен только для совместимости с Dart 1.x
- D) `typedef` здесь запрещает передавать лямбды, только именованные функции

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `typedef` — псевдоним для сложного типа функции; улучшает читаемость, но не создаёт новый тип**

`typedef` в Dart — это alias для типа (в т.ч. function type). `Predicate<int>` и `bool Function(int)` — полностью совместимые типы, можно передавать лямбды напрямую. Основная польза — именовать сложные сигнатуры и делать код самодокументируемым.
</details>

### Вопрос 5 (7.5): Генераторы: sync* и async*

Что вернёт эта функция и какого типа будет результат?

```dart
Iterable<int> range(int start, int end) sync* {
  for (var i = start; i < end; i++) {
    yield i;
  }
}
```

- A) `List<int>` — все значения вычислены сразу
- B) `Stream<int>` — значения передаются асинхронно
- C) `Iterable<int>` — ленивая последовательность, значения вычисляются по запросу
- D) `Future<Iterable<int>>` — нужно ждать завершения

<details>
<summary>Правильный ответ</summary>

**Ответ: C) `Iterable<int>` — ленивая последовательность, значения вычисляются по запросу**

`sync*` с `yield` создаёт синхронный генератор, возвращающий `Iterable<T>`. Важно: генератор выполняется лениво — тело функции не запускается до начала итерации. Каждый вызов `moveNext()` у итератора выполняет код до следующего `yield`. `async*` + `yield` создаёт `Stream<T>`.
</details>
