### Вопрос 1 (1.1): Установка SDK и редакторов

Вы установили Dart SDK и хотите убедиться, что всё работает. Какая команда выведет версию установленного SDK?

- A) `dart version`
- B) `dart --version`
- C) `dartc --version`
- D) `sdk version`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `dart --version`**

Флаг `--version` — стандартный способ проверить версию любого CLI-инструмента Dart. Команда `dart version` не существует, `dartc` — устаревший компилятор Dart 1.x.
</details>

### Вопрос 2 (1.2): Dart CLI и dartdev инструменты

Что делает команда `dart run bin/server.dart`?

- A) Компилирует файл в нативный бинарник и сразу запускает его
- B) Запускает файл через JIT-компилятор в рамках Dart VM
- C) Запускает файл только если он предварительно скомпилирован командой `dart compile`
- D) Создаёт snapshot-файл и запускает его

<details>
<summary>Правильный ответ</summary>

**Ответ: B) Запускает файл через JIT-компилятор в рамках Dart VM**

`dart run` запускает исходный `.dart`-файл напрямую через JIT без предварительной компиляции. Это основной режим разработки — быстрый старт, полная поддержка hot reload. Нативный бинарник создаётся командой `dart compile exe`.
</details>

### Вопрос 3 (1.3): pub и pubspec.yaml

Что произойдёт, если выполнить `dart pub get` после добавления новой зависимости в `pubspec.yaml`?

- A) Зависимость будет скачана и добавлена в `pubspec.lock`, но не станет доступна в коде до перезапуска IDE
- B) Зависимость будет скачана в `~/.pub-cache`, а `pubspec.lock` — обновлён с точной версией
- C) Зависимость скачается прямо в папку проекта `lib/packages/`
- D) `pubspec.lock` будет удалён и пересоздан заново только с новой зависимостью

<details>
<summary>Правильный ответ</summary>

**Ответ: B) Зависимость будет скачана в `~/.pub-cache`, а `pubspec.lock` — обновлён с точной версией**

`dart pub get` загружает пакеты в глобальный кэш `~/.pub-cache` (не в папку проекта) и обновляет `pubspec.lock` с зафиксированными точными версиями всех транзитивных зависимостей. Папки `lib/packages/` в Dart не существует.
</details>

### Вопрос 4 (1.4): Форматирование, анализ и линтеры

Какая команда проверит форматирование кода, но **не будет** его изменять (только сообщит об ошибках)?

- A) `dart format --check .`
- B) `dart format --dry-run .`
- C) `dart format --output=none --set-exit-if-changed .`
- D) `dart analyze --format .`

<details>
<summary>Правильный ответ</summary>

**Ответ: C) `dart format --output=none --set-exit-if-changed .`**

Флаг `--output=none` запрещает запись изменённых файлов, а `--set-exit-if-changed` возвращает ненулевой exit code при обнаружении нарушений форматирования. Именно эта комбинация используется в CI-пайплайнах. Флаг `--check` не существует в `dart format`.
</details>
