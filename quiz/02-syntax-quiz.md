### Вопрос 1 (2.1): Hello World и структура программы

Что является обязательным для запуска Dart-программы как исполняемого скрипта?

- A) Класс с именем `Main` и метод `run()`
- B) Функция верхнего уровня `main()` без параметров или с параметром `List<String> args`
- C) Функция `main()` обязательно должна быть внутри класса
- D) Файл должен называться `main.dart`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) Функция верхнего уровня `main()` без параметров или с параметром `List<String> args`**

В Dart точка входа — функция верхнего уровня `main`. Она может быть без параметров или принимать `List<String> args` для чтения аргументов командной строки. Имя файла не имеет значения; класс с методом `run()` — это паттерн Java, не Dart.

</details>

### Вопрос 2 (2.2): Объявление переменных

Что выведет этот код?

```dart
var x = 10;
dynamic y = 10;
x = 'hello';  // строка 3
y = 'hello';  // строка 4
print(y);
```

- A) `hello` — обе строки выполняются успешно
- B) Ошибка компиляции на строке 3
- C) Ошибка компиляции на строке 4
- D) Ошибка времени выполнения на строке 3

<details>
<summary>Правильный ответ</summary>

**Ответ: B) Ошибка компиляции на строке 3**

`var x = 10` — Dart выводит тип как `int`. Попытка присвоить `String` переменной типа `int` — ошибка компиляции. `dynamic y`, напротив, отключает статическую проверку типов: любое присваивание проходит, и при несовпадении типов ошибка возникла бы только в runtime.

</details>

### Вопрос 3 (2.3): Типы и вывод типов

Какой тип будет у переменной `result` после этого кода?

```dart
var items = [1, 2, 3];
var result = items.map((e) => e * 2);
```

- A) `List<int>`
- B) `Iterable<int>`
- C) `List<dynamic>`
- D) `Iterable<dynamic>`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `Iterable<int>`**

Метод `map()` возвращает `Iterable<T>`, а не `List<T>`. Dart выводит тип параметра как `int`, потому что `items` имеет тип `List<int>`. Чтобы получить `List<int>`, нужно явно вызвать `.toList()`.

</details>

### Вопрос 4 (2.4): const и final

В чём ключевое отличие `const` от `final`?

- A) `final` можно использовать только для примитивных типов, `const` — для любых
- B) `const` — константа времени компиляции, `final` — значение фиксируется в runtime при первом присваивании
- C) `const` и `final` полностью эквивалентны, разница только в синтаксисе
- D) `final` нельзя использовать как параметр конструктора

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `const` — константа времени компиляции, `final` — значение фиксируется в runtime при первом присваивании**

`const` вычисляется во время компиляции и разделяется в памяти (canonicalized). `final` означает, что переменная присваивается ровно один раз, но значение может быть известно только в runtime — например, `final now = DateTime.now()`. Поэтому `const DateTime.now()` вызвало бы ошибку.

</details>

### Вопрос 5 (2.5): Null Safety

Что выведет следующий код?

```dart
String? name;
print(name?.length ?? 0);
```

- A) Ошибка компиляции: нельзя вызывать `.length` на nullable значении без `!`
- B) `null`
- C) `0`
- D) Ошибка времени выполнения: Null check operator used on null value

<details>
<summary>Правильный ответ</summary>

**Ответ: C) `0`**

Оператор `?.` — null-aware: если `name` равно `null`, выражение `name?.length` вернёт `null`, а не вызовет ошибку. Затем оператор `??` возвращает правую часть (`0`) когда левая равна `null`. Результат: выводится `0`.

</details>
