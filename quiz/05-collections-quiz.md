### Вопрос 1 (5.1): List — списки

Что выведет этот код?

```dart
final list = [1, 2, 3];
list.add(4);
print(list);
```

- A) Ошибка компиляции: `final` список нельзя изменять
- B) `[1, 2, 3, 4]`
- C) Ошибка времени выполнения: нельзя изменить `final` переменную
- D) `[1, 2, 3]`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `[1, 2, 3, 4]`**

`final` запрещает переприсвоение переменной (нельзя сделать `list = [...]`), но **не** делает содержимое списка иммутабельным. Сам объект списка можно мутировать — вызывать `add`, `remove` и т.д. Для создания неизменяемого списка используется `List.unmodifiable(...)` или `const [...]`.
</details>

### Вопрос 2 (5.2): Set — множества

Что выведет следующий код?

```dart
final s = {3, 1, 2, 1, 3};
print(s.length);
print(s.first);
```

- A) `5` и `3`
- B) `3` и `3`
- C) `3` и `1`
- D) `2` и `1`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `3` и `3`**

`Set` хранит только уникальные значения, поэтому `{3, 1, 2, 1, 3}` даёт множество из 3 элементов. По умолчанию Dart создаёт `LinkedHashSet`, который сохраняет порядок вставки. Первым вставлено `3`, значит `s.first == 3`.
</details>

### Вопрос 3 (5.3): Map — карты

Что выведет код при обращении к несуществующему ключу?

```dart
final map = {'a': 1, 'b': 2};
print(map['c']);
```

- A) Ошибка компиляции
- B) Ошибка времени выполнения: `KeyError`
- C) `null`
- D) `0`

<details>
<summary>Правильный ответ</summary>

**Ответ: C) `null`**

В Dart оператор `[]` для `Map` возвращает `T?` — nullable тип. При отсутствии ключа метод возвращает `null`, а не бросает исключение. Поэтому тип `map['c']` — `int?`, а значение — `null`. Чтобы получить исключение при отсутствии ключа, используется `map['c']!` (или проверка перед использованием).
</details>

### Вопрос 4 (5.4): Методы коллекций высшего порядка

Что выведет этот код?

```dart
final result = [1, 2, 3, 4, 5]
    .where((e) => e.isOdd)
    .map((e) => e * 10)
    .toList();
print(result);
```

- A) `[10, 20, 30, 40, 50]`
- B) `[10, 30, 50]`
- C) `[1, 3, 5]`
- D) Ошибка: нельзя цепочкой вызывать `where` и `map`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `[10, 30, 50]`**

`where((e) => e.isOdd)` отбирает нечётные: `[1, 3, 5]`. Затем `map((e) => e * 10)` умножает каждый на 10: `[10, 30, 50]`. Методы возвращают ленивые `Iterable`, которые материализуются в `List` через `.toList()`.
</details>

### Вопрос 5 (5.5): Литералы с условиями и spread

Что создаст следующий код?

```dart
final extras = [4, 5];
final includeExtras = true;
final list = [1, 2, 3, if (includeExtras) ...extras];
print(list);
```

- A) Ошибка компиляции: `if` нельзя использовать внутри литерала списка
- B) `[1, 2, 3]` — `if` в литерале работает только с `const`
- C) `[1, 2, 3, [4, 5]]`
- D) `[1, 2, 3, 4, 5]`

<details>
<summary>Правильный ответ</summary>

**Ответ: D) `[1, 2, 3, 4, 5]`**

В Dart поддерживаются collection `if` и spread (`...`) прямо в литерале коллекции. `if (includeExtras) ...extras` разворачивает список `extras` в родительский список, только если условие истинно. Результат: `[1, 2, 3, 4, 5]`.
</details>
