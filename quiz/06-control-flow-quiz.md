### Вопрос 1 (6.1): if / else и тернарный оператор

Что выведет код?

```dart
int x = 5;
String label = x > 3 ? x > 7 ? 'large' : 'medium' : 'small';
print(label);
```

- A) `large`
- B) `medium`
- C) `small`
- D) Ошибка компиляции: вложенные тернарные операторы запрещены

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `medium`**

Тернарный оператор ассоциируется справа: `x > 3` — истина (`5 > 3`), затем `x > 7` — ложь (`5 > 7`), поэтому выбирается `'medium'`. Вложенные тернарники синтаксически разрешены в Dart (хотя style guide их не рекомендует).
</details>

### Вопрос 2 (6.2): switch: statements и expressions (Dart 3)

Что выведет этот код на Dart 3?

```dart
int code = 200;
String status = switch (code) {
  200 => 'OK',
  404 => 'Not Found',
  500 => 'Server Error',
  _ => 'Unknown',
};
print(status);
```

- A) Ошибка компиляции: `switch` нельзя использовать как выражение
- B) `OK`
- C) `Unknown`
- D) `null`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `OK`**

В Dart 3 появились switch expressions — `switch` можно использовать как выражение, возвращающее значение. Паттерн `_` — wildcard, совпадает с любым значением. Здесь `code = 200` совпадает с первой веткой `200 => 'OK'`.
</details>

### Вопрос 3 (6.3): Паттерны и деструктуризация (Dart 3)

Что выведет этот код?

```dart
final point = (1, 2);
final (x, y) = point;
print('$x $y');
```

- A) `(1, 2)`
- B) `1 2`
- C) Ошибка компиляции: деструктуризация кортежей не поддерживается
- D) `x y`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `1 2`**

В Dart 3 позиционный Record `(1, 2)` деструктурируется паттерном `(x, y)`, где `x` и `y` становятся локальными переменными со значениями `1` и `2`. Затем `'$x $y'` выводит `1 2`.
</details>

### Вопрос 4 (6.4): if-case и guard clauses (Dart 3)

Какой синтаксис корректен для `if-case` с дополнительным условием в Dart 3?

- A) `if (value case int n && n > 0) { ... }`
- B) `if (value case int n where n > 0) { ... }`
- C) `if (value case int n when n > 0) { ... }`
- D) `if (value is int n, n > 0) { ... }`

<details>
<summary>Правильный ответ</summary>

**Ответ: C) `if (value case int n when n > 0) { ... }`**

В Dart 3 guard clause в `if-case` и в ветках `switch` записывается через ключевое слово `when`. Конструкция сначала проверяет паттерн (`int n` — является ли значение `int`), а затем дополнительное условие (`n > 0`). Оба условия должны быть истинны для входа в блок.
</details>

### Вопрос 5 (6.5): Циклы: for, for-in, while

Что выведет этот код?

```dart
for (var i = 0; i < 3; i++) {
  if (i == 1) continue;
  print(i);
}
```

- A) `0`, `1`, `2`
- B) `0`, `2`
- C) `0`
- D) `1`, `2`

<details>
<summary>Правильный ответ</summary>

**Ответ: B) `0`, `2`**

`continue` пропускает оставшуюся часть текущей итерации и переходит к следующей. При `i == 1` вызов `print(1)` пропускается. Итого выводятся `0` (первая итерация) и `2` (третья итерация).
</details>

### Вопрос 6 (6.6): assert и отладочные проверки

В каком режиме выполняются проверки `assert`?

- A) Всегда — и в debug, и в release
- B) Только в release-сборках для повышения надёжности
- C) Только в debug-режиме; в production `assert` полностью игнорируются
- D) `assert` в Dart — синтаксический сахар над `if + throw`, всегда выполняется

<details>
<summary>Правильный ответ</summary>

**Ответ: C) Только в debug-режиме; в production `assert` полностью игнорируются**

`assert` в Dart активны только при запуске с флагом `--enable-asserts` (или режим `dart run` в разработке). В AOT-скомпилированных бинарниках (`dart compile exe`) и Flutter release-сборках `assert` не выполняются. Это позволяет использовать их для документирования инвариантов без влияния на производительность продакшна.
</details>
