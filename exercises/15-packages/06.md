# Задание 06 (15.3): Семантическое версионирование — 🐛 Найти баг

**Тема:** 15. Публикация и управление пакетами | **Подтема:** 15.3 Семантическое версионирование

---

В CHANGELOG и `pubspec.yaml` есть ошибки с версионированием. Найди все проблемы.

```yaml
# pubspec.yaml пакета my_lib
name: my_lib
version: 1.5.3

dependencies:
  http:
    ^0.13.0 # bug 1 — ^0.13.0 означает >=0.13.0 <0.14.0 (не <1.0.0!)
    #         нужно обновить до актуальной версии http ^1.0.0
  meta: 1.9.0 # bug 2 — нет caret: патч-обновления безопасности заблокированы
  path: ">=1.0.0" # bug 3 — нет верхней границы: вдруг выйдет path 3.0 с breaking changes
```

```markdown
<!-- CHANGELOG.md -->

## 1.5.3 — 2024-05-01 ← текущая версия

### Fixed

- parseDate() теперь поддерживает ISO 8601

## 1.5.2 — 2024-04-15

### Added

- Добавлен класс DateRange (новый класс — публичное API) <!-- bug 4: неверная версия! -->
- Добавлены методы start и end в DateRange

## 1.5.0 — 2024-04-01

### Changed

- Переименован параметр format → pattern в функции formatDate() <!-- bug 5: breaking change! -->

## 1.4.0 — 2024-03-01

### Removed

- Удалён метод legacyParse() (был @deprecated с 1.0.0) <!-- bug 6: breaking change! -->
```

<summary>Решение</summary>

```yaml
# pubspec.yaml — исправленная версия
name: my_lib
version: 2.0.0 # см. ниже — версия должна быть 2.x из-за breaking changes

dependencies:
  # fix 1: обновляем до актуальной мажорной версии http
  http: ^1.2.0 # ^1.2.0 = >=1.2.0 <2.0.0

  # fix 2: добавляем caret для патч-обновлений
  meta: ^1.9.0

  # fix 3: добавляем верхнюю границу
  path: ^1.9.0 # или '>=1.9.0 <2.0.0'
```

```markdown
<!-- CHANGELOG.md — исправленная версия -->

## 2.0.0 — 2024-05-01 ← major из-за breaking changes

### Breaking Changes

- Переименован параметр `format` → `pattern` в `formatDate()` (было в 1.5.0 — ошибка версии!)
- Удалён метод `legacyParse()` (был @deprecated — удаление тоже breaking change!)

### Fixed

- `parseDate()` теперь поддерживает ISO 8601

## 1.6.0 — 2024-04-15 ← minor (добавление нового класса)

### Added

- Добавлен класс `DateRange` с методами `start` и `end`

## 1.5.0 — 2024-04-01 ← оригинальная 1.5.0 должна была быть 2.0.0!

(и т.д.)
```

**Шесть проблем:** 1) `^0.13.0` для версий `0.x.y` работает иначе: `^0.13.0 = >=0.13.0 <0.14.0` — очень узкий диапазон; нужно обновить до актуальной `^1.x`. 2) `meta: 1.9.0` без `^` — точная версия блокирует любые обновления. 3) `path: '>=1.0.0'` без верхней границы — опасно: мажорное обновление `path 3.0` с breaking changes попадёт автоматически. 4) Добавление нового публичного класса `DateRange` — это MINOR изменение, версия должна быть `1.6.0`, а не `1.5.2`. 5) Переименование параметра — breaking change → MAJOR (должна быть `2.0.0`, а не `1.5.0`). 6) Удаление метода, пусть даже `@deprecated` — тоже breaking change → MAJOR.
