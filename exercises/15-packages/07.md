Разберись с конфликтом зависимостей и реши его.

**Ситуация:** Проект использует пакеты `pkg_a` (требует `http: ^1.0.0`) и `pkg_b` (требует `http: ^0.13.0`). `dart pub get` выдаёт ошибку несовместимости.

**Задача:**

1. Объясни, почему возникает конфликт и что означает ошибка:

   ```
   Because pkg_a >=1.0.0 depends on http ^1.0.0 and
   pkg_b >=0.5.0 depends on http ^0.13.0,
   pkg_a >=1.0.0 is incompatible with pkg_b >=0.5.0.
   ```

2. Напиши все способы решения конфликта (команды и изменения в `pubspec.yaml`).

3. Напиши скрипт `pubspec_check.dart`, который читает `pubspec.yaml` и выводит предупреждение для зависимостей с версией `any` или без верхней границы. Используй `dart:io` и `package:yaml`.

```dart
import 'dart:io';

```

<details>
<summary>Решение</summary>

**1. Объяснение конфликта:**

`pkg_a` требует `http: ^1.0.0` → допустимы версии `[1.0.0, 2.0.0)`.  
`pkg_b` требует `http: ^0.13.0` → допустимы версии `[0.13.0, 0.14.0)`.  
Эти диапазоны **не пересекаются** — `pub` не может выбрать единственную версию `http`, удовлетворяющую обоим.

**2. Способы решения:**

```bash
# Способ 1: обновить pkg_b (если автор выпустил новую версию совместимую с http ^1.0.0)
dart pub upgrade pkg_b

# Способ 2: использовать dependency_overrides (временный хак для разработки)
# В pubspec.yaml:
dependency_overrides:
  http: ^1.2.0   # принудительно использовать эту версию для всех

# Способ 3: заменить pkg_b на совместимый альтернативный пакет

# Способ 4: форкнуть pkg_b и обновить зависимость
dependency_overrides:
  pkg_b:
    git:
      url: https://github.com/my-fork/pkg_b
      ref: fix-http-dependency

# Проверить дерево зависимостей
dart pub deps
dart pub deps --style=tree
```

**3. Скрипт проверки pubspec.yaml:**

```dart
import 'dart:io';

void main() async {
  final file = File('pubspec.yaml');
  if (!await file.exists()) {
    print('pubspec.yaml not found');
    exit(1);
  }

  final content = await file.readAsString();
  final lines = content.split('\n');

  bool inDeps = false;
  int warnings = 0;

  for (final (i, line) in lines.indexed) {
    final trimmed = line.trim();

    // Определяем секцию зависимостей
    if (trimmed == 'dependencies:' || trimmed == 'dev_dependencies:') {
      inDeps = true;
      continue;
    }
    if (inDeps && !line.startsWith(' ') && trimmed.isNotEmpty) {
      inDeps = false;
    }

    if (!inDeps) continue;

    // Проверяем версионные ограничения
    if (trimmed.contains(': any')) {
      print('WARNING line ${i + 1}: version "any" is unsafe — $trimmed');
      warnings++;
    } else if (RegExp(r": '?>=\d[\d.]*'?$").hasMatch(trimmed)) {
      print('WARNING line ${i + 1}: no upper bound — $trimmed');
      warnings++;
    }
  }

  if (warnings == 0) {
    print('No dependency warnings found.');
  } else {
    print('$warnings warning(s) found.');
    exit(1);
  }
}
```

`dependency_overrides` — это **временный** инструмент: он позволяет принудительно установить версию для разрешения конфликта, но может скрыть настоящую несовместимость. Всегда документируй причину override и планируй его убрать после обновления зависимостей.
