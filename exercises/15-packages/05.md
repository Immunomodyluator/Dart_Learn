Заполни пропуски (`___`) с правильными версиями и обоснованием по правилам SemVer.

**Правила SemVer:** `MAJOR.MINOR.PATCH`
- PATCH: исправление бага, не меняющее публичное API
- MINOR: добавление новой функциональности, обратно совместимое  
- MAJOR: изменение, ломающее обратную совместимость

**Задача 1 — определи следующую версию:**

```
Текущая версия: 1.4.2

Изменение A: исправлен баг в функции parse() возвращавшей null для пустой строки
→ следующая версия: ___  (тип: ___)

Изменение B: добавлена новая функция formatDate() без изменения существующего API
→ следующая версия: ___  (тип: ___)

Изменение C: переименован обязательный параметр функции validate(data:) → validate(input:)
→ следующая версия: ___  (тип: ___)

Изменение D: удалён устаревший класс LegacyParser (был помечен @deprecated с v1.0.0)
→ следующая версия: ___  (тип: ___)
```

**Задача 2 — версионные ограничения в pubspec.yaml:**

```yaml
dependencies:
  # хочу получать только патч-обновления пакета http версии 1.2.0
  http: ___

  # разрешить любые минорные обновления начиная с 2.0.0, но не 3.x
  json_utils: ___

  # ровно версия 0.9.0-beta.1 (pre-release, без обновлений)
  experimental_pkg: ___

  # использовать пакет из локальной директории ../my_local_pkg
  my_local_pkg:
    ___: ../my_local_pkg
```

<summary>Решение</summary>

**Задача 1:**

```
Текущая версия: 1.4.2

Изменение A: исправлен баг
→ следующая версия: 1.4.3  (тип: PATCH)

Изменение B: новая функция formatDate(), обратно совместимое добавление
→ следующая версия: 1.5.0  (тип: MINOR — сбрасывает PATCH до 0)

Изменение C: переименован обязательный именованный параметр
→ следующая версия: 2.0.0  (тип: MAJOR — breaking change: старый вызов validate(data: x) сломается)

Изменение D: удалён @deprecated класс
→ следующая версия: 2.0.0  (тип: MAJOR — удаление публичного API = breaking change,
                              даже если он был помечен deprecated)
```

**Задача 2:**

```yaml
dependencies:
  # только патч-обновления 1.2.x: >=1.2.0 <1.3.0
  http: '>=1.2.0 <1.3.0'
  # или эквивалентно через tilde (не стандарт Dart но понятно концептуально):
  # http: ^1.2.0 разрешит 1.x.x ≥ 1.2.0 — это минорные, не только патч
  # точный патч-контроль: http: '>=1.2.0 <1.3.0'

  # минорные обновления 2.x начиная с 2.0.0
  json_utils: ^2.0.0   # ^2.0.0 → >=2.0.0 <3.0.0

  # ровно pre-release версия
  experimental_pkg: 0.9.0-beta.1

  # локальный пакет
  my_local_pkg:
    path: ../my_local_pkg
```

В Dart `^1.2.3` означает `>=1.2.3 <2.0.0` (для версий ≥1.0.0). Для pre-release версий (x.y.z-label) caret не включает их автоматически — нужно указывать явно. Локальные пакеты (`path:`) удобны при разработке монорепозитория. Breaking changes (удаление, переименование параметров, изменение типов) всегда требуют MAJOR-версии.
