# Задание 08 (15.4): Работа с зависимостями и конфликты — 🐛 Найти баг

**Тема:** 15. Публикация и управление пакетами | **Подтема:** 15.4 Работа с зависимостями и конфликты

---

В `pubspec.yaml` и `pubspec.lock` есть проблемы с зависимостями. Найди их все.

```yaml
# pubspec.yaml
name: my_app
version: 1.0.0

environment:
  sdk: '>=3.0.0 <4.0.0'

dependencies:
  http: ^1.2.0
  json_annotation: ^4.8.0
  
  # bug 1 — дублирование: http уже указан выше
  http: ^1.1.0

dev_dependencies:
  build_runner: ^2.4.0
  json_serializable: ^6.7.0
  
  # bug 2 — json_annotation должна быть в dependencies, не dev_dependencies
  # (она нужна в runtime, не только для генерации кода)
  json_annotation: ^4.8.0

  test: ^1.0.0

dependency_overrides:
  # bug 3 — override без причины нарушает совместимость
  # если http 1.3.0 вышел — caret автоматически его подхватит
  http: 1.2.3

flutter:
  # bug 4 — flutter секция в обычном Dart пакете (не Flutter), будет предупреждение
  uses-material-design: true
```

<details>
<summary>Решение</summary>

```yaml
# pubspec.yaml — исправленная версия
name: my_app
version: 1.0.0

environment:
  sdk: '>=3.0.0 <4.0.0'

dependencies:
  http: ^1.2.0              # fix 1: одна запись http (дубликат удалён)
                            # Dart pub берёт последнюю при дублях, но это баг в конфиге
  json_annotation: ^4.8.0   # fix 2: json_annotation ОСТАЁТСЯ в dependencies
                            # она нужна в runtime (аннотации @JsonSerializable и т.д.)

dev_dependencies:
  build_runner: ^2.4.0
  json_serializable: ^6.7.0  # json_serializable — только dev (генератор кода)
  test: ^1.0.0

# fix 3: убираем dependency_overrides — caret сам подхватит обновления
# dependency_overrides нужен только при реальном конфликте версий,
# не для «закрепления» конкретной версии

# fix 4: убираем flutter секцию — это не Flutter приложение
```

**Пояснения:**

```
bug 1: Два ключа http в одном YAML-маппинге — технически только последний учитывается,
       но это вводит в заблуждение и сигнализирует об ошибке. Dart pub выдаст предупреждение.

bug 2: json_annotation нужна в RUNTIME — аннотации типа @JsonSerializable и 
       сгенерированный код ссылаются на её типы. 
       Только json_serializable (генератор) идёт в dev_dependencies.
       Правило: если код ссылается на типы из пакета в production — dependencies.
                если только для сборки/генерации — dev_dependencies.

bug 3: dependency_overrides: http: 1.2.3 не позволит получать патч-обновления.
       Если выйдет 1.2.4 с фиксом безопасности — override заблокирует его.
       Используй override только для разрешения реальных конфликтов.

bug 4: Секция flutter: uses-material-design в обычном Dart пакете
       не имеет смысла и вызовет предупреждение dart pub publish.
```

Ключевое правило разделения: `dependencies` — всё что нужно в production runtime; `dev_dependencies` — инструменты для разработки (генераторы, тесты, линтеры). Неправильное разбиение увеличивает размер финальной программы.
</details>
