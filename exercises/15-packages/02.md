# Задание 02 (15.1): Структура пакета и документация — 🐛 Найти баг

**Тема:** 15. Публикация и управление пакетами | **Подтема:** 15.1 Структура пакета и документация

---

В структуре пакета и `pubspec.yaml` допущены ошибки. Найди их все.

```yaml
# pubspec.yaml
name: MyUtils        # bug 1 — имя пакета должно быть snake_case
description: utils   # bug 2 — слишком короткое описание (<60 символов)
version: 1           # bug 3 — неверный формат SemVer
homepage: not-a-url  # bug 4 — невалидный URL

environment:
  sdk: '3.0.0'       # bug 5 — нет диапазона, только точная версия

dependencies:
  http: 1.0.0        # bug 6 — нет каретки/caret: блокирует патч-обновления

dev_dependencies:
  test: ^1.0.0
```

```dart
// lib/my_utils.dart
import 'src/utils.dart';    // bug 7 — нужен export, а не import

// lib/src/utils.dart
class _Helper {             // bug 8 — приватный класс не может быть в публичном API
  static String format(String s) => s.trim();
}

// Публичная функция использует приватный класс — это нормально
String formatString(String s) => _Helper.format(s);
```

<details>
<summary>Решение</summary>

```yaml
# pubspec.yaml
name: my_utils                     # fix 1: snake_case
description: >-                    # fix 2: описание 60-180 символов
  A collection of utility functions for working with strings and collections.
  Includes formatting, validation, and transformation helpers.
version: 0.1.0                     # fix 3: SemVer major.minor.patch
homepage: https://github.com/user/my_utils  # fix 4: валидный URL

environment:
  sdk: '>=3.0.0 <4.0.0'           # fix 5: диапазон версий

dependencies:
  http: ^1.0.0                     # fix 6: caret ^ позволяет патч и минорные обновления

dev_dependencies:
  test: ^1.0.0
```

```dart
// lib/my_utils.dart
export 'src/utils.dart';    // fix 7: export делает символы из src/ частью публичного API

// lib/src/utils.dart — без изменений!
// _Helper — приватный вспомогательный класс, это НОРМАЛЬНО и ПРАВИЛЬНО
// он не экспортируется и пользователи пакета его не видят
class _Helper {
  static String format(String s) => s.trim();
}

// Только публичная функция попадает в API
String formatString(String s) => _Helper.format(s);
```

**Восемь проблем:** 1) Имена пакетов на pub.dev — только строчные `snake_case`. 2) Описание менее 60 символов — pub.dev отклонит публикацию. 3) `version: 1` — не SemVer; должно быть `major.minor.patch`. 4) `homepage` — не URL, pub.dev проверяет формат. 5) Точная версия SDK запрещает пользователям с более новыми SDK использовать пакет. 6) `http: 1.0.0` без `^` — блокирует патч-обновления безопасности зависимости. 7) `import` добавляет символы в область видимости файла, но не экспортирует их пользователям — для публичного API нужен `export`. 8) Приватный `_Helper` — нет бага; это правильный паттерн скрытия реализации.
</details>
