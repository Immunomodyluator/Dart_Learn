# Задание 08 (11.4): Isolate — 🐛 Найти баг

**Тема:** 11. Асинхронность | **Подтема:** 11.4 Isolate

---

Код пытается передать нераспаковываемый объект между изолятами. Найди и исправь.

```dart
import 'dart:isolate';

class Config {
  final String host;
  final int port;
  // Нет константного конструктора

  Config(this.host, this.port);
}

Future<String> connectInIsolate(Config cfg) async {
  // Передаём Config напрямую в Isolate.run
  return await Isolate.run(() {
    return 'Connected to ${cfg.host}:${cfg.port}';
  });
}

void main() async {
  final cfg = Config('localhost', 8080);
  print(await connectInIsolate(cfg));
}
```

<summary>Решение</summary>

```dart
import 'dart:isolate';

// Вариант 1: передать простые данные (не объект класса)
Future<String> connectInIsolate(String host, int port) async {
  return await Isolate.run(() {
    // Лямбда захватывает host и port — они примитивы, безопасно копируются
    return 'Connected to $host:$port';
  });
}

// Вариант 2: сделать Config передаваемым (SendPort-совместимым)
// Dart умеет передавать: примитивы, List, Map, Record, объекты с @pragma('vm:isolate-unsendable' false)
// Наиболее портабельно — передавать Map или Record
Future<String> connectViaRecord((String host, int port) cfg) async {
  return await Isolate.run(() {
    return 'Connected to ${cfg.$1}:${cfg.$2}';
  });
}

void main() async {
  print(await connectInIsolate('localhost', 8080));
  // Connected to localhost:8080

  print(await connectViaRecord(('db.example.com', 5432)));
  // Connected to db.example.com:5432
}
```

**Баг:** Между изолятами можно передавать только **сериализуемые** типы (примитивы, `List`, `Map`, `Record`, некоторые встроенные типы). Произвольные классы без специальной аннотации не будут скопированы — возникнет ошибка сериализации. Решение: передавать примитивы или Record вместо пользовательского объекта.
