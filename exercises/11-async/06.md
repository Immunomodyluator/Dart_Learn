# Задание 06 (11.3): StreamController — 🐛 Найти баг

**Тема:** 11. Асинхронность | **Подтема:** 11.3 StreamController

---

Код добавляет события в закрытый контроллер и не освобождает ресурсы. Найди и исправь все ошибки.

```dart
import 'dart:async';

class Counter {
  final _ctrl = StreamController<int>();

  Stream<int> get stream => _ctrl.stream;

  void start() async {
    for (var i = 0; i < 5; i++) {
      await Future.delayed(Duration(milliseconds: 100));
      _ctrl.add(i);    // что если контроллер уже закрыт?
    }
    _ctrl.close();
    _ctrl.close();     // двойное закрытие
  }
}

void main() async {
  final c = Counter();

  // Нет подписки — добавляем в никуда (для single-subscription это ошибка)
  c.start();

  await Future.delayed(Duration(milliseconds: 300));
  // Подписываемся поздно (пропустим события 0, 1, 2)
  c.stream.listen(print);

  await Future.delayed(Duration(milliseconds: 500));
}
```

<summary>Решение</summary>

```dart
import 'dart:async';

class Counter {
  final _ctrl = StreamController<int>.broadcast(); // broadcast — множество подписчиков

  Stream<int> get stream => _ctrl.stream;

  Future<void> start() async {
    for (var i = 0; i < 5; i++) {
      await Future.delayed(const Duration(milliseconds: 100));
      if (!_ctrl.isClosed) _ctrl.add(i); // проверка перед add
    }
    if (!_ctrl.isClosed) _ctrl.close();  // единственное закрытие
  }
}

void main() async {
  final c = Counter();

  // Подписаться ДО start чтобы не пропустить события
  c.stream.listen(print); // 0 1 2 3 4

  await c.start();
  await Future.delayed(const Duration(milliseconds: 100));
}
```

**Три бага:** 1) Без проверки `_ctrl.isClosed` повторный `add` после `close()` бросает `StateError`. 2) Двойной вызов `_ctrl.close()` — ошибка; добавлена защита `!isClosed`. 3) Подписка после начала `start()` пропускает уже отправленные события — подписаться нужно **до** запуска или использовать `broadcast` + хранить историю.
