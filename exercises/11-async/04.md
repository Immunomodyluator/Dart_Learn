# Задание 04 (11.2): Stream — 🐛 Найти баг

**Тема:** 11. Асинхронность | **Подтема:** 11.2 Stream

---

Код падает или ведёт себя неожиданно при повторной подписке на `SingleSubscription` stream. Найди и исправь.

```dart
import 'dart:async';

Stream<int> singleSource() async* {
  yield 1;
  yield 2;
  yield 3;
}

Future<void> main() async {
  final stream = singleSource();

  // Первая подписка
  await for (final v in stream) {
    print('First: $v');
  }

  // Вторая подписка на тот же stream
  await for (final v in stream) {   // StateError: Stream has already been listened to
    print('Second: $v');
  }
}
```

<details>
<summary>Решение</summary>

```dart
import 'dart:async';

Stream<int> singleSource() async* {
  yield 1;
  yield 2;
  yield 3;
}

Future<void> main() async {
  // Исправление 1: создавать новый stream для каждой подписки
  await for (final v in singleSource()) {
    print('First: $v');
  }

  await for (final v in singleSource()) {
    print('Second: $v');
  }

  // Исправление 2: broadcast stream (если нужно несколько подписчиков)
  final broadcast = singleSource().asBroadcastStream();
  broadcast.listen((v) => print('A: $v'));
  broadcast.listen((v) => print('B: $v'));

  await Future.delayed(const Duration(milliseconds: 50));
}
```

`async*`-функция создаёт **single-subscription** stream — его можно слушать только **один раз**. Повторный `await for` или `.listen()` бросает `StateError`. Решения: 1) вызывать функцию снова (создаёт новый stream). 2) `.asBroadcastStream()` — broadcast stream допускает несколько подписчиков, но события, пришедшие до подписки, пропускаются.
