Настрой `analysis_options.yaml` для строгой проверки кода и CI-шаг с принудительным завершением при нарушениях.

**Требования:**

1. Файл `analysis_options.yaml` — подключи `package:lints/recommended.yaml`, добавь строгие настройки анализатора и включи не менее 8 дополнительных lint-правил (prefer_const_constructors, prefer_final_locals, avoid_print, unawaited_futures, cancel_subscriptions, close_sinks, directives_ordering, prefer_single_quotes).
2. Файл `analysis_options_strict.yaml` — ещё строже: `strict-raw-types: true`, `strict-inference: true` и дополнительные правила (avoid_dynamic_calls, always_declare_return_types).
3. Шаг в `ci.yml` — запустить `dart analyze` с флагом, превращающим предупреждения (info) в ошибки; вывести статистику `dart fix --dry-run`.
4. Dart-файл `tool/check_todos.dart` — скрипт, ищущий `TODO:` в `.dart`-файлах папки `lib/` и завершающийся с `exit(1)`, если найдено хотя бы одно.

```dart
// Шаблон tool/check_todos.dart
import 'dart:io';

void main(List<String> args) {
  final dir = Directory(/* ??? */);
  // Обойди все .dart-файлы рекурсивно

  // Выведи: путь:номер_строки: TODO-строка
```

<summary>Решение</summary>

```yaml
# analysis_options.yaml
include: package:lints/recommended.yaml

analyzer:
  language:
    strict-casts: true
    strict-raw-types: false # только в strict-варианте
    strict-inference: false
  errors:
    # Трактуем предупреждения как ошибки для ключевых правил
    missing_return: error
    dead_code: warning
    todo: info # TODO не блокируют, но видны
  exclude:
    - build/**
    - "**/*.g.dart"
    - "**/*.freezed.dart"
    - "**/*.gen.dart"

linter:
  rules:
    # Качество кода
    - prefer_const_constructors
    - prefer_const_declarations
    - prefer_final_locals
    - prefer_final_fields
    # Безопасность async
    - unawaited_futures
    - cancel_subscriptions
    - close_sinks
    # Стиль
    - directives_ordering
    - prefer_single_quotes
    - avoid_print # использовать dart:developer log
    - avoid_empty_else
    - unnecessary_this
    - unnecessary_new
```

```yaml
# analysis_options_strict.yaml — наследуем основной + ужесточаем
include: analysis_options.yaml

analyzer:
  language:
    strict-casts: true
    strict-raw-types: true # запрещает List без type parameter
    strict-inference: true # запрещает неявный dynamic

linter:
  rules:
    - avoid_dynamic_calls # нельзя вызывать методы на dynamic
    - always_declare_return_types
    - always_specify_types
    - prefer_typing_uninitialized_variables
```

```yaml
# Добавить в .github/workflows/ci.yml
- name: Analyze (fatal infos)
  run: dart analyze --fatal-infos
  # --fatal-infos: info-уровень тоже завершает с ненулевым кодом

- name: Check for auto-fixable issues
  run: |
    dart fix --dry-run 2>&1 | tee fix_report.txt
    # Показать, сколько автоматических исправлений доступно
    FIXES=$(grep -c "fix" fix_report.txt || true)
    echo "Potential auto-fixes: $FIXES"

- name: Check TODOs
  run: dart run tool/check_todos.dart
```

```dart
// tool/check_todos.dart
import 'dart:io';

void main(List<String> args) {
  final targetDir = args.isNotEmpty ? args[0] : 'lib';
  final dir = Directory(targetDir);

  if (!dir.existsSync()) {
    stderr.writeln('Directory "$targetDir" not found');
    exit(2);
  }

  final found = <String>[];

  final dartFiles = dir
      .listSync(recursive: true)
      .whereType<File>()
      .where((f) => f.path.endsWith('.dart'));

  for (final file in dartFiles) {
    final lines = file.readAsLinesSync();
    for (var i = 0; i < lines.length; i++) {
      if (lines[i].contains('TODO:')) {
        found.add('${file.path}:${i + 1}: ${lines[i].trim()}');
      }
    }
  }

  if (found.isEmpty) {
    print('No TODOs found. ✓');
    exit(0);
  }

  stderr.writeln('Found ${found.length} TODO(s):');
  for (final line in found) {
    stderr.writeln('  $line');
  }
  exit(1);  // ненулевой код — CI-шаг завершится с ошибкой
}
```

`dart analyze --fatal-infos` делает info-предупреждения ошибками — весь анализ становится строже без изменения `analysis_options.yaml`. `dart fix --dry-run` показывает, сколько проблем можно исправить автоматически. Скрипт `check_todos.dart` позволяет запрещать коммит кода с незакрытыми TODO через CI, что поддерживает технический долг под контролем.
