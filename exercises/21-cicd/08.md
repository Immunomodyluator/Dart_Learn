В воркфлоу публикации пакета есть критические ошибки. Найди их.

```yaml
# .github/workflows/publish.yml (намеренно испорченная версия)
name: Publish

on:
  push:
    branches: [main] # bug 1: Публикуем при каждом push в main, а не только по тегу

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      # bug 2: нет id-token: write — OIDC-авторизация для pub.dev не сработает

    steps:
      - uses: actions/checkout@v4

      - uses: dart-lang/setup-dart@v1
        with:
          sdk: stable

      # bug 3: нет dart pub get — зависимости не установлены
      # bug 4: нет dart test — публикуем непроверенный код

      - name: Verify version
        run: |
          PUBSPEC_VERSION=$(grep 'version:' pubspec.yaml | awk '{print $2}')
          # bug 5: grep без ^ — может схватить "# version: old" из комментариев
          echo $PUBSPEC_VERSION  # bug 6: переменная без кавычек и не сохраняется
          TAG_VERSION=${GITHUB_REF_NAME}  # bug 7: не убираем 'v' из тега v1.2.3
          if [ $PUBSPEC_VERSION != $TAG_VERSION ]; then  # bug 8: сравнение без кавычек
            exit 1
          fi

      # bug 9: нет dry-run перед реальной публикацией
      - name: Publish
        run: dart pub publish # bug 10: нет --force, зависнет в CI ожидая ввода
```

```dart
// tool/bump_version.dart (намеренно испорченная версия)
import 'dart:io';

void main(List<String> args) {
  final bumpType = args[0];  // bug 11: нет проверки args.isEmpty — IndexError при пустых аргументах

  final content = File('pubspec.yaml').readAsStringSync();

  // bug 12: нежадный паттерн не зафиксирован на строку — может поймать version: в зависимостях
  final match = RegExp(r'version: (\d+\.\d+\.\d+)').firstMatch(content);

  final current = match!.group(1)!;  // bug 13: нет null-проверки — NPE если версии нет
  final parts = current.split('.').map(int.parse).toList();

  if (bumpType == 'patch') parts[2]++;
  if (bumpType == 'minor') parts[1]++;  // bug 14: minor должен обнулять patch
  if (bumpType == 'major') parts[0]++;  // bug 15: major должен обнулять minor и patch

  final next = parts.join('.');

  // bug 16: replaceAll вместо replaceFirst — может заменить версию в комментарии или зависимости
  final updated = content.replaceAll('version: $current', 'version: $next');

  File('pubspec.yaml').writeAsStringSync(updated);
  print('Done');  // bug 17: невнятное сообщение, не показывает что изменилось
}
```

```yaml
# .github/workflows/publish.yml — исправленная версия
name: Publish

# fix 1: только при создании тега формата vX.Y.Z
on:
  push:
    tags:
      - "v[0-9]+.[0-9]+.[0-9]+"

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      # fix 2: id-token: write — обязателен для OIDC с pub.dev
      id-token: write

    steps:
      - uses: actions/checkout@v4

      - uses: dart-lang/setup-dart@v1
        with:
          sdk: stable

      # fix 3: устанавливаем зависимости
      - name: Install dependencies
        run: dart pub get

      # fix 4: запускаем тесты перед публикацией
      - name: Test
        run: dart test

      - name: Analyze
        run: dart analyze --fatal-infos

      - name: Verify version
        run: |
          # fix 5: ^ якорь гарантирует начало строки, не комментарий
          PUBSPEC_VERSION=$(grep '^version:' pubspec.yaml | awk '{print $2}')
          # fix 6: кавычки вокруг переменной, выводим для диагностики
          echo "pubspec version: $PUBSPEC_VERSION"
          # fix 7: убираем 'v' из тега
          TAG_VERSION="${GITHUB_REF_NAME#v}"
          echo "tag version: $TAG_VERSION"
          # fix 8: кавычки вокруг переменных в условии
          if [ "$PUBSPEC_VERSION" != "$TAG_VERSION" ]; then
            echo "ERROR: version mismatch: pubspec=$PUBSPEC_VERSION tag=$TAG_VERSION"
            exit 1
          fi

      # fix 9: dry-run для ранней диагностики проблем
      - name: Publish dry-run
        run: dart pub publish --dry-run

      # fix 10: --force обязателен в неинтерактивном CI
      - name: Publish
        run: dart pub publish --force
```

```dart
// tool/bump_version.dart — исправленная версия
import 'dart:io';

void main(List<String> args) {
  // fix 11: проверяем аргументы заранее
  if (args.isEmpty || !{'major', 'minor', 'patch'}.contains(args[0])) {
    stderr.writeln('Usage: dart tool/bump_version.dart <major|minor|patch>');
    exit(1);
  }

  final bumpType = args[0];
  final pubspec = File('pubspec.yaml');

  if (!pubspec.existsSync()) {
    stderr.writeln('pubspec.yaml not found');
    exit(1);
  }

  final content = pubspec.readAsStringSync();

  // fix 12: ^ якорь — только строка вида "version: X.Y.Z" в начале строки
  final match = RegExp(r'^version:\s*(\d+\.\d+\.\d+)', multiLine: true)
      .firstMatch(content);

  // fix 13: явная обработка отсутствия версии
  if (match == null) {
    stderr.writeln('Could not find version field in pubspec.yaml');
    exit(1);
  }

  final current = match.group(1)!;
  final parts = current.split('.').map(int.parse).toList();

  switch (bumpType) {
    case 'major':
      parts[0]++;
      // fix 15: обнуляем minor и patch
      parts[1] = 0;
      parts[2] = 0;
    case 'minor':
      parts[1]++;
      // fix 14: обнуляем patch
      parts[2] = 0;
    case 'patch':
      parts[2]++;
  }

  final next = parts.join('.');

  // fix 16: replaceFirst — меняем только первое точное вхождение в начале строки
  final updated = content.replaceFirst(
    RegExp(r'^version:\s*\d+\.\d+\.\d+', multiLine: true),
    'version: $next',
  );

  pubspec.writeAsStringSync(updated);
  // fix 17: информативное сообщение
  print('Bumped $current → $next');
}
```

**17 ошибок:** 1) `branches: [main]` — публикуем при каждом push, должны публиковать только по тегу. 2) Нет `id-token: write` — OIDC-аутентификация не работает. 3) Нет `dart pub get`. 4) Нет `dart test`. 5) `grep 'version:'` без `^` — может поймать строку вида `# version: 0.0.1` из комментария или версию зависимости. 6) `echo $PUBSPEC_VERSION` без кавычек — word splitting; не сохраняется в переменную для сравнения. 7) `${GITHUB_REF_NAME}` без стрипа `v` — сравниваем `v1.2.3` с `1.2.3`. 8) Сравнение без кавычек — ошибка при пробелах или пустой переменной. 9) Нет `--dry-run` — сразу публикуем без проверки. 10) `dart pub publish` без `--force` зависает в CI, ожидая интерактивного подтверждения. 11) `args[0]` без проверки — `RangeError` при пустых аргументах. 12) Regex без `^` якоря — может поймать версию зависимости в pubspec. 13) `match!.group(1)!` — `NPE` если `match == null`. 14) `minor` не обнуляет `patch` — `1.2.3 minor` даёт `1.3.3` вместо `1.3.0`. 15) `major` не обнуляет `minor`/`patch` — `1.2.3 major` даёт `2.2.3` вместо `2.0.0`. 16) `replaceAll` может заменить строку версии внутри комментария. 17) `print('Done')` — не видно, что именно было обновлено.
