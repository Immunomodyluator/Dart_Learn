# Задание 02 (21.1): Настройка CI (GitHub Actions) — 🐛 Найти баг

**Тема:** 21. CI/CD и развёртывание | **Подтема:** 21.1 Настройка CI (GitHub Actions)

---

В конфигурации GitHub Actions для Dart-проекта есть ошибки. Найди их.

```yaml
# .github/workflows/ci.yml  (намеренно испорченная версия)
name: CI

on: push # bug 1: запускается только при push, PR не проверяются

jobs:
  test:
    runs-on: ubuntu-20.04 # bug 2: зафиксирована устаревшая версия вместо ubuntu-latest

    steps:
      - uses: actions/checkout@v2 # bug 3: устаревшая версия action (нужна v4)

      - uses: dart-lang/setup-dart@v1
        # bug 4: нет параметра sdk — всегда берёт stable, не тестируем beta

      # bug 5: зависимости устанавливаются ПОСЛЕ анализа
      - name: Analyze
        run: dart analyze

      - name: Install dependencies
        run: dart pub get

      - name: Test
        run: dart test # bug 6: нет --coverage, покрытие не собирается

      # bug 7: тест запускается второй раз без необходимости
      - name: Run tests again
        run: dart test --reporter expanded

      # bug 8: секрет логируется в открытую
      - name: Publish check
        run: |
          echo "Token: ${{ secrets.PUB_TOKEN }}"  # bug 8: секрет в echo!
          dart pub publish --dry-run

  deploy:
    runs-on: ubuntu-latest
    # bug 9: deploy запускается параллельно с test, не ждёт успеха
    steps:
      - uses: actions/checkout@v4
      - uses: dart-lang/setup-dart@v1
        with:
          sdk: stable
      - run: dart compile exe bin/server.dart -o server
      - run: scp server deploy@prod.example.com:/app/ # условно
```

<details>
<summary>Решение</summary>

```yaml
# .github/workflows/ci.yml — исправленная версия
name: CI

# fix 1: добавляем pull_request — проверяем код до мержа
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  test:
    # fix 2: ubuntu-latest — всегда актуальный runner
    runs-on: ubuntu-latest

    # fix 4: матрица версий SDK для широкого покрытия
    strategy:
      matrix:
        sdk: [stable, beta]
      fail-fast: false

    steps:
      # fix 3: actions/checkout@v4
      - uses: actions/checkout@v4

      - uses: dart-lang/setup-dart@v1
        with:
          sdk: ${{ matrix.sdk }} # fix 4: берём версию из матрицы

      # fix 5: зависимости СНАЧАЛА, потом анализ
      - name: Install dependencies
        run: dart pub get

      - name: Format check
        run: dart format --output=none --set-exit-if-changed .

      - name: Analyze
        run: dart analyze --fatal-infos

      # fix 6: добавляем --coverage
      - name: Test
        run: dart test --coverage=coverage

      # fix 7: убираем дублированный запуск тестов
      # (expanded reporter можно добавить к основному шагу: --reporter expanded)

      - name: Convert coverage to lcov
        run: |
          dart pub global activate coverage
          dart pub global run coverage:format_coverage \
            --lcov --in=coverage --out=coverage/lcov.info \
            --packages=.dart_tool/package_config.json

      # fix 8: НЕ логируем секреты (никогда не echo ${{ secrets.X }})
      - name: Publish dry-run
        run: dart pub publish --dry-run
        # Секрет используется неявно через OIDC или настроен как env без логирования

  deploy:
    runs-on: ubuntu-latest
    # fix 9: needs — ждём успешного завершения test
    needs: test
    # Деплоим только из main, не из PR
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'

    steps:
      - uses: actions/checkout@v4
      - uses: dart-lang/setup-dart@v1
        with:
          sdk: stable
      - run: dart pub get
      - run: dart compile exe bin/server.dart -o server
      # В реальном деплое: используем Actions secrets для учётных данных
      - name: Deploy
        env:
          DEPLOY_HOST: ${{ secrets.DEPLOY_HOST }}
          DEPLOY_KEY: ${{ secrets.DEPLOY_KEY }}
        run: |
          # Используем секреты через env, не echo
          echo "$DEPLOY_KEY" > /tmp/deploy_key
          chmod 600 /tmp/deploy_key
          scp -i /tmp/deploy_key server deploy@$DEPLOY_HOST:/app/
          rm /tmp/deploy_key
```

**Девять ошибок:** 1) `on: push` — PR на long-lived ветках не проверяются; нужно добавить `pull_request`. 2) `ubuntu-20.04` — зафиксированная старая версия; `ubuntu-latest` обновляется автоматически. 3) `actions/checkout@v2` устарел; последняя стабильная — `@v4`. 4) Нет параметра `sdk:` — всегда берётся stable, beta не тестируется. 5) `dart analyze` запускается до `dart pub get` — нет зависимостей, анализ провалится. 6) `dart test` без `--coverage` — покрытие не собирается. 7) Тест дублируется вторым шагом — лишняя трата минут CI. 8) `echo "Token: ${{ secrets.PUB_TOKEN }}"` — секрет виден в открытых логах GitHub Actions. 9) `deploy` запускается параллельно с `test` — деплой может начаться при падающих тестах; нужен `needs: test`.

</details>
