# Задание 12 (6.6): assert — 🐛 Найти баг

**Тема:** 06. Поток управления | **Подтема:** 6.6 assert

---

Разработчик использует `assert` неправильно. Найди все проблемы.

```dart
double sqrt(double x) {
  assert(x >= 0);          // ok
  return x * x;            // должен быть настоящий корень
}

void processOrder(int quantity) {
  // Валидация пользовательского ввода через assert
  assert(quantity > 0, 'Quantity must be positive');
  print('Processing $quantity items');
}

void main() {
  // assert с побочным эффектом
  assert(sqrt(4) == 2.0);  // 4*4 = 16 ≠ 2.0, но assert отключён в prod

  processOrder(0);  // должно падать всегда, но в prod assert выключен
  processOrder(-5);
}
```

<details>
<summary>Решение</summary>

```dart
import 'dart:math' as math;

double safeSqrt(double x) {
  assert(x >= 0, 'x must be non-negative'); // assert — ok для инварианта
  return math.sqrt(x);                       // исправлено: настоящий корень
}

void processOrder(int quantity) {
  // Пользовательский ввод — throw, не assert
  if (quantity <= 0) {
    throw ArgumentError.value(quantity, 'quantity', 'Must be positive');
  }
  print('Processing $quantity items');
}

void main() {
  print(safeSqrt(4));  // 2.0

  try {
    processOrder(0);
  } on ArgumentError catch (e) {
    print(e.message); // Must be positive
  }
}
```

**Три проблемы:** 1) `sqrt` возводил в квадрат вместо корня — используй `dart:math`. 2) `assert` с побочным эффектом в `main` — при компиляции assert игнорируется, и ошибка не обнаружится. 3) Валидация бизнес-данных (ввод пользователя) не должна зависеть от режима сборки — используй `throw`, а не `assert`.

</details>
