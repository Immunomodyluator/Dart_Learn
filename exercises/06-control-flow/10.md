# Задание 10 (6.5): Исключения в потоке управления — 🐛 Найти баг

**Тема:** 06. Поток управления | **Подтема:** 6.5 Исключения в потоке управления

---

Блок `finally` мешает корректному возврату значения. Найди и объясни баг.

```dart
String readConfig() {
  try {
    // имитация успешного чтения
    return 'config data';
  } catch (e) {
    return 'default';
  } finally {
    return 'cleanup done';  // всегда перезаписывает return выше
  }
}

void main() {
  print(readConfig()); // ожидается 'config data', получаем 'cleanup done'
}
```

<details>
<summary>Решение</summary>

```dart
String readConfig() {
  try {
    return 'config data';
  } catch (e) {
    return 'default';
  } finally {
    // finally не должен содержать return — только побочные эффекты
    print('cleanup done');
  }
}

void main() {
  print(readConfig()); // config data
}
```

**Баг:** `return` в блоке `finally` **заменяет** `return` из `try` или `catch`. Это валидный Dart, но почти всегда ошибка: функция всегда будет возвращать значение из `finally`, игнорируя результат основного блока. `finally` предназначен для побочных эффектов (закрытие ресурсов, логирование) — не для возврата значений.

</details>
