# Задание 08 (6.4): Паттерн-матчинг switch — 🐛 Найти баг

**Тема:** 06. Поток управления | **Подтема:** 6.4 Паттерн-матчинг switch (Dart 3)

---

Компилятор жалуется на `non-exhaustive switch` и неверный синтаксис. Найди и исправь все ошибки.

```dart
sealed class Shape {}
class Circle extends Shape { final double radius; Circle(this.radius); }
class Square extends Shape { final double side;   Square(this.side);   }
class Triangle extends Shape {}

double area(Shape s) => switch (s) {
  Circle c  => 3.14159 * c.radius * c.radius,
  Square sq => sq.side * sq.side,
  // Triangle не обработан
};

void main() {
  print(area(Circle(5)));   // ~78.5
  print(area(Square(4)));   // 16.0
  print(area(Triangle())); // ошибка
}
```

<details>
<summary>Решение</summary>

```dart
sealed class Shape {}
class Circle   extends Shape { final double radius; Circle(this.radius); }
class Square   extends Shape { final double side;   Square(this.side);   }
class Triangle extends Shape {}

double area(Shape s) => switch (s) {
  Circle c   => 3.14159 * c.radius * c.radius,
  Square sq  => sq.side * sq.side,
  Triangle _ => 0.0,   // добавлен обработчик для Triangle
};

void main() {
  print(area(Circle(5)));   // 78.53975
  print(area(Square(4)));   // 16.0
  print(area(Triangle())); // 0.0
}
```

**Баг:** `sealed class` требует **исчерпывающего** `switch` — все подтипы должны быть явно обработаны (иначе ошибка компиляции). Добавлен case `Triangle _ => 0.0`. Компилятор знает все прямые подтипы `sealed`-класса и проверяет полноту статически.

</details>
