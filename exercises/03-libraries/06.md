# Задание 06 (3.3): part, part of и директива library — 🐛 Найти баг

**Тема:** 03. Библиотеки и импорты | **Подтема:** 3.3 part, part of и директива library

---

Два файла дают ошибку компиляции. Найди все проблемы.

```dart
// lib/logger.dart
part 'formatter.dart';

class Logger {
  void log(String msg) => print(_format(msg));
}
```

```dart
// lib/formatter.dart
import 'logger.dart';

String _format(String msg) => '[LOG] $msg';
```

<details>
<summary>Решение</summary>

```dart
// lib/logger.dart
part 'formatter.dart';

class Logger {
  void log(String msg) => print(_format(msg));
}
```

```dart
// lib/formatter.dart
part of 'logger.dart'; // ← исправлено: убран import, добавлен part of

String _format(String msg) => '[LOG] $msg';
```

Две проблемы:

1. В `formatter.dart` нужна директива `part of 'logger.dart'` вместо `import` — файл является частью библиотеки, а не отдельным модулем.
2. Part-файлы не могут содержать `import` для родительской библиотеки — они уже её часть и автоматически видят все её объявления.
</details>
