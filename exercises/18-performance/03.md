**Задание 1 — Object Pool**

Реализуй `ObjectPool<T>` — пул переиспользуемых объектов:

```dart
class ObjectPool<T> {
  // create — фабрика для создания объекта
  // reset  — сброс состояния перед повторным использованием
  ObjectPool({required T Function() create, required void Function(T) reset});

  T acquire();          // взять объект из пула (или создать новый)
  void release(T obj);  // вернуть объект (сбросить + положить в пул)
  int get poolSize;
}
```

**Задание 2 — StringBuilder вместо конкатенации**

Напиши функцию `buildHtmlTable(List<List<String>> rows)`, которая строит HTML-таблицу:

- `<table>`, `<tr>`, `<td>` теги
- Используй `StringBuffer`, а не `+=` конкатенацию

**Задание 3 — const и идентичность**

Объясни (в комментарии) и продемонстрируй разницу между примерами:

```dart
final a1 = [1, 2, 3];       // новый список каждый раз
const a2 = [1, 2, 3];       // ???
final a3 = const [1, 2, 3]; // ???
```

Напиши тест `assertConstIdentity()`, доказывающий, что два `const` литерала — **один объект**.

<summary>Решение</summary>

```dart
import 'dart:collection';

/// Задание 1: ObjectPool
class ObjectPool<T> {
  final T Function() _create;
  final void Function(T) _reset;
  final Queue<T> _pool = Queue();

  ObjectPool({required T Function() create, required void Function(T) reset})
      : _create = create,
        _reset = reset;

  T acquire() => _pool.isEmpty ? _create() : _pool.removeFirst();

  void release(T obj) {
    _reset(obj);
    _pool.addLast(obj);
  }

  int get poolSize => _pool.length;
}

// Пример с List<int> — избегаем постоянного создания буферов
void demonstratePool() {
  final pool = ObjectPool<List<int>>(
    create: () => List.filled(1024, 0),
    reset: (buf) {
      for (var i = 0; i < buf.length; i++) {
        buf[i] = 0;
      }
    },
  );

  final buf1 = pool.acquire();
  buf1[0] = 42;
  pool.release(buf1); // сбросить и вернуть

  final buf2 = pool.acquire(); // переиспользованный buf1 (buf2[0] == 0)
  assert(buf2[0] == 0, 'Буфер должен быть сброшен');
  assert(identical(buf1, buf2), 'Тот же объект из пула');

  print('Pool demo: buf1 === buf2: ${identical(buf1, buf2)}');
}

/// Задание 2: StringBuffer вместо конкатенации
String buildHtmlTable(List<List<String>> rows) {
  if (rows.isEmpty) return '<table></table>';

  final sb = StringBuffer('<table>\n'); // не накапливаем через +=
  for (final row in rows) {
    sb.write('  <tr>');
    for (final cell in row) {
      sb.write('<td>');
      sb.write(cell);
      sb.write('</td>');
    }
    sb.writeln('</tr>');
  }
  sb.write('</table>');
  return sb.toString(); // одна аллокация в конце
}

/// Задание 3: const и идентичность
void assertConstIdentity() {
  final a1 = [1, 2, 3]; // новый List каждый вызов — разные объекты
  final a2 = [1, 2, 3];
  assert(!identical(a1, a2), 'final-литералы — разные объекты');

  // const: канонизированные объекты — компилятор гарантирует один экземпляр
  const c1 = [1, 2, 3];
  const c2 = [1, 2, 3];
  assert(identical(c1, c2), 'const-литералы — один объект! (канонизация)');

  // final + const: указывает на тот же const-экземпляр
  final a3 = const [1, 2, 3];
  assert(identical(c1, a3), 'final = const [1,2,3] тот же канон');

  // Практическое следствие: используй const для неизменяемых коллекций-значений
  // — это не только экономит память, но и может ускорить == сравнение до identity-check
  print('const identity: c1 === c2: ${identical(c1, c2)}'); // true
  print('final identity: a1 === a2: ${identical(a1, a2)}'); // false
}

void main() {
  demonstratePool();

  final table = buildHtmlTable([
    ['Name', 'Age'],
    ['Alice', '30'],
    ['Bob', '25'],
  ]);
  print(table);

  assertConstIdentity();
}
```

**Ключевые паттерны оптимизации:** 1) `ObjectPool` устраняет цикл создание→GC для часто создаваемых объектов. 2) `StringBuffer` делает одну аллокацию в конце (`toString()`), тогда как `+=` создаёт промежуточный `String` на каждом шаге. 3) `const`-литералы канонизируются компилятором — два `const [1,2,3]` в разных местах кода указывают на один объект в памяти.
