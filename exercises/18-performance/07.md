Реализуй шаблоны, помогающие снизить нагрузку на GC в Dart.

**Задание 1 — Weak references**

Реализуй кеш `WeakCache<K, V>` на основе `Expando` и `WeakReference`:

- `set(K key, V value)` — сохранить значение
- `V? get(K key)` — вернуть значение, если ещё не собрано GC

**Задание 2 — Finalizer**

Напиши класс `NativeBuffer` (симуляция нативного ресурса), использующий `Finalizer` для автоматической очистки при сборке GC:

```dart
class NativeBuffer {
  final int sizeBytes;
  NativeBuffer(this.sizeBytes);
  void dispose();      // явное освобождение
  bool get isDisposed;
}
```

**Задание 3 — Мониторинг памяти**

Напиши функцию `measureMemory(String label, void Function() action)` через `ProcessInfo.currentRss` (или `dart:developer`'s `Service.getVM()`), выводящую отпечаток памяти до и после `action`.

<summary>Решение</summary>

```dart
import 'dart:developer' as dev;
import 'dart:io';

/// Задание 1: WeakCache
/// Expando<V> привязывает значения к объектам слабо (по ключу-объекту),
/// но объект-ключ должен быть Object (не примитив).
/// В Dart нет встроенного WeakMap по произвольным ключам — эмулируем через Expando.
class WeakCache<K extends Object, V extends Object> {
  // Expando не позволяет хранить ключи — пишем WeakReference на value
  // и храним список WeakReference<(K, V)> пар
  // Простая реализация: Dart 3.1+ — WeakReference<V>
  final _entries = <K, WeakReference<V>>{};

  void set(K key, V value) {
    _entries[key] = WeakReference(value);
  }

  V? get(K key) {
    final ref = _entries[key];
    if (ref == null) return null;
    final value = ref.target; // null если GC собрал объект
    if (value == null) {
      _entries.remove(key); // чистим мёртвые записи
    }
    return value;
  }

  void prune() {
    _entries.removeWhere((k, ref) => ref.target == null);
  }

  int get size => _entries.length;
}

/// Задание 2: NativeBuffer с Finalizer
// Глобальный Finalizer — срабатывает когда объект NativeBuffer собирается GC
final _bufferFinalizer = Finalizer<int>((sizeBytes) {
  // В реальности: вызов нативного free()
  print('[GC Finalizer] Освобождён нативный буфер $sizeBytes байт');
  _totalNativeBytes -= sizeBytes;
});

int _totalNativeBytes = 0;

class NativeBuffer {
  final int sizeBytes;
  bool _disposed = false;

  NativeBuffer(this.sizeBytes) {
    _totalNativeBytes += sizeBytes;
    print('Выделен нативный буфер $sizeBytes байт (total: $_totalNativeBytes)');
    // Регистрируем finalizer — если dispose() не вызвали, GC подчистит
    _bufferFinalizer.attach(this, sizeBytes, detach: this);
  }

  void dispose() {
    if (_disposed) return;
    _disposed = true;
    _bufferFinalizer.detach(this); // отменяем финализатор — не нужен
    _totalNativeBytes -= sizeBytes;
    print('NativeBuffer.dispose() $sizeBytes байт (total: $_totalNativeBytes)');
  }

  bool get isDisposed => _disposed;

  static int get totalAllocated => _totalNativeBytes;
}

/// Задание 3: Мониторинг памяти через dart:io ProcessInfo
void measureMemory(String label, void Function() action) {
  final before = ProcessInfo.currentRss; // RSS в байтах
  action();
  final after = ProcessInfo.currentRss;
  final delta = after - before;
  final sign = delta >= 0 ? '+' : '';
  print('[$label] RSS: ${_fmt(before)} → ${_fmt(after)} ($sign${_fmt(delta)})');
}

String _fmt(int bytes) {
  if (bytes.abs() < 1024) return '${bytes}B';
  if (bytes.abs() < 1024 * 1024) return '${(bytes / 1024).toStringAsFixed(1)}KB';
  return '${(bytes / 1024 / 1024).toStringAsFixed(2)}MB';
}

void main() {
  // WeakCache
  final cache = WeakCache<String, List<int>>();
  var bigList = List.generate(10000, (i) => i);
  cache.set('data', bigList);
  print('Cache get: ${cache.get('data')?.length} items');
  // bigList = null; // позволяем GC собрать — в Dart нет гарантий когда

  // NativeBuffer
  measureMemory('allocate buffers', () {
    final b1 = NativeBuffer(1024 * 1024); // 1MB
    final b2 = NativeBuffer(512 * 1024);  // 512KB
    b1.dispose(); // явное освобождение
    print('b2 disposed: ${b2.isDisposed}'); // false — не освобождён явно
    // b2 будет освобождён финализатором при GC
  });

  print('Total native: ${NativeBuffer.totalAllocated} bytes after scope');
  // b2 ещё не собрана GC — итог может не быть 0

  // Демонстрация: большие аллокации
  measureMemory('create 1M list', () {
    // ignore: unused_local_variable
    final _ = List.generate(1000000, (i) => i * 2.0);
  });
}
```

**Ключевые концепции:** 1) `WeakReference<T>` — ссылка, не препятствующая GC; `.target` возвращает `null` если объект собран. 2) `Finalizer` — callback при GC-сборке объекта; `attach(obj, token)` регистрирует, `detach(obj)` снимает (например, при явном `dispose()`). 3) `ProcessInfo.currentRss` из `dart:io` — RSS (Resident Set Size) процесса в байтах; удобен для грубой оценки потребления до/после операции.
