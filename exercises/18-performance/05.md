**Задание 1 — Параллельные vs последовательные запросы**

Напиши два варианта функции загрузки данных пользователей по списку ID:

- `fetchSequential(List<int> ids)` — последовательно
- `fetchParallel(List<int> ids)` — параллельно через `Future.wait`
- `fetchBatched(List<int> ids, {int batchSize = 5})` — параллельно пачками

Используй `Future.delayed` для симуляции сетевого запроса (100 мс).

**Задание 2 — Дросселирование (throttle)**

Реализуй `ThrottledExecutor` — выполняет не более N конкурентных Future одновременно:

```dart
class ThrottledExecutor {
  ThrottledExecutor(int maxConcurrent);
  Future<T> run<T>(Future<T> Function() task);
}
```

**Задание 3 — Изоляция тяжёлых вычислений**

```dart
import 'dart:async';
import 'dart:isolate';

// Симуляция сетевого запроса
Future<String> fetchUser(int id) async {
  await Future.delayed(const Duration(milliseconds: 100));
  return 'User#$id';
}

/// Задание 1: Три варианта загрузки

// Последовательно: суммарное время = N * 100ms
Future<List<String>> fetchSequential(List<int> ids) async {
  final results = <String>[];
  for (final id in ids) {
    results.add(await fetchUser(id));
  }
  return results;
}

// Параллельно: суммарное время ≈ 100ms (все одновременно)
Future<List<String>> fetchParallel(List<int> ids) {
  return Future.wait(ids.map(fetchUser));
}

// Пачками: суммарное время = ceil(N/batchSize) * 100ms
Future<List<String>> fetchBatched(
  List<int> ids, {
  int batchSize = 5,
}) async {
  final results = <String>[];
  for (var i = 0; i < ids.length; i += batchSize) {
    final batch = ids.skip(i).take(batchSize).toList();
    final batchResults = await Future.wait(batch.map(fetchUser));
    results.addAll(batchResults);
  }
  return results;
}

/// Задание 2: ThrottledExecutor
class ThrottledExecutor {
  final int _maxConcurrent;
  int _running = 0;
  final Queue<Completer<void>> _queue = Queue();

  ThrottledExecutor(this._maxConcurrent);

  Future<T> run<T>(Future<T> Function() task) async {
    if (_running >= _maxConcurrent) {
      // Поставить в очередь ожидания
      final completer = Completer<void>();
      _queue.add(completer);
      await completer.future;
    }
    _running++;
    try {
      return await task();
    } finally {
      _running--;
      if (_queue.isNotEmpty) {
        // Разбудить следующую задачу
        _queue.removeFirst().complete();
      }
    }
  }

  int get running => _running;
  int get queued => _queue.length;
}

/// Задание 3: Тяжёлые вычисления в Isolate
Future<R> computeInIsolate<T, R>(R Function(T) fn, T arg) {
  return Isolate.run(() => fn(arg));
}

// Пример тяжёлого вычисления: n-ое число Фибоначчи (наивный рекурсивный)
int fibSlow(int n) => n <= 1 ? n : fibSlow(n - 1) + fibSlow(n - 2);

import 'dart:collection';

Future<void> main() async {
  final ids = List.generate(10, (i) => i + 1);

  // Сравнение времён
  final sw = Stopwatch()..start();
  await fetchSequential(ids);
  print('Sequential: ${sw.elapsedMilliseconds}ms'); // ~1000ms

  sw.reset(); sw.start();
  await fetchParallel(ids);
  print('Parallel:   ${sw.elapsedMilliseconds}ms'); // ~100ms

  sw.reset(); sw.start();
  await fetchBatched(ids, batchSize: 3);
  print('Batched(3): ${sw.elapsedMilliseconds}ms'); // ~400ms

  // ThrottledExecutor — не более 3 одновременных
  final executor = ThrottledExecutor(3);
  sw.reset(); sw.start();
  await Future.wait(
    ids.map((id) => executor.run(() => fetchUser(id))),
  );
  print('Throttled(3): ${sw.elapsedMilliseconds}ms'); // ~400ms, max 3 одновременно

  // Изоляция — не блокирует event loop
  sw.reset(); sw.start();
  final fib35 = await computeInIsolate(fibSlow, 35);
  print('fib(35) in isolate: $fib35 (${sw.elapsedMilliseconds}ms)');
}
```

**Ключевые паттерны:** 1) `Future.wait` запускает все Future одновременно — O(1) по времени ожидания. 2) Пакетная обработка балансирует нагрузку на сервер vs скорость. 3) `ThrottledExecutor` через `Queue<Completer>` не блокирует поток — дожидающиеся задачи освобождают event loop. 4) `Isolate.run()` — простейший способ вынести тяжёлое вычисление (CPU-bound) из main isolate, чтобы не морозить UI или сервер.
