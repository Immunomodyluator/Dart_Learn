Реализуй `Result<T>`-паттерн — альтернативу исключениям, популярную в функциональном программировании.

**Задача:**

1. Объяви `sealed class Result<T>` с двумя вариантами:

   - `T getOrElse(T fallback)` — возвращает value или fallback

   - `Result<double> parseAndDivide(String a, String b)` — комбинирует обе через `map` и `flatMap` (реализуй `flatMap` тоже)

```dart
sealed class Result<T> {
  const Result();

  bool get isOk => this is Ok<T>;
  bool get isErr => this is Err<T>;

  T getOrElse(T fallback) => switch (this) {
        Ok(:final value) => value,
        Err() => fallback,
      };

  Result<R> map<R>(R Function(T) fn) => switch (this) {
        Ok(:final value) => Ok(fn(value)),
        Err(:final error) => Err(error),
      };

  Result<R> flatMap<R>(Result<R> Function(T) fn) => switch (this) {
        Ok(:final value) => fn(value),
        Err(:final error) => Err(error),
      };

  @override
  String toString() => switch (this) {
        Ok(:final value) => 'Ok($value)',
        Err(:final error) => 'Err($error)',
      };
}

final class Ok<T> extends Result<T> {
  const Ok(this.value);
  final T value;
}

final class Err<T> extends Result<T> {
  const Err(this.error);
  final String error;
}

Result<int> tryParseInt(String s) {
  final n = int.tryParse(s);
  return n != null ? Ok(n) : Err('Cannot parse "$s" as int');
}

Result<double> safeDiv(double a, double b) {
  if (b == 0) return Err('Division by zero');
  return Ok(a / b);
}

Result<double> parseAndDivide(String a, String b) {
  return tryParseInt(a)
      .flatMap((x) => tryParseInt(b).map((y) => (x, y)))
      .flatMap((pair) => safeDiv(pair.$1.toDouble(), pair.$2.toDouble()));
}

void main() {
  print(tryParseInt('42'));          // Ok(42)
  print(tryParseInt('abc'));         // Err(Cannot parse "abc" as int)
  print(safeDiv(10, 0));            // Err(Division by zero)
  print(parseAndDivide('10', '4'));  // Ok(2.5)
  print(parseAndDivide('10', '0')); // Err(Division by zero)
  print(parseAndDivide('x', '4'));  // Err(Cannot parse "x" as int)

  final result = tryParseInt('100');
  print(result.map((n) => n * 2));  // Ok(200)
  print(result.getOrElse(-1));      // 100
}
```

`Result<T>` с `sealed class` позволяет обрабатывать ошибки через исчерпывающий `switch` без необходимости ловить исключения. Метод `flatMap` (или `andThen`) позволяет цепочкой соединять операции, каждая из которых может завершиться ошибкой, — если любая вернёт `Err`, остальные пропускаются.
