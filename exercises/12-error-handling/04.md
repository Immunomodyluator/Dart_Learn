# Задание 04 (12.2): try/catch/finally в async коде — 🐛 Найти баг

**Тема:** 12. Обработка ошибок | **Подтема:** 12.2 try/catch/finally в async коде

---

Программа должна поймать ошибку загрузки данных и напечатать сообщение об ошибке. Найди все баги.

```dart
Future<String> fetchData() async {
  await Future.delayed(const Duration(milliseconds: 10));
  throw Exception('server error');
}

Future<void> processData() async {
  final result = fetchData();   // bug 1

  try {
    print('Result: $result');
  } catch (e) {                 // bug 2 — catch здесь ничего не поймает
    print('Error: $e');
  }
}

void handleAll() {              // bug 3 — не async
  try {
    processData();              // bug 4 — нет await
    print('Done');
  } catch (e) {
    print('Caught: $e');
  }
}

void main() {
  handleAll();
}
```

<details>
<summary>Решение</summary>

```dart
Future<String> fetchData() async {
  await Future.delayed(const Duration(milliseconds: 10));
  throw Exception('server error');
}

Future<void> processData() async {
  final result = await fetchData();   // fix 1: await

  // fix 2: try/catch должен окружать await-вызов
  print('Result: $result');
}

Future<void> handleAll() async {      // fix 3: async
  try {
    await processData();              // fix 4: await
    print('Done');
  } catch (e) {
    print('Caught: $e');             // Caught: Exception: server error
  }
}

void main() async {
  await handleAll();
}
```

**Четыре бага:**

1. `fetchData()` без `await` возвращает `Future<String>`, а не `String` — исключение остаётся «внутри» Future.
2. `try/catch` вокруг `print` не охватывает `await`-вызов — ошибка уходит дальше вверх по стеку.
3. `handleAll()` — не `async`, поэтому `await` внутри запрещён, а `try/catch` не перехватит ошибки из Future.
4. `processData()` без `await` — вызов запускается «и забывается», исключение становится необработанным (unhandled).
</details>
