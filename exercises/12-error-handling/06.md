# Задание 06 (12.3): Zone и обработка необработанных ошибок — 🐛 Найти баг

**Тема:** 12. Обработка ошибок | **Подтема:** 12.3 Zone и обработка необработанных ошибок

---

Код должен перехватывать все необработанные асинхронные ошибки. Найди все баги.

```dart
import 'dart:async';

void setupErrorHandling() {
  runZonedGuarded(            // bug 1 — неверное использование: setup вне тела зоны
    () => print('Zone created'),
    (e, st) => print('Error: $e'),
  );
}

void doRiskyWork() {
  Future(() => throw Exception('oops!'));
}

Future<void> main() async {
  setupErrorHandling();       // зона уже завершилась к этому моменту
  doRiskyWork();              // эта ошибка НЕ будет поймана зоной выше

  // попытка поймать вручную — тоже не сработает
  try {
    await Future.delayed(const Duration(milliseconds: 50));
  } catch (e) {               // bug 2 — Future из doRiskyWork не awaited
    print('Caught: $e');
  }
}
```

<details>
<summary>Решение</summary>

```dart
import 'dart:async';

Future<void> main() async {
  // fix: весь код приложения должен выполняться ВНУТРИ тела runZonedGuarded
  runZonedGuarded(
    () async {
      doRiskyWork();
      await Future.delayed(const Duration(milliseconds: 50));
      print('Done');
    },
    (e, st) => print('Caught by zone: $e'),
    // Caught by zone: Exception: oops!
  );

  // Даём время асинхронным событиям отработать
  await Future.delayed(const Duration(milliseconds: 100));
}

void doRiskyWork() {
  Future(() => throw Exception('oops!'));
}
```

**Два ключевых бага:**

1. `runZonedGuarded` создаёт зону **только на время выполнения своего первого аргумента** (`body`). Если тело зоны просто печатает строку и завершается, зона исчезает. Весь код, который должен быть защищён зоной, обязан выполняться **внутри** тела (`body`).
2. `try/catch` не поможет для необработанного Future — `doRiskyWork()` запускает Future без `await`, и его ошибка не может попасть в `catch` блок вызывающего кода. Только зона может перехватить такие ошибки.
