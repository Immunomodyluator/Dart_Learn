# Задание 02 (12.1): Пользовательские исключения — 🐛 Найти баг

**Тема:** 12. Обработка ошибок | **Подтема:** 12.1 Пользовательские исключения

---

Исключения объявлены неверно и перехватываются в неправильном порядке. Найди и исправь все ошибки.

```dart
class NetworkError {     // не реализует Exception/Error
  final int statusCode;
  NetworkError(this.statusCode);
}

class TimeoutError extends NetworkError {
  TimeoutError() : super(408);
}

void fetch(String url) {
  if (url.isEmpty) throw NetworkError(400);
  if (url == 'slow') throw TimeoutError();
  print('Fetched: $url');
}

void main() {
  try {
    fetch('slow');
  } on Exception catch (e) {        // NetworkError не Exception — не поймает
    print('Exception: $e');
  } on TimeoutError catch (e) {     // более специфичный тип — должен быть выше!
    print('Timeout: $e');
  } on NetworkError catch (e) {
    print('Network: ${e.statusCode}');
  } catch (e) {
    print('Unknown: $e');           // сюда попадёт TimeoutError
  }
}
```

<summary>Решение</summary>

```dart
class NetworkError implements Exception {   // implements Exception
  final int statusCode;
  final String message;

  NetworkError(this.statusCode, [this.message = 'Network error']);

  @override
  String toString() => 'NetworkError($statusCode): $message';
}

class TimeoutError extends NetworkError {
  TimeoutError() : super(408, 'Request timed out');
}

void fetch(String url) {
  if (url.isEmpty) throw NetworkError(400, 'Empty URL');
  if (url == 'slow') throw TimeoutError();
  print('Fetched: $url');
}

void main() {
  try {
    fetch('slow');
  } on TimeoutError catch (e) {    // специфичный — первым!
    print('Timeout: $e');
  } on NetworkError catch (e) {    // базовый — после специфичных
    print('Network: ${e.statusCode}');
  } on Exception catch (e) {
    print('Exception: $e');
  }
  // Timeout: NetworkError(408): Request timed out
}
```

**Два бага:** 1) `class NetworkError` без `implements Exception` — блок `on Exception` его не поймает; нужно `implements Exception`. 2) Порядок `on`-блоков: более специфичный тип (`TimeoutError`) должен быть **до** базового (`NetworkError`) — иначе он никогда не выполнится (базовый перехватит первым).

</details>
