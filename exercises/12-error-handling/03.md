Напиши три функции для работы с ошибками в асинхронном коде:

1. `loadConfig(String path) → Future<Map<String, String>>` — читает «файл» (симулируй задержку `Future.delayed`). Если `path` пустой — выбрасывай `ArgumentError`; если `path == 'missing'` — выбрасывай `FileSystemException` из `dart:io`. В блоке `finally` печатай `'cleanup: $path'`.

2. `loadWithFallback(String path, Map<String, String> defaults) → Future<Map<String, String>>` — вызывает `loadConfig`, при любой ошибке возвращает `defaults`. Используй `.catchError()` (метод на Future, а не `try/catch`).

3. `loadAll(List<String> paths) → Future<List<Map<String, String>>>` — загружает все конфигурации параллельно. Если хотя бы одна упала — возвращает результаты успешных (через `Future.wait` с `eagerError: false` или `Future.wait` + `catchError` на каждый).

```dart
import 'dart:io';

Future<Map<String, String>> loadConfig(String path) async {
  try {
    if (path.isEmpty) throw ArgumentError('path must not be empty');
    await Future.delayed(const Duration(milliseconds: 50));
    if (path == 'missing') throw FileSystemException('File not found', path);
    return {'env': 'production', 'version': '1.0'};
  } finally {
    print('cleanup: $path');
  }
}

Future<Map<String, String>> loadWithFallback(
  String path,
  Map<String, String> defaults,
) {
  return loadConfig(path).catchError((_) => defaults);
}

Future<List<Map<String, String>>> loadAll(List<String> paths) async {
  final futures = paths.map(
    (p) => loadConfig(p).catchError((_) => <String, String>{}),
  );
  final results = await Future.wait(futures);
  return results.where((m) => m.isNotEmpty).toList();
}

void main() async {
  // 1. try/catch/finally
  try {
    final cfg = await loadConfig('app.json');
    print(cfg);
  } on ArgumentError catch (e) {
    print('Bad argument: $e');
  } on FileSystemException catch (e) {
    print('FS error: ${e.message}');
  }
  // cleanup: app.json
  // {env: production, version: 1.0}

  // 2. catchError
  final cfg = await loadWithFallback('missing', {'env': 'dev'});
  print(cfg); // cleanup: missing → {env: dev}

  // 3. параллельная загрузка с частичным успехом
  final all = await loadAll(['app.json', 'missing', 'other.json']);
  print('loaded: ${all.length}'); // loaded: 2
}
```

В `async`-функциях `try/catch/finally` работает так же, как в синхронном коде — `await` «разворачивает» Future и позволяет перехватывать ошибки стандартным способом. `.catchError()` удобен для перехвата ошибок в цепочках Future без `async/await`. При параллельной загрузке оборачивай каждый Future в `.catchError`, чтобы один сбой не отменял остальные.
