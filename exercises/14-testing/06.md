Интеграционные тесты содержат антипаттерны и ошибки. Найди их.

```dart
import 'package:test/test.dart';
import 'dart:io';

// DB-like хранилище
class FileStorage {
  final String path;
  FileStorage(this.path);

  Future<void> write(String key, String value) async {
    final file = File('$path/$key.txt');
    await file.writeAsString(value);
  }

  Future<String?> read(String key) async {
    final file = File('$path/$key.txt');
    return file.existsSync() ? file.readAsStringSync() : null;
  }
}

// Глобальное состояние — bug 1: тесты делят общее состояние
final storage = FileStorage('/tmp/integration_test');

void main() {
  // bug 2 — setUp не создаёт директорию и не очищает её между тестами
  setUp(() async {
    // пусто
  });

  test('write and read value', () async {
    await storage.write('name', 'Alice');
    expect(await storage.read('name'), 'Alice');
  });

  test('missing key returns null', () async {
    // bug 3 — предыдущий тест записал 'name' → этот тест зависит от порядка?
    // на самом деле тест проверяет несуществующий ключ — должен работать
    // но bug: нужно проверить 'missing', не 'name'
    expect(await storage.read('name'), isNull);  // провалится!
  });

  test('overwrite value', () async {
    await storage.write('config', 'v1');
    await storage.write('config', 'v2');
    // bug 4 — нет assert что значение именно v2
    expect(await storage.read('config'), isNotNull);
  });

  // bug 5 — нет tearDown для очистки временных файлов
}
```

<summary>Решение</summary>

```dart
import 'package:test/test.dart';
import 'dart:io';

class FileStorage {
  final String path;
  FileStorage(this.path);

  Future<void> write(String key, String value) async {
    final dir = Directory(path);
    if (!dir.existsSync()) await dir.create(recursive: true);
    await File('$path/$key.txt').writeAsString(value);
  }

  Future<String?> read(String key) async {
    final file = File('$path/$key.txt');
    return file.existsSync() ? await file.readAsString() : null;
  }
}

void main() {
  // fix 1: каждый тест получает изолированное хранилище во временной директории
  late FileStorage storage;
  late Directory tempDir;

  setUp(() async {
    // fix 2: создаём свежую временную директорию перед каждым тестом
    tempDir = await Directory.systemTemp.createTemp('dart_test_');
    storage = FileStorage(tempDir.path);
  });

  // fix 5: очищаем временные файлы после каждого теста
  tearDown(() async {
    await tempDir.delete(recursive: true);
  });

  test('write and read value', () async {
    await storage.write('name', 'Alice');
    expect(await storage.read('name'), equals('Alice'));
  });

  test('missing key returns null', () async {
    // fix 3: 'missing' — ключ, которого точно нет
    expect(await storage.read('missing'), isNull);
  });

  test('overwrite replaces value', () async {
    await storage.write('config', 'v1');
    await storage.write('config', 'v2');
    // fix 4: проверяем точное итоговое значение
    expect(await storage.read('config'), equals('v2'));
  });
}
```

**Пять антипаттернов:** 1) Глобальный объект `storage` разделяется между тестами — состояние накапливается. 2) `setUp` пустой — директория не создаётся и не очищается. 3) Тест `isNull` для ключа `'name'` провалится если предыдущий тест записал это значение (зависимость от порядка). 4) `isNotNull` не гарантирует правильное значение после перезаписи. 5) Нет `tearDown` — временные файлы засоряют `/tmp` и мешают повторному запуску тестов.
