# Задание 02 (14.1): Unit-тесты с пакетом test — 🐛 Найти баг

**Тема:** 14. Тестирование | **Подтема:** 14.1 Unit-тесты с пакетом test

---

В тестах есть ошибки — некоторые тесты дают ложноположительный или ложноотрицательный результат. Найди все проблемы.

```dart
import 'package:test/test.dart';

String reverse(String s) => s.split('').reversed.join();

Future<List<int>> fetchNumbers() async {
  await Future.delayed(const Duration(milliseconds: 10));
  return [1, 2, 3];
}

void main() {
  // bug 1 — тест всегда проходит, даже если reverse работает неверно
  test('reverse empty string', () {
    expect(reverse(''), '');
  });

  test('reverse hello', () {
    expect(reverse('hello'), 'olleh');
  });

  // bug 2 — async тест без async/await: Future игнорируется, тест не ждёт
  test('fetchNumbers returns list', () {
    fetchNumbers().then((nums) {
      expect(nums, [1, 2, 3]);  // если провалится — это UncaughtError в Future
    });
  });

  // bug 3 — неверный матчер, equals сравнивает по содержимому, это правильно,
  //         но здесь тест НИКОГДА не упадёт даже если список пустой
  test('fetchNumbers is not empty', () async {
    final nums = await fetchNumbers();
    expect(nums.isNotEmpty, true);  // не информативно при падении
  });

  // bug 4 — expect на исключение написан неверно
  test('throws on null input', () {
    try {
      reverse(null as dynamic);
      fail('Should have thrown');
    } catch (e) {
      expect(e is TypeError, true);  // bug: isA<TypeError>() надёжнее
    }
  });

  // bug 5 — tearDown вместо setUp для инициализации
  late List<int> data;
  tearDown(() => data = [10, 20, 30]);

  test('data sum', () {
    expect(data.fold(0, (a, b) => a + b), 60);
  });
}
```

<summary>Решение</summary>

```dart
import 'package:test/test.dart';

String reverse(String s) => s.split('').reversed.join();

Future<List<int>> fetchNumbers() async {
  await Future.delayed(const Duration(milliseconds: 10));
  return [1, 2, 3];
}

void main() {
  // 1. Тест пустой строки корректен — это не баг; но добавим более строгий check
  test('reverse empty string returns empty', () {
    expect(reverse(''), isEmpty); // более информативно
  });

  test('reverse hello', () {
    expect(reverse('hello'), equals('olleh'));
  });

  // fix 2: async тест должен быть async и await Future
  test('fetchNumbers returns list', () async {
    final nums = await fetchNumbers();
    expect(nums, equals([1, 2, 3]));
  });

  // fix 3: используй конкретный матчер — isNotEmpty (информативнее при провале)
  test('fetchNumbers is not empty', () async {
    final nums = await fetchNumbers();
    expect(nums, isNotEmpty);
    expect(nums, hasLength(3));
  });

  // fix 4: используй throwsA(isA<T>()) — надёжно и типобезопасно
  test('throws on null input', () {
    expect(() => reverse(null as dynamic), throwsA(isA<TypeError>()));
  });

  // fix 5: для инициализации переменных используй setUp, не tearDown
  late List<int> data;
  setUp(() => data = [10, 20, 30]);

  test('data sum', () {
    expect(data.fold<int>(0, (a, b) => a + b), equals(60));
  });
}
```

**Пять багов:** 1) Тест с пустой строкой не баг сам по себе, но `isEmpty` информативнее `''`. 2) Async тест без `async/await` — исключение в `.then()` не попадает в тест-раннер, тест всегда «зелёный». 3) `expect(isNotEmpty, true)` — при провале выводит `Expected: true, Actual: false` без контекста; `isNotEmpty` матчер даёт `Expected: non-empty iterable`. 4) `catch + is` вместо `throwsA(isA<T>())` — менее надёжно и многословно; если исключение не бросается — `fail` работает, но блок catch может поймать неверный тип. 5) `tearDown` выполняется **после** теста — `data` инициализируется после проверки, `data.fold` упадёт с LateInitializationError. Нужен `setUp`.
