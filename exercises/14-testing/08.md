# Задание 08 (14.4): Покрытие кода и CI-интеграция — 🐛 Найти баг

**Тема:** 14. Тестирование | **Подтема:** 14.4 Покрытие кода и CI-интеграция

---

В конфигурации CI и тестах есть проблемы. Найди все ошибки.

```yaml
# .github/workflows/ci.yml
name: CI
on: push

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dart-lang/setup-dart@v1

      # bug 1 — зависимости не устанавливаются перед тестами
      
      - run: dart test                      # bug 2 — нет анализа кода
      - run: dart test --coverage=coverage  # bug 3 — тесты запускаются дважды!
      
      - run: dart pub global activate coverage
      - run: dart pub global run coverage:format_coverage
          --lcov --in=coverage --out=lcov.info  # bug 4 — отсутствует --packages флаг
```

```dart
// test/api_test.dart
import 'package:test/test.dart';

// bug 5 — тест без тега запустится и в unit, и в integration suite
// медленный тест должен иметь тег
test('calls real external API', () async {
  // симулируем: реальный HTTP-запрос без мока
  await Future.delayed(const Duration(seconds: 3));
  expect(true, isTrue);
}, timeout: Timeout(Duration(seconds: 2)));  // bug 6 — таймаут меньше задержки
```

<details>
<summary>Решение</summary>

```yaml
# .github/workflows/ci.yml
name: CI
on: push

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dart-lang/setup-dart@v1
        with:
          sdk: stable

      - run: dart pub get                   # fix 1: устанавливаем зависимости

      - run: dart analyze                   # fix 2: статический анализ до тестов

      # fix 3: запускаем тесты ОДИН РАЗ с флагом покрытия
      - run: dart test --coverage=coverage

      - run: dart pub global activate coverage
      - run: |
          dart pub global run coverage:format_coverage \
            --lcov \
            --in=coverage \
            --out=coverage/lcov.info \
            --packages=.dart_tool/package_config.json   # fix 4: указываем packages
```

```dart
// test/api_test.dart  
import 'package:test/test.dart';

// fix 5: помечаем медленные тесты тегом, чтобы их можно было пропустить
// запуск без медленных тестов: dart test --exclude-tags=integration
test(
  'calls external API (simulated)',
  () async {
    await Future.delayed(const Duration(milliseconds: 50));
    expect(true, isTrue);
  },
  tags: ['integration'],           // fix 5: тег для фильтрации
  timeout: const Timeout(Duration(seconds: 5)),  // fix 6: таймаут > задержки
);
```

**Шесть багов:** 1) `dart pub get` пропущен — тесты упадут с «package not found». 2) Нет `dart analyze` — статические ошибки не обнаруживаются. 3) Тесты запускаются дважды: без покрытия и с покрытием — первый прогон лишний. 4) `--packages` не указан — `format_coverage` не сможет сопоставить файлы с пакетами. 5) Медленный/интеграционный тест без тега нельзя исключить из быстрого CI-прогона. 6) Таймаут 2 секунды при задержке 3 секунды — тест всегда будет завершаться по таймауту.
</details>
