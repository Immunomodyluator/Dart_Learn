Напиши unit-тесты для готового класса `Calculator`.

```dart
// lib/calculator.dart — уже существует
class Calculator {
  double add(double a, double b) => a + b;
  double subtract(double a, double b) => a - b;
  double multiply(double a, double b) => a * b;
  double divide(double a, double b) {
    if (b == 0) throw ArgumentError('Division by zero');
    return a / b;
  }
  int factorial(int n) {
    if (n < 0) throw RangeError.value(n, 'n', 'must be >= 0');
    if (n == 0) return 1;
    return n * factorial(n - 1);
  }
}
```

Напиши файл `test/calculator_test.dart`, который:

1. Содержит `group('Calculator', ...)` с вложенными группами для каждой операции.
2. Для `divide`: проверяет нормальное деление **и** что `ArgumentError` выбрасывается при делении на 0.
3. Для `factorial`: тестирует `0! = 1`, `5! = 120`, а также что `RangeError` выбрасывается для `n < 0`.
4. Содержит асинхронный тест: `Future<double> asyncAdd(double a, double b)` — функция с задержкой 10 мс; проверь результат через `expectLater`.

```dart
import 'dart:async';
import 'package:test/test.dart';

// Встраиваем Calculator прямо в тест для примера
class Calculator {
  double add(double a, double b) => a + b;
  double subtract(double a, double b) => a - b;
  double multiply(double a, double b) => a * b;
  double divide(double a, double b) {
    if (b == 0) throw ArgumentError('Division by zero');
    return a / b;
  }
  int factorial(int n) {
    if (n < 0) throw RangeError.value(n, 'n', 'must be >= 0');
    if (n == 0) return 1;
    return n * factorial(n - 1);
  }
}

Future<double> asyncAdd(double a, double b) async {
  await Future.delayed(const Duration(milliseconds: 10));
  return a + b;
}

void main() {
  late Calculator calc;

  setUp(() => calc = Calculator());

  group('Calculator', () {
    group('add', () {
      test('returns sum of two numbers', () {
        expect(calc.add(2, 3), equals(5));
        expect(calc.add(-1, 1), equals(0));
      });
    });

    group('subtract', () {
      test('returns difference', () {
        expect(calc.subtract(10, 3), equals(7));
      });
    });

    group('multiply', () {
      test('returns product', () {
        expect(calc.multiply(4, 5), equals(20));
        expect(calc.multiply(0, 100), equals(0));
      });
    });

    group('divide', () {
      test('returns quotient', () {
        expect(calc.divide(10, 2), equals(5));
        expect(calc.divide(7, 2), equals(3.5));
      });

      test('throws ArgumentError when dividing by zero', () {
        expect(() => calc.divide(5, 0), throwsA(isA<ArgumentError>()));
      });
    });

    group('factorial', () {
      test('0! equals 1', () => expect(calc.factorial(0), equals(1)));
      test('5! equals 120', () => expect(calc.factorial(5), equals(120)));
      test('throws RangeError for negative input', () {
        expect(() => calc.factorial(-1), throwsRangeError);
      });
    });
  });

  group('asyncAdd', () {
    test('returns sum asynchronously', () async {
      final result = asyncAdd(3, 4);
      await expectLater(result, completion(equals(7)));
    });
  });
}
```

`setUp` создаёт свежий экземпляр перед каждым тестом. `throwsA(isA<MyException>())` — типобезопасная проверка типа исключения; для стандартных есть сокращения (`throwsArgumentError`, `throwsRangeError`). `expectLater` + `completion` используется для проверки Future — альтернатива `expect(await future, ...)`.
