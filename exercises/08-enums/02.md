# Задание 02 (8.1): Базовые enum — 🐛 Найти баг

**Тема:** 08. Перечисления | **Подтема:** 8.1 Базовые enum

---

Код не компилируется или ведёт себя неожиданно. Найди и исправь все ошибки.

```dart
enum Direction { north, south, east, west }

Direction opposite(Direction d) {
  switch (d) {
    case Direction.north: return Direction.south;
    case Direction.south: return Direction.north;
    case Direction.east:  return Direction.west;
    // west не обработан — switch неполный, но Dart не всегда ловит в statement
  }
  // функция не всегда возвращает значение
}

void main() {
  print(opposite(Direction.north)); // south
  print(opposite(Direction.west));  // должен вернуть east — но вернёт null

  // Парсинг из строки
  final d = Direction.valueOf('east'); // нет такого метода
  print(d);
}
```

<details>
<summary>Решение</summary>

```dart
enum Direction { north, south, east, west }

Direction opposite(Direction d) => switch (d) {
  Direction.north => Direction.south,
  Direction.south => Direction.north,
  Direction.east  => Direction.west,
  Direction.west  => Direction.east,   // добавлен west
};

void main() {
  print(opposite(Direction.north)); // Direction.south
  print(opposite(Direction.west));  // Direction.east

  // Парсинг из строки — правильный способ
  final d = Direction.values.byName('east'); // byName, не valueOf
  print(d); // Direction.east
}
```

**Три бага:** 1) `switch`-оператор не обработал `Direction.west` — функция возвращала `null` (не компилируется в Dart с non-nullable типами). 2) `switch`-выражение вместо оператора гарантирует исчерпывающий анализ компилятором. 3) `Direction.valueOf` не существует — для парсинга строки в enum используют `EnumName.values.byName('string')`.

</details>
