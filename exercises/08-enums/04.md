# Задание 04 (8.2): Расширенные enum (поля, методы, интерфейсы) — 🐛 Найти баг

**Тема:** 08. Перечисления | **Подтема:** 8.2 Расширенные enum

---

Код не компилируется. Найди и исправь все ошибки в расширенном enum.

```dart
enum HttpMethod {
  get('GET'),
  post('POST'),
  put('PUT'),
  delete('DELETE')   // нет точки с запятой перед телом

  final String value;       // поле должно быть до конструктора? нет...
  HttpMethod(this.value);   // конструктор не const

  String toHeader() => 'X-Method: ' + value;

  // нельзя переопределять name — это встроенный getter
  String get name => value.toLowerCase();
}

void main() {
  print(HttpMethod.get.value);     // GET
  print(HttpMethod.post.toHeader()); // X-Method: POST
}
```

<details>
<summary>Решение</summary>

```dart
enum HttpMethod {
  get('GET'),
  post('POST'),
  put('PUT'),
  delete('DELETE');   // точка с запятой перед телом enum

  final String value;
  const HttpMethod(this.value);   // конструктор обязательно const

  String toHeader() => 'X-Method: $value';

  // name — зарезервированный getter, переименовать
  String get displayName => value.toLowerCase();
}

void main() {
  print(HttpMethod.get.value);           // GET
  print(HttpMethod.post.toHeader());     // X-Method: POST
  print(HttpMethod.delete.displayName);  // delete
}
```

**Три бага:** 1) Отсутствует `;` после последнего варианта — обязательно при наличии тела. 2) Конструктор enum должен быть `const` (enum — compile-time константы). 3) `name` — встроенный getter `String` в каждом enum-варианте (возвращает имя варианта); переопределить его нельзя — используй другое имя.
