# Задание 10 (7.5): typedef и функциональные типы — 🐛 Найти баг

**Тема:** 07. Функции | **Подтема:** 7.5 typedef и функциональные типы

---

Код не компилируется из-за неверного использования `typedef` и несовместимых типов. Найди и исправь все ошибки.

```dart
typedef IntOp = int Function(int, int);

IntOp add = (a, b) => a + b;
IntOp mul = (a, b, c) => a * b * c;  // три аргумента вместо двух

// Старый стиль typedef — определяет функцию, а не тип
typedef void Printer(String msg);     // устаревший синтаксис

Printer log = (msg) => print('[LOG] $msg');

void apply(IntOp op, int x, int y) => print(op(x, y));

void main() {
  apply(add, 3, 4);      // 7
  apply(mul, 3, 4);      // должно быть 12 (3*4)
  log('hello');          // [LOG] hello
}
```

<details>
<summary>Решение</summary>

```dart
typedef IntOp = int Function(int, int);

// Современный синтаксис typedef (Dart 2+)
typedef Printer = void Function(String msg);

final IntOp add = (a, b) => a + b;
final IntOp mul = (a, b) => a * b;  // два аргумента — соответствует IntOp

final Printer log = (msg) => print('[LOG] $msg');

void apply(IntOp op, int x, int y) => print(op(x, y));

void main() {
  apply(add, 3, 4); // 7
  apply(mul, 3, 4); // 12
  log('hello');     // [LOG] hello
}
```

**Три бага:** 1) `mul = (a, b, c) => a * b * c` — три параметра не совместимы с `IntOp = int Function(int, int)`; сигнатура должна совпадать точно. 2) `typedef void Printer(String msg)` — устаревший синтаксис, не поддерживает generics; используй `typedef Printer = void Function(String)`. 3) Переменные верхнего уровня стоит объявлять через `final`.
</details>
