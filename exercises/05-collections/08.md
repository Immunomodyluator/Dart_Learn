Код выводит неожиданный результат и работает медленно при большом списке. Найди и объясни баг, исправь его.

```dart
void main() {
  final numbers = [1, 2, 3, 4, 5];

  final evens = numbers.where((n) => n.isEven);

  // "Кешируем" длину
  final len = evens.length;

  // Проверка совпадения
  if (evens.length == len) {
    print('Lengths match: $len');
  }

  // Конвертация в список строк
  final strs = evens.map((n) => n.toString());
  print(strs.runtimeType); // не []
  print(strs.first);
  print(strs.first);       // вычисляется снова
}
```

<summary>Решение</summary>

```dart
void main() {
  final numbers = [1, 2, 3, 4, 5];

  // Материализовать Iterable → List один раз
  final evens = numbers.where((n) => n.isEven).toList();

  final len = evens.length;         // O(1) на List

  if (evens.length == len) {
    print('Lengths match: $len');   // 2
  }

  final strs = evens.map((n) => n.toString()).toList();
  print(strs.runtimeType);          // List<String>
  print(strs.first);                // 2
  print(strs.first);                // 2  — без повторного вычисления
}
```

`.where()` и `.map()` возвращают **ленивые** `Iterable`. Каждый вызов `.length`, `.first` и т.п. проходит исходную цепочку заново — O(n) при каждом обращении. Вызов `.toList()` материализует результат в `List`: последующие операции O(1) и элементы не пересчитываются.
