# Задание 08 (9.4): Интерфейсы и implements — 🐛 Найти баг

**Тема:** 09. ООП | **Подтема:** 9.4 Интерфейсы и implements

---

Код не компилируется: класс не реализует все члены интерфейса. Найди и исправь все ошибки.

```dart
abstract class Serializable {
  String serialize();
  static Serializable deserialize(String data);  // статика в интерфейсе
}

class User implements Serializable {
  final String name;
  final int age;

  User(this.name, this.age);

  @override
  String serialize() => '$name:$age';

  // deserialize не реализован
}

void main() {
  final u = User('Alice', 30);
  print(u.serialize());  // Alice:30

  final u2 = Serializable.deserialize('Bob:25');  // статика не наследуется
  print(u2.serialize());
}
```

<summary>Решение</summary>

```dart
abstract class Serializable {
  String serialize();
  // Статические методы не входят в интерфейс и не наследуются — убираем из abstract
}

class User implements Serializable {
  final String name;
  final int age;

  User(this.name, this.age);

  @override
  String serialize() => '$name:$age';

  // Фабричный конструктор или статический метод на самом классе
  static User deserialize(String data) {
    final parts = data.split(':');
    return User(parts[0], int.parse(parts[1]));
  }
}

void main() {
  final u = User('Alice', 30);
  print(u.serialize());              // Alice:30

  final u2 = User.deserialize('Bob:25');
  print(u2.serialize());             // Bob:25
}
```

**Два бага:** 1) `static` методы в `abstract class` не являются частью интерфейса — их нельзя переопределить или вызвать полиморфно; перенеси `deserialize` в конкретный класс. 2) При `implements` нужно реализовать все **нестатические** абстрактные члены — здесь `serialize` уже реализован; `deserialize` как статику удаляем из требований интерфейса.

</details>
