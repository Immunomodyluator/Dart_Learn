# Задание 07 (9.4): Интерфейсы и implements — 🖊 Написать код

**Тема:** 09. ООП | **Подтема:** 9.4 Интерфейсы и implements

---

Реализуй паттерн «стратегия» через неявные интерфейсы Dart:

1. Объяви абстрактный класс `Sorter<T>` с методом `List<T> sort(List<T> items)`.
2. `AscSorter<T extends Comparable>` — сортирует по возрастанию.
3. `DescSorter<T extends Comparable>` — сортирует по убыванию.
4. Функция `printSorted<T extends Comparable>(List<T> items, Sorter<T> sorter)`.

<details>
<summary>Решение</summary>

```dart
abstract class Sorter<T> {
  List<T> sort(List<T> items);
}

class AscSorter<T extends Comparable<T>> implements Sorter<T> {
  @override
  List<T> sort(List<T> items) {
    final copy = List<T>.from(items);
    copy.sort((a, b) => a.compareTo(b));
    return copy;
  }
}

class DescSorter<T extends Comparable<T>> implements Sorter<T> {
  @override
  List<T> sort(List<T> items) {
    final copy = List<T>.from(items);
    copy.sort((a, b) => b.compareTo(a));
    return copy;
  }
}

void printSorted<T extends Comparable<T>>(List<T> items, Sorter<T> sorter) {
  print(sorter.sort(items));
}

void main() {
  final nums = [3, 1, 4, 1, 5, 9];
  printSorted(nums, AscSorter());  // [1, 1, 3, 4, 5, 9]
  printSorted(nums, DescSorter()); // [9, 5, 4, 3, 1, 1]
}
```

В Dart каждый класс неявно определяет интерфейс. `implements` требует переопределить **все** методы — нет наследования реализации. Это позволяет применять несколько «интерфейсов» (нескольких `implements`): `class Foo implements A, B, C`.
