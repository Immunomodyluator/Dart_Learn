# Задание 10 (9.5): Mixin — 🐛 Найти баг

**Тема:** 09. ООП | **Подтема:** 9.5 Mixin

---

Код не компилируется. Найди и исправь все ошибки, связанные с миксинами.

```dart
mixin Logging {
  int _logCount = 0;     // переменная в миксине — ok?

  void log(String msg) {
    _logCount++;
    print('[$_logCount] $msg');
  }
}

// on ограничение — только для подклассов Animal
mixin Trainable on Animal {
  void train(String command) => print('Training: $command');
}

class Animal {}

// Применяем Trainable к классу без extends Animal
class Robot with Trainable {
  final String id;
  Robot(this.id);
}

// Mixin с конструктором
mixin Identifiable {
  Identifiable(String id);  // конструктор в mixin — запрещено
  String get identity;
}

void main() {
  final r = Robot('R2D2');
  r.train('sit');
}
```

<details>
<summary>Решение</summary>

```dart
mixin Logging {
  int _logCount = 0; // ok — нестатическое поле в mixin разрешено

  void log(String msg) {
    _logCount++;
    print('[$_logCount] $msg');
  }
}

class Animal {}

mixin Trainable on Animal {
  void train(String command) => print('Training: $command');
}

// Robot должен наследовать Animal для on-ограничения
class Robot extends Animal with Logging {
  final String id;
  Robot(this.id);
}

// Mixin без конструктора — только геттер/методы
mixin Identifiable {
  String get identity; // абстрактный геттер — ok
}

void main() {
  final r = Robot('R2D2');
  r.log('booted');   // [1] booted
}
```

**Три бага:** 1) `Robot with Trainable` — нарушает `on Animal`: `Robot` должен расширять `Animal`. 2) `mixin Identifiable { Identifiable(String id); }` — миксины **не могут** иметь конструкторы; убрать. 3) Поля (нестатические) в миксинах разрешены — `_logCount` корректен.
