# Задание 01 (9.1): Классы и конструкторы — 🖊 Написать код

**Тема:** 09. ООП | **Подтема:** 9.1 Классы и конструкторы

---

Напиши класс `BankAccount` со следующей структурой:

- Приватное поле `double _balance`.
- Основной конструктор: принимает `String owner` и начальный `double balance` (по умолчанию 0.0).
- Именованный конструктор `BankAccount.empty(String owner)` — баланс 0.
- Метод `void deposit(double amount)` — бросает `ArgumentError` при amount ≤ 0.
- Метод `void withdraw(double amount)` — бросает `StateError` при недостатке средств.
- Getter `double balance`.
- `@override String toString()`.

<details>
<summary>Решение</summary>

```dart
class BankAccount {
  final String owner;
  double _balance;

  BankAccount(this.owner, [double balance = 0.0]) : _balance = balance;

  BankAccount.empty(this.owner) : _balance = 0.0;

  double get balance => _balance;

  void deposit(double amount) {
    if (amount <= 0) throw ArgumentError('Amount must be positive');
    _balance += amount;
  }

  void withdraw(double amount) {
    if (amount <= 0) throw ArgumentError('Amount must be positive');
    if (amount > _balance) throw StateError('Insufficient funds');
    _balance -= amount;
  }

  @override
  String toString() => 'BankAccount($owner, \$$_balance)';
}

void main() {
  final acc = BankAccount('Alice', 100.0);
  acc.deposit(50);
  acc.withdraw(30);
  print(acc); // BankAccount(Alice, $120.0)

  final empty = BankAccount.empty('Bob');
  print(empty.balance); // 0.0
}
```

`this.field` в параметре конструктора — сокращение для `_balance = balance`. Инициализатор после `:` запускается до тела конструктора. Именованный конструктор `BankAccount.empty` при вызове явно документирует намерение создать пустой счёт.
