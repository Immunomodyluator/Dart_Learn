# Задание 19 (9.10): Sealed-классы — 🖊 Написать код

**Тема:** 09. ООП | **Подтема:** 9.10 Sealed-классы (Dart 3)

---

Реализуй результирующий тип `Result<T>` через `sealed class`:

- `sealed class Result<T>`
- `final class Ok<T> extends Result<T>` — поле `T value`
- `final class Err<T> extends Result<T>` — поле `String message`
- Функция `Result<int> parseInt(String s)` — `Ok` при успехе, `Err` при исключении.
- Функция `void handle<T>(Result<T> r)` — exhaustive switch для обработки.

<details>
<summary>Решение</summary>

```dart
sealed class Result<T> {}

final class Ok<T> extends Result<T> {
  final T value;
  const Ok(this.value);
}

final class Err<T> extends Result<T> {
  final String message;
  const Err(this.message);
}

Result<int> parseInt(String s) {
  try {
    return Ok(int.parse(s));
  } on FormatException catch (e) {
    return Err(e.message);
  }
}

void handle<T>(Result<T> r) {
  switch (r) {
    case Ok(:final value):
      print('Success: $value');
    case Err(:final message):
      print('Error: $message');
  }
}

void main() {
  handle(parseInt('42'));    // Success: 42
  handle(parseInt('abc'));   // Error: Invalid radix-10 number (at character 1)
  handle(parseInt('-7'));    // Success: -7
}
```

`sealed class` — все прямые подтипы должны быть в одном файле; это даёт компилятору знание об исчерпывающем наборе вариантов. `switch` по `sealed class` без `_` — компилятор статически проверяет полноту. Паттерн `case Ok(:final value)` — деструктуризация поля напрямую.

</details>
