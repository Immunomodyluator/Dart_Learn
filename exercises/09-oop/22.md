# Задание 22 (9.11): Cascade notation — 🐛 Найти баг

**Тема:** 09. ООП | **Подтема:** 9.11 Cascade notation

---

Каскад использован неверно: возвращается не тот объект. Найди и исправь все ошибки.

```dart
class Builder {
  String _name = '';
  int _value = 0;

  Builder setName(String name) {
    _name = name;
    return this;
  }

  Builder setValue(int value) {
    _value = value;
    return this;
  }

  Map<String, Object> build() => {'name': _name, 'value': _value};
}

void main() {
  // Попытка использовать cascade и получить результат build()
  final result = Builder()
    ..setName('Alice')
    ..setValue(42)
    ..build();    // cascade возвращает Builder, не Map

  print(result);  // ожидается {name: Alice, value: 42}
                  // получается Instance of 'Builder'
}
```

<summary>Решение</summary>

```dart
class Builder {
  String _name = '';
  int _value = 0;

  Builder setName(String name) {
    _name = name;
    return this;
  }

  Builder setValue(int value) {
    _value = value;
    return this;
  }

  Map<String, Object> build() => {'name': _name, 'value': _value};
}

void main() {
  // Вариант 1: каскад для настройки, потом .build() обычным вызовом
  final builder = Builder()
    ..setName('Alice')
    ..setValue(42);
  final result = builder.build();
  print(result); // {name: Alice, value: 42}

  // Вариант 2: метод-chaining (return this) без каскада
  final result2 = Builder()
      .setName('Bob')
      .setValue(99)
      .build();
  print(result2); // {name: Bob, value: 99}
}
```

**Баг:** `..build()` применяет `build()` через каскад, но **возвращает `Builder`**, а не `Map`. Каскадный оператор всегда возвращает **получателя** (левый операнд), игнорируя возвращаемое значение метода. Решение: завершить каскад, а `build()` вызвать отдельно — либо применить метод-chaining (`.` вместо `..`) и вызвать `build()` последним.
