# Задание 16 (9.8): Фабричные конструкторы — 🐛 Найти баг

**Тема:** 09. ООП | **Подтема:** 9.8 Фабричные конструкторы

---

`factory` конструктор не работает как ожидается. Найди и исправь все ошибки.

```dart
class Database {
  static Database? _instance;

  factory Database() {
    if (_instance == null) {
      _instance = Database();  // рекурсивный вызов factory
    }
    return _instance!;
  }

  void query(String sql) => print('Running: $sql');
}

class Shape {
  factory Shape(String type) {
    switch (type) {
      case 'circle':
        return Circle();
      case 'square':
        return Square();
    }
    // нет default — factory может не вернуть ничего
  }
}

class Circle extends Shape {}
class Square extends Shape {}

void main() {
  final db = Database();
  db.query('SELECT 1');
}
```

<details>
<summary>Решение</summary>

```dart
class Database {
  static Database? _instance;

  // Приватный обычный конструктор
  Database._internal();

  factory Database() {
    _instance ??= Database._internal(); // без рекурсии
    return _instance!;
  }

  void query(String sql) => print('Running: $sql');
}

class Shape {
  factory Shape(String type) {
    return switch (type) {
      'circle' => Circle._(),
      'square' => Square._(),
      _ => throw ArgumentError('Unknown shape: $type'),
    };
  }
}

class Circle extends Shape { Circle._() : super._(); }
class Square extends Shape { Square._() : super._(); }

// Добавим приватный конструктор в Shape для подклассов
// Упрощённый вариант без super:
void main() {
  final db1 = Database();
  final db2 = Database();
  print(identical(db1, db2)); // true

  db1.query('SELECT 1'); // Running: SELECT 1
}
```

**Два бага:** 1) `_instance = Database()` в `factory` рекурсивно вызывает тот же `factory` — бесконечная рекурсия и `StackOverflow`; нужен приватный конструктор `Database._internal()`. 2) `factory Shape` без `default`/`_` ветки — компилятор требует, чтобы все пути возвращали значение; добавлен `throw ArgumentError`.

</details>
