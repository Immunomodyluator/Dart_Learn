# Задание 12 (9.6): Extension methods — 🐛 Найти баг

**Тема:** 09. ООП | **Подтема:** 9.6 Extension methods

---

Extension-метод вызывает конфликт имён и неверно определён. Найди и исправь все ошибки.

```dart
extension IntExtensions on int {
  // Попытка переопределить существующий метод abs()
  int abs() => this < 0 ? -this : this;

  int clamp(int min, int max) => this < min ? min : this > max ? max : this;
  // clamp уже есть у num — конфликт

  // Статический метод в extension
  static int fromHex(String hex) => int.parse(hex, radix: 16);
}

void main() {
  print((-5).abs());      // ожидается 5
  print(10.clamp(0, 8));  // ожидается 8
  print(IntExtensions.fromHex('FF')); // 255
}
```

<details>
<summary>Решение</summary>

```dart
extension IntExtensions on int {
  // abs() и clamp() уже существуют у int/num — переименовать
  int absoluteValue() => this < 0 ? -this : this;

  // Убрать clamp — конфликт с num.clamp; если нужно — другое имя
  int clamped(int lo, int hi) => this < lo ? lo : this > hi ? hi : this;

  // Статический метод в extension — допустим, но вызывается через имя extension
  static int fromHex(String hex) => int.parse(hex, radix: 16);
}

void main() {
  print((-5).absoluteValue());             // 5
  print(10.clamped(0, 8));                 // 8
  print(IntExtensions.fromHex('FF'));      // 255
  print((-3).abs());                       // встроенный abs() — 3
}
```

**Два бага:** 1) Нельзя переопределить реальный метод класса через extension — если метод уже существует у типа, extension-версия **никогда не вызовется** (встроенный побеждает). Нужно другое имя. 2) `clamp` тоже существует у `num` — те же правила; переименовать в `clamped`. Статические методы в extension — допустимы, вызываются через имя extension.

</details>
