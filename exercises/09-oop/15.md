# Задание 15 (9.8): Фабричные конструкторы — 🖊 Написать код

**Тема:** 09. ООП | **Подтема:** 9.8 Фабричные конструкторы

---

Реализуй синглтон и кэширующую фабрику:

1. `class Config` — синглтон через `factory` конструктор. Поле `Map<String, String> _data`. Метод `String? get(String key)`.
2. `class Color` — кэш уже созданных объектов. `factory Color(int r, int g, int b)` возвращает существующий экземпляр, если цвет уже создавался (ключ `'$r,$g,$b'`).

<details>
<summary>Решение</summary>

```dart
class Config {
  static final Config _instance = Config._internal();
  final Map<String, String> _data = {};

  factory Config() => _instance;

  Config._internal();

  void set(String key, String value) => _data[key] = value;
  String? get(String key) => _data[key];
}

class Color {
  static final Map<String, Color> _cache = {};

  final int r, g, b;

  factory Color(int r, int g, int b) {
    final key = '$r,$g,$b';
    return _cache.putIfAbsent(key, () => Color._create(r, g, b));
  }

  Color._create(this.r, this.g, this.b);

  @override
  String toString() => 'Color($r, $g, $b)';
}

void main() {
  final c1 = Config();
  c1.set('theme', 'dark');
  final c2 = Config();
  print(c2.get('theme'));      // dark
  print(identical(c1, c2));   // true — один объект

  final red1 = Color(255, 0, 0);
  final red2 = Color(255, 0, 0);
  print(identical(red1, red2)); // true — из кэша
}
```

`factory` конструктор может вернуть **существующий** объект, не обязательно новый. Синглтон: `static final` поле инициализируется один раз, `factory` всегда возвращает его. Кэш: `putIfAbsent` добавляет только если ключа нет — иначе возвращает существующее значение.
