# Задание 03 (9.2): Наследование — 🖊 Написать код

**Тема:** 09. ООП | **Подтема:** 9.2 Наследование

---

Реализуй иерархию классов:

- `Animal` — базовый класс с полем `String name`, конструктором и методом `String speak()` → `'...'`.
- `Dog extends Animal` — переопределяет `speak()` → `'Woof!'`, добавляет `void fetch(String item)` → печатает `'$name fetches $item'`.
- `GoldenRetriever extends Dog` — переопределяет `speak()` → `'Woof woof!'`.

Конструктор каждого подкласса должен вызывать `super(name)`.

<details>
<summary>Решение</summary>

```dart
class Animal {
  final String name;
  Animal(this.name);
  String speak() => '...';
  @override
  String toString() => '$runtimeType($name)';
}

class Dog extends Animal {
  Dog(super.name);

  @override
  String speak() => 'Woof!';

  void fetch(String item) => print('$name fetches $item');
}

class GoldenRetriever extends Dog {
  GoldenRetriever(super.name);

  @override
  String speak() => 'Woof woof!';
}

void main() {
  final a = Animal('Cat');
  final d = Dog('Rex');
  final g = GoldenRetriever('Buddy');

  print(a.speak());       // ...
  print(d.speak());       // Woof!
  d.fetch('ball');        // Rex fetches ball
  print(g.speak());       // Woof woof!
  g.fetch('stick');       // Buddy fetches stick

  // Полиморфизм
  for (final animal in <Animal>[a, d, g]) {
    print('${animal.name}: ${animal.speak()}');
  }
}
```

`super.name` в параметрах конструктора (Dart 2.17+) — краткая запись вместо `Dog(String name) : super(name)`. `@override` даёт компилятору проверить, что метод действительно существует в базовом классе. `runtimeType` возвращает тип объекта во время выполнения.
