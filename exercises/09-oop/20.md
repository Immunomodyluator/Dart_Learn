# Задание 20 (9.10): Sealed-классы — 🐛 Найти баг

**Тема:** 09. ООП | **Подтема:** 9.10 Sealed-классы (Dart 3)

---

Код не компилируется или sealed-иерархия нарушена. Найди и исправь все ошибки.

```dart
// файл: shape.dart
sealed class Shape {}

class Circle extends Shape {   // не final — можно расширять снаружи
  final double radius;
  Circle(this.radius);
}

// Подтип sealed-класса объявлен в другом файле (предположим)
// — это нарушение sealed

double area(Shape s) => switch (s) {
  Circle c => 3.14 * c.radius * c.radius,
  // Rectangle не обработан — допустим?
};

class Rectangle extends Shape {
  final double w, h;
  Rectangle(this.w, this.h);
}

void main() {
  print(area(Circle(5)));
  print(area(Rectangle(3, 4))); // switch неполный
}
```

<details>
<summary>Решение</summary>

```dart
// Все подтипы sealed-класса должны быть в том же файле

sealed class Shape {}

final class Circle extends Shape {    // final — нельзя расширять снаружи
  final double radius;
  Circle(this.radius);
}

final class Rectangle extends Shape { // перенесён сюда же, добавлен final
  final double w, h;
  Rectangle(this.w, this.h);
}

double area(Shape s) => switch (s) {
  Circle c    => 3.14159 * c.radius * c.radius,
  Rectangle r => r.w * r.h,            // добавлен case Rectangle
};

void main() {
  print(area(Circle(5)).toStringAsFixed(2));    // 78.54
  print(area(Rectangle(3, 4)));                 // 12.0
}
```

**Три бага:** 1) Подтипы `sealed class` обязаны быть в **том же файле** — иначе ошибка компиляции (в Dart это same-library restriction). 2) `Circle` не `final` — рекомендуется `final` или `base` для подтипов sealed, чтобы исключить сторонние расширения. 3) `switch` не обрабатывает `Rectangle` — неисчерпывающий switch по sealed-типу является ошибкой компиляции.

</details>
