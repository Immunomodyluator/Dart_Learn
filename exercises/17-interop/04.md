# Задание 04 (17.2): JS interop для веба — 🐛 Найти баг

**Тема:** 17. Interop: FFI и Web | **Подтема:** 17.2 JS interop для веба

---

В JS interop коде есть ошибки. Найди их.

```dart
import 'dart:js_interop';

// bug 1 — устаревший API: не используй dart:js в новом коде (Dart 3.3+)
import 'dart:js' as js;

// bug 2 — неверный синтаксис привязки: @JS должен быть на extension type
@JS()
class Window {          // должен быть extension type, не class
  external String get location;
  external void alert(String message);
}

// bug 3 — геттер не помечен как external
extension type Document._(JSObject _) implements JSObject {
  String get title {   // нет 'external' — это Dart реализация, не JS
    return 'hardcoded';
  }
  external void getElementById(String id);  // возвращает void, должен возвращать JSObject?
}

@JS('console')
external JSObject get console;

void logToConsole(String msg) {
  // bug 4 — обращаемся к console.log неверно (нужен extension type или callMethod)
  console.log(msg);  // JSObject не имеет метода .log
}

void main() {
  // bug 5 — dart:html работает только в браузере,
  //         но здесь может быть импортирован вместе с dart:js_interop конфликтуя
  final doc = Document._();  // bug 6 — нельзя создать extension type через приватный конструктор
  print(doc.title);
}
```

<details>
<summary>Решение</summary>

```dart
import 'dart:js_interop';
// fix 1: убираем dart:js — используем только dart:js_interop

// fix 2: правильный синтаксис — extension type
@JS('Window')
extension type Window._(JSObject _) implements JSObject {
  external String get location;
  external void alert(String message);
}

// fix 3: все методы/геттеры должны быть external
extension type Document._(JSObject _) implements JSObject {
  external String get title;             // fix: external getter
  external JSObject? getElementById(String id); // fix: правильный возвращаемый тип
}

// Правильная привязка для console
@JS('console')
extension type Console._(JSObject _) implements JSObject {
  external void log(JSAny? value);
  external void error(JSAny? value);
}

@JS('console')
external Console get jsConsole;

// fix 4: используем правильный extension type
void logToConsole(String msg) {
  jsConsole.log(msg.toJS); // конвертируем String → JSString
}

// Глобальный window и document
@JS()
external Window get window;

@JS()
external Document get document;

void main() {
  // fix 6: не создаём extension type напрямую — получаем через JS
  logToConsole('Hello from Dart!');

  final title = document.title;
  print('Page title: $title');

  final btn = document.getElementById('myButton');
  if (btn != null) {
    // работаем с элементом...
  }
}
```

**Шесть проблем:** 1) `dart:js` устарел в Dart 3.3 — используй `dart:js_interop`. 2) `@JS()` + `class` — старый синтаксис; нужен `extension type`. 3) Геттер `title` без `external` — это Dart-реализация, возвращающая `'hardcoded'`, а не реальный заголовок страницы. 4) `JSObject` не имеет метода `.log`; нужен `extension type Console` с `external void log(...)`. 5) (В данном примере `dart:html` не импортирован — это предупреждение, не активный баг.) 6) `Document._()` — приватный конструктор extension type нельзя вызвать из внешнего кода; экземпляры получают через `@JS()` геттеры.
