# Задание 02 (17.1): Dart FFI — 🐛 Найти баг

**Тема:** 17. Interop: FFI и Web | **Подтема:** 17.1 Dart FFI

В коде FFI допущены опасные ошибки (утечки памяти, неверные типы). Найди их все.

```dart
import 'dart:ffi';
import 'package:ffi/ffi.dart';

// Нативная структура Point
final class NativePoint extends Struct {
  @Int32()
  external int x;
  @Int32()
  external int y;
}

typedef NativeDistance = Double Function(Pointer<NativePoint>, Pointer<NativePoint>);
typedef DartDistance = double Function(Pointer<NativePoint>, Pointer<NativePoint>);

// Предположим, есть C-функция: double distance(Point* a, Point* b)
late final DartDistance nativeDistance;

void init() {
  final lib = DynamicLibrary.open('mylib.so');
  nativeDistance = lib.lookupFunction<NativeDistance, DartDistance>('distance');
}

double calculateDistance(int x1, int y1, int x2, int y2) {
  // bug 1 — используем malloc вместо calloc (uninitialized memory)
  final p1 = malloc<NativePoint>();
  final p2 = malloc<NativePoint>();

  p1.ref.x = x1;
  p1.ref.y = y1;
  p2.ref.x = x2;
  p2.ref.y = y2;

  final result = nativeDistance(p1, p2);

  // bug 2 — нет освобождения памяти при исключении (нужен try/finally)
  // bug 3 — освобождаем только p1
  malloc.free(p1);

  return result; // p2 утечка!
}

String readNativeString(Pointer<Char> ptr) {
  // bug 4 — используем ptr после потенциального освобождения
  final result = ptr.cast<Utf8>().toDartString();
  calloc.free(ptr);  // предполагаем что ptr выделен через calloc
  return result;     // безопасно: строка уже скопирована в Dart heap
  // НО: метод не документирует что берёт ownership ptr — опасный API
}

void main() {
  init();
  // bug 5 — нет проверки: init() мог не инициализировать nativeDistance
  final d = calculateDistance(0, 0, 3, 4);
  print(d); // должно быть 5.0
}
```

<summary>Решение</summary>

```dart
import 'dart:ffi';
import 'package:ffi/ffi.dart';

final class NativePoint extends Struct {
  @Int32()
  external int x;
  @Int32()
  external int y;
}

typedef NativeDistance = Double Function(Pointer<NativePoint>, Pointer<NativePoint>);
typedef DartDistance = double Function(Pointer<NativePoint>, Pointer<NativePoint>);

DartDistance? _nativeDistance;

void init() {
  final lib = DynamicLibrary.open('mylib.so');
  _nativeDistance = lib.lookupFunction<NativeDistance, DartDistance>('distance');
}

double calculateDistance(int x1, int y1, int x2, int y2) {
  // fix 1: используем calloc — он инициализирует память нулями
  final p1 = calloc<NativePoint>();
  final p2 = calloc<NativePoint>();

  try {  // fix 2: try/finally гарантирует освобождение при любом исходе
    p1.ref.x = x1;
    p1.ref.y = y1;
    p2.ref.x = x2;
    p2.ref.y = y2;

    return _nativeDistance!(p1, p2);
  } finally {
    calloc.free(p1);  // fix 3: освобождаем оба указателя
    calloc.free(p2);
  }
}

/// Читает строку из нативного указателя и освобождает указатель.
///
/// ВНИМАНИЕ: после вызова [ptr] становится недействительным.
/// Указатель должен быть выделен через [calloc].
String readNativeString(Pointer<Char> ptr) {
  // fix 4: toDartString() копирует строку — это безопасно
  // Документируем ownership явно через dartdoc
  final result = ptr.cast<Utf8>().toDartString();
  calloc.free(ptr);
  return result;
}

void main() {
  init();
  // fix 5: проверяем инициализацию
  if (_nativeDistance == null) {
    throw StateError('FFI not initialized. Call init() first.');
  }
  final d = calculateDistance(0, 0, 3, 4);
  print(d); // 5.0
}
```

**Пять проблем:** 1) `malloc` выделяет **неинициализированную** память — поля могут содержать мусор; `calloc` обнуляет выделенную область. 2) Нет `try/finally` — при исключении в `nativeDistance` оба указателя утекут. 3) Забыт `malloc.free(p2)` — явная утечка нативной памяти (GC Dart не управляет ею). 4) Не баг по сути (строка уже скопирована), но API небезопасен без документирования ownership — добавь dartdoc. 5) `nativeDistance` как `late final` без инициализации бросит `LateInitializationError`; лучше `DartDistance?` с явной проверкой.
