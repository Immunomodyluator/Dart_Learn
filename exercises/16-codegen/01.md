Заполни пропуски (`___`) в командах и конфигурационных файлах для работы с `build_runner`.

**Команды build_runner:**

```bash
# Однократная генерация кода
dart run ___ build

# Запустить генерацию и следить за изменениями
dart run ___ ___

# Генерация только для конкретного пакета/файла
dart run ___ build ___ lib/models/

# Удалить все сгенерированные файлы
dart run ___ ___

# Генерация с принудительной перезаписью существующих файлов
dart run ___ build ___
```

**pubspec.yaml — нужные зависимости:**

```yaml
dependencies:
  json_annotation: ^4.8.0  # runtime аннотации

dev_dependencies:
  build_runner: ___   # сам build_runner: ^2.4.0
  ___: ^6.7.0         # генератор JSON: json_serializable
```

**build.yaml — конфигурация генераторов:**

```yaml
targets:
  $default:
    builders:
      ___:json_serializable:           # namespace: имя_пакета:имя_билдера
        ___:                           # ключ: options (настройки)
          # генерировать поле toJson?
          ___: true                    # ключ: create_to_json
          # использовать explicit_to_json для вложенных объектов?
          ___: true                    # ключ: explicit_to_json
          # какой суффикс у сгенерированных файлов?
          ___: '.g.dart'               # ключ: header (нет) — это field_rename...
```

<details>
<summary>Решение</summary>

**Команды:**

```bash
# Однократная генерация кода
dart run build_runner build

# Запустить и следить за изменениями
dart run build_runner watch

# Генерация для конкретной директории
dart run build_runner build --build-filter="lib/models/**"

# Удалить все сгенерированные файлы
dart run build_runner clean

# Перезаписать существующие файлы без вопросов
dart run build_runner build --delete-conflicting-outputs
```

**pubspec.yaml:**

```yaml
dependencies:
  json_annotation: ^4.8.0

dev_dependencies:
  build_runner: ^2.4.0
  json_serializable: ^6.7.0
```

**build.yaml:**

```yaml
targets:
  $default:
    builders:
      json_serializable:json_serializable:
        options:
          # включить генерацию toJson()
          create_to_json: true
          # для вложенных объектов вызывать их .toJson() явно
          explicit_to_json: true
          # переименование полей: none / snake_case / kebab_case / pascal_case
          field_rename: snake_case
    # опционально: ограничить какие файлы обрабатываются
    generate_for:
      include:
        - lib/models/**
      exclude:
        - lib/models/legacy/**
```

**build_runner** — это Dart-инструмент для запуска кодогенераторов. Генераторы описываются как «builders» и запускаются по настроенным правилам. Сгенерированные файлы (`.g.dart`, `.freezed.dart`) обычно коммитятся в репозиторий. `--delete-conflicting-outputs` помогает при конфликтах, но используй с осторожностью — затирает файлы без подтверждения.
