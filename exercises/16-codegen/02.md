# Задание 02 (16.1): build_runner: основы — 🐛 Найти баг

Настройка генерации кода содержит ошибки. Найди их все.

```dart
// lib/models/user.dart

import 'package:json_annotation/json_annotation.dart';

// bug 1 — не хватает part директивы для сгенерированного файла
// (без неё сгенерированный код не подключится)

@JsonSerializable()
class User {
  final int id;
  final String name;

  // bug 2 — аннотация @JsonKey используется неверно
  @JsonKey(name: 'email_address', required: true, disallowNullValue: true)
  final String email;

  // bug 3 — DateTime не сериализуется автоматически без fromJson/toJson
  final DateTime createdAt;

  User(this.id, this.name, this.email, this.createdAt);

  // bug 4 — фабричный конструктор написан вручную вместо делегирования в сгенерированный
  factory User.fromJson(Map<String, dynamic> json) {
    return User(
      json['id'] as int,
      json['name'] as String,
      json['email_address'] as String,
      DateTime.parse(json['createdAt'] as String),
    );
  }

  // bug 5 — toJson написан вручную; нужно делегировать в _$UserToJson
  Map<String, dynamic> toJson() => {
        'id': id,
        'name': name,
        'email': email,    // bug 6 — неверное имя поля (должно быть 'email_address')
        'createdAt': createdAt.toString(),
      };
}
```

<details>
<summary>Решение</summary>

```dart
// lib/models/user.dart

import 'package:json_annotation/json_annotation.dart';

part 'user.g.dart';        // fix 1: подключаем сгенерированный файл

@JsonSerializable(
  explicitToJson: true,    // для вложенных объектов
)
class User {
  final int id;
  final String name;

  // fix 2: @JsonKey(name:) — корректная аннотация;
  // required и disallowNullValue устарели в новых версиях json_annotation
  @JsonKey(name: 'email_address')
  final String email;

  // fix 3: DateTime сериализуется автоматически как ISO 8601 строка
  // через встроенную поддержку json_serializable — дополнительная аннотация не нужна
  final DateTime createdAt;

  const User({
    required this.id,
    required this.name,
    required this.email,
    required this.createdAt,
  });

  // fix 4+5: делегируем в сгенерированные функции
  factory User.fromJson(Map<String, dynamic> json) => _$UserFromJson(json);
  Map<String, dynamic> toJson() => _$UserToJson(this);
}
```

После исправления запусти: `dart run build_runner build --delete-conflicting-outputs`

Сгенерированный `user.g.dart` будет содержать:

```dart
// GENERATED CODE - DO NOT MODIFY BY HAND

part of 'user.dart';

User _$UserFromJson(Map<String, dynamic> json) => User(
      id: json['id'] as int,
      name: json['name'] as String,
      email: json['email_address'] as String,
      createdAt: DateTime.parse(json['createdAt'] as String),
    );

Map<String, dynamic> _$UserToJson(User instance) => <String, dynamic>{
      'id': instance.id,
      'name': instance.name,
      'email_address': instance.email,    // правильное имя поля!
      'createdAt': instance.createdAt.toIso8601String(),
    };
```

**Шесть проблем:** 1) Без `part 'user.g.dart'` генерация пройдёт, но код в `user.g.dart` не будет доступен. 2) `required`/`disallowNullValue` устарели — используй `required` в конструкторе Dart. 3) `DateTime` поддерживается нативно — ручная сериализация не нужна. 4) Ручной `fromJson` дублирует логику и её нужно поддерживать вручную — при изменении модели легко забыть обновить. 5) Ручной `toJson` тоже дублирование. 6) В ручном `toJson` используется `'email'` вместо `'email_address'` — несоответствие с `@JsonKey(name: 'email_address')`.
