# Задание 04 (16.2): Популярные генераторы: json_serializable, freezed — 🐛 Найти баг

**Тема:** 16. Генерация кода и build_runner | **Подтема:** 16.2 Популярные генераторы: json_serializable, freezed

---

В коде с `freezed` и `json_serializable` допущены ошибки. Найди их.

```dart
import 'package:freezed_annotation/freezed_annotation.dart';
import 'package:json_annotation/json_annotation.dart';

part 'product.freezed.dart';
// bug 1 — отсутствует part 'product.g.dart' (нужен для json_serializable)

@freezed                                             // bug 2 — нет @JsonSerializable() или
class Product with _$Product {                       //         fromJson factory
  const factory Product({
    required int id,
    required String name,
    double? price,
    @Default([]) List<String> tags,
  }) = _Product;

  // bug 3 — вычисляемый геттер в freezed-классе без const Product._()
  String get displayName => '$name (\$$price)';
}

@freezed
class Cart with _$Cart {
  const factory Cart({
    required List<Product> items, // bug 4 — List<Product>, но Product без fromJson
    @Default(0.0) double total,
  }) = _Cart;

  factory Cart.fromJson(Map<String, dynamic> json) => _$CartFromJson(json);
}

void main() {
  final p = Product(id: 1, name: 'Apple', price: 1.5, tags: ['fruit']);
  
  // bug 5 — copyWith на freezed-классе не поддерживает добавление в List
  final p2 = p.copyWith(tags: p.tags + ['organic']); // это работает, не баг
  
  // bug 6 — сравнение изменённого объекта
  print(p == p2);  // false — правильно (разные tags)
  
  // bug 7 — попытка изменить поле напрямую
  p.price = 2.0;   // compile error: final field
}
```

<summary>Решение</summary>

```dart
import 'package:freezed_annotation/freezed_annotation.dart';

part 'product.freezed.dart';
part 'product.g.dart';        // fix 1: добавляем для JSON сериализации

@freezed
class Product with _$Product {
  // fix 3: добавляем приватный const-конструктор для кастомных геттеров
  const Product._();

  const factory Product({
    required int id,
    required String name,
    double? price,
    @Default([]) List<String> tags,
  }) = _Product;

  // fix 2: добавляем fromJson для JSON сериализации
  factory Product.fromJson(Map<String, dynamic> json) => _$ProductFromJson(json);

  // теперь геттер работает (const Product._() разрешает кастомные члены)
  String get displayName => '$name (\$${price?.toStringAsFixed(2) ?? "N/A"})';
}

@freezed
class Cart with _$Cart {
  // fix 4: теперь Product имеет fromJson — Cart.fromJson будет работать
  const factory Cart({
    required List<Product> items,
    @Default(0.0) double total,
  }) = _Cart;

  factory Cart.fromJson(Map<String, dynamic> json) => _$CartFromJson(json);
}

void main() {
  final p = Product(id: 1, name: 'Apple', price: 1.5, tags: ['fruit']);

  // copyWith с расширенным списком — правильный паттерн
  final p2 = p.copyWith(tags: [...p.tags, 'organic']);
  print(p2.displayName); // Apple ($1.50)

  print(p == p2);        // false — правильно

  // fix 7: нельзя изменить поле напрямую — freezed поля final
  // используй copyWith:
  final p3 = p.copyWith(price: 2.0);
  print(p3.price); // 2.0
}
```

**Семь проблем:** 1) Без `part 'product.g.dart'` JSON сериализация не будет работать — `_$ProductFromJson` не сгенерируется. 2) Нет `factory Product.fromJson(...)` — нельзя десериализовать из JSON. 3) Кастомные геттеры в `freezed`-классе требуют `const ClassName._()` — без него компилятор выдаст ошибку. 4) `Cart` использует `List<Product>`, но если у `Product` нет `fromJson` — `_$CartFromJson` не сможет десериализовать продукты. 5) `copyWith` + `+` для списка — _работает_, не баг. 6) `p == p2` → `false` — _правильно_, не баг. 7) `p.price = 2.0` — compile error; правильный способ — `copyWith`.
