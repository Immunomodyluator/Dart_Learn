# Задание 05 (16.3): Создание собственного генератора — 🖊 Написать код

**Тема:** 16. Генерация кода и build_runner | **Подтема:** 16.3 Создание собственного генератора

---

Создай простой генератор кода, который по аннотации `@GenerateToString` генерирует метод `toString()`.

**Задача:**

1. Объяви аннотацию `@GenerateToString` в `lib/src/annotation.dart`
2. Создай `GenerateToStringBuilder` в `lib/builder.dart`, который:
   - Находит классы с аннотацией `@GenerateToString`
   - Генерирует extension с методом `toString()` включающим все публичные поля
3. Зарегистрируй builder в `build.yaml`

**Пример входного кода:**

```dart
@GenerateToString()
class Point {
  final int x;
  final int y;
  Point(this.x, this.y);
}
```

**Ожидаемый сгенерированный код:**

```dart
// point.g.dart
extension PointToString on Point {
  String toStringGenerated() => 'Point(x: $x, y: $y)';
}
```

<details>
<summary>Решение</summary>

```dart
// lib/src/annotation.dart
class GenerateToString {
  const GenerateToString();
}
```

```dart
// lib/builder.dart
import 'package:build/build.dart';
import 'package:source_gen/source_gen.dart';
import 'package:analyzer/dart/element/element.dart';

import 'src/annotation.dart';

// Основной генератор
class ToStringGenerator extends GeneratorForAnnotation<GenerateToString> {
  @override
  String generateForAnnotatedElement(
    Element element,
    ConstantReader annotation,
    BuildStep buildStep,
  ) {
    if (element is! ClassElement) {
      throw InvalidGenerationSourceError(
        '@GenerateToString can only be applied to classes.',
        element: element,
      );
    }

    final className = element.name;

    // Собираем публичные final/non-final поля
    final fields = element.fields
        .where((f) => !f.isStatic && !f.name.startsWith('_'))
        .map((f) => '${f.name}: \$${f.name}')
        .join(', ');

    return '''
extension ${className}ToString on $className {
  String toStringGenerated() => '$className($fields)';
}
''';
  }
}

// Фабрика builder для build_runner
Builder toStringBuilder(BuilderOptions options) =>
    SharedPartBuilder([ToStringGenerator()], 'to_string');
```

```yaml
# build.yaml — регистрация
builders:
  to_string:
    import: "package:my_generator/builder.dart"
    builder_factories: ["toStringBuilder"]
    build_extensions: { ".dart": [".g.dart"] }
    auto_apply: dependents
    build_to: source
    applies_builders: ["source_gen|combining_builder"]
```

**Пример сгенерированного файла:**

```dart
// point.g.dart — GENERATED CODE - DO NOT MODIFY BY HAND
part of 'point.dart';

extension PointToString on Point {
  String toStringGenerated() => 'Point(x: $x, y: $y)';
}
```

Генераторы в Dart строятся на `source_gen` (высокоуровневый API над `build`). `GeneratorForAnnotation<T>` автоматически находит все элементы с аннотацией типа `T`. `SharedPartBuilder` создаёт `part`-файлы (`.g.dart`), а `LibraryBuilder` — отдельные файлы.
