В коде, работающем с типами и компиляцией Dart, есть ошибки. Найди их.

```dart
// bug 1: нарушение const semantics — изменяемый объект помечен const
const mutableList = [1, 2, 3];

void main() {
  mutableList.add(4);  // bug 1: runtime UnsupportedError
}

// bug 2: неиспользованный результат sealed switch — пропущена ветка
sealed class Shape {}
class Circle extends Shape { final double radius; Circle(this.radius); }
class Square extends Shape { final double side; Square(this.side); }
class Triangle extends Shape { final double base, height; Triangle(this.base, this.height); }

double area(Shape s) => switch (s) {
  Circle(:final radius) => 3.14 * radius * radius,
  Square(:final side) => side * side,
  // bug 2: Triangle не обработан — compile error если sealed и exhaustive
};

// bug 3: небезопасный downcast без is-проверки
Object getValue() => 'hello';
void useValue() {
  final value = getValue() as int;  // bug 3: CastError в runtime
  print(value + 1);
}

// bug 4: const нельзя использовать для runtime-значений
class Config {
  final String host;
  // bug 4: const конструктор — но инициализация из Platform.environment не const
  const Config(this.host);
}

void makeConfig() {
  const config = Config(String.fromEnvironment('HOST'));
  // String.fromEnvironment — compile-time только если значение известно при компиляции
  // В runtime переменная окружения недоступна этому механизму
  print(config.host);
}

// bug 5: extension type теряет методы базового типа неожиданно
extension type Meters(double value) {}
extension type Seconds(double value) {}

void measureDemo() {
  final d = Meters(100.0);
  final t = Seconds(10.0);
  
  // bug 5: Meters не реализует double, нельзя прибавить напрямую
  final speed = d / t;  // bug 5: нет оператора / между Meters и Seconds
}

// bug 6: late final повторная инициализация
class Service {
  late final String _name;
  
  void init(String name) {
    _name = name;
  }
  
  void reinit(String name) {
    _name = name;  // bug 6: повторное присвоение late final — LateInitializationError (уже инициализировано)
  }
}

// bug 7: неправильное использование variance — запись в ковариантный список
void addToList(List<num> numbers) {
  numbers.add(3.14);
}

void varianceBug() {
  final ints = <int>[];
  addToList(ints);  // bug 7: передаём List<int> куда нужен List<num> — CastError при add
}

// bug 8: Type erasure — List<String> и List<int> неразличимы через is в некоторых случаях
void typeErasure() {
  final list = <String>['a', 'b'];
  print(list is List<String>);  // true — Dart сохраняет generic в runtime (reified generics)
  print(list is List<int>);     // false — OK
  
  // Но:
  void check(Object obj) {
    // bug 8: тип стёрт когда приходит как Object
    if (obj is List) {
      // Мы знаем что это List, но не знаем тип элементов без reified check
      final first = (obj as List).first as String;  // может быть CastError если List<int>
    }
  }
  check([1, 2, 3]);  // передаём List<int> но внутри кастуем к String
}
```

<summary>Решение</summary>

```dart
// fix 1: const создаёт immutable list — используйте final для изменяемых
// НЕПРАВИЛЬНО: const mutableList = [1, 2, 3]; затем .add()
// ПРАВИЛЬНО:
final mutableList = [1, 2, 3];
// или если нужна immutable версия — const, но тогда не вызывать add()
const fixedList = [1, 2, 3];  // читаем только

void main() {
  mutableList.add(4);  // OK — final, не const
  // fixedList.add(4); // UnsupportedError — правильно, это ожидаемое поведение const
}

// fix 2: добавляем недостающую ветку Triangle
sealed class Shape {}
class Circle extends Shape { final double radius; Circle(this.radius); }
class Square extends Shape { final double side; Square(this.side); }
class Triangle extends Shape { final double base, height; Triangle(this.base, this.height); }

double area(Shape s) => switch (s) {
  Circle(:final radius) => 3.14 * radius * radius,
  Square(:final side) => side * side,
  // fix 2: добавляем Triangle
  Triangle(:final base, :final height) => 0.5 * base * height,
};

// fix 3: проверяем тип через is перед кастом
Object getValue() => 'hello';
void useValue() {
  final value = getValue();
  // fix 3: is-check до каста — безопасный downcast
  if (value is int) {
    print(value + 1);
  } else {
    print('Not an int: $value (${value.runtimeType})');
  }
  // Или с явной обработкой исключения:
  // try {
  //   final n = value as int; ...
  // } on TypeError catch (e) { ... }
}

// fix 4: для runtime-значений используем factory constructor или обычный final
class Config {
  final String host;
  // fix 4: убираем const — Platform.environment недоступен в compile-time const
  Config(this.host);

  // Для compile-time констант — String.fromEnvironment с дефолтом
  static const defaultHost = String.fromEnvironment('HOST', defaultValue: 'localhost');
}

void makeConfig() {
  // fix 4: обычный (не const) конструктор для runtime-значений
  final config = Config(const String.fromEnvironment('HOST', defaultValue: 'localhost'));
  // или:
  final runtimeConfig = Config(Platform.environment['HOST'] ?? 'localhost');
  print(runtimeConfig.host);
}

// fix 5: extension type с реализацией операций или явным .value
extension type Meters(double value) {
  // Определяем оператор деления явно
  double operator /(Seconds other) => value / other.value;
}

extension type Seconds(double value) {}

void measureDemo() {
  final d = Meters(100.0);
  final t = Seconds(10.0);
  
  // fix 5: используем определённый оператор
  final speed = d / t;  // double — метры в секунду
  print('Speed: $speed m/s');
  
  // Или явно через .value:
  final speed2 = d.value / t.value;
  print('Speed2: $speed2 m/s');
}

// fix 6: late final нельзя переинициализировать — убираем reinit или используем late (без final)
class Service {
  // Вариант 1: late final — инициализируется ровно один раз
  late final String _name;

  void init(String name) {
    _name = name;  // первое присвоение — OK
  }
  
  // fix 6: если нужна переинициализация — убираем final
  // late String _mutableName;
  // void reinit(String name) { _mutableName = name; }
  
  // Или вариант с nullable + guard:
  String? _optionalName;
  void safeinit(String name) {
    if (_optionalName != null) {
      throw StateError('Already initialized');
    }
    _optionalName = name;
  }
}

// fix 7: передаём совместимый тип — List<num> или используем covariant правильно
void addToList(List<num> numbers) {
  numbers.add(3.14);
}

void varianceBug() {
  // fix 7a: создаём конкретно List<num>
  final nums = <num>[];
  addToList(nums);  // OK

  // fix 7b: если нужны именно int — используем отдельный метод
  final ints = <int>[];
  // addToList(ints); — опасно! List<int> is ковариантен List<num> в Dart
  // но добавление double в List<int> даст TypeError в runtime
  // Правильно: в сигнатуре принять только List<int>
}

void addIntsToList(List<int> numbers, int value) {
  numbers.add(value);
}

// fix 8: явная параметрическая проверка типа элементов
void typeErasure() {
  final list = <String>['a', 'b'];
  print(list is List<String>);  // true — Dart поддерживает reified generics
  print(list is List<int>);     // false

  void checkSafely(Object obj) {
    // fix 8: проверяем конкретный параметрический тип
    if (obj is List<String>) {
      // Теперь знаем, что это List<String>
      final first = obj.first;  // String, безопасно
      print('String list, first: $first');
    } else if (obj is List<int>) {
      final first = obj.first;  // int
      print('Int list, first: $first');
    } else if (obj is List) {
      // Неизвестный тип элементов — не кастуем бездумно
      print('Unknown list type: ${obj.runtimeType}');
    }
  }
  
  checkSafely(['a', 'b']);  // String list
  checkSafely([1, 2, 3]);   // Int list — не CastError!
}
```

**8 проблем:** 1) `const list` + `list.add()` — `UnsupportedError`: const-коллекции immutable. 2) `sealed class` с пропущенным вариантом в `switch` — compile error или неполный перебор. 3) `as int` без проверки на объекте, который возвращает `String` — `TypeError`. 4) `const Config(Platform.environment['HOST'])` — `Platform.environment` недоступен в compile-time const. 5) `d / t` где `d: Meters` и `t: Seconds` — extension types не наследуют операторы базового типа. 6) Повторное присвоение `late final` поля — `LateInitializationError`. 7) `List<int>` передаётся туда, где ожидается `List<num>`, и в неё добавляется `double` — `TypeError` в runtime из-за ковариантности списков в Dart. 8) `(obj as List).first as String` когда `obj` — `List<int>` — `TypeError`; нужна параметрическая проверка `obj is List<String>`.
</details>
