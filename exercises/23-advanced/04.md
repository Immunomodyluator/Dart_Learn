В структуре и коде публикабельного Dart-пакета есть ошибки. Найди их.

```yaml
# pubspec.yaml (намеренно испорченная версия)
name: My Cool Package  # bug 1: пробелы и заглавные буквы — невалидное имя пакета
description: A short one.  # bug 2: описание слишком короткое (pub.dev требует 60–180 символов)
version: 1.0  # bug 3: нет patch-сегмента — не semver (должно быть 1.0.0)
# bug 4: нет homepage/repository — теряем pub points за документацию

environment:
  sdk: '3.0.0'  # bug 5: нет верхней границы — пакет несовместим с Dart 4

dependencies:
  http: any  # bug 6: 'any' — нет ограничения версии, сломается при major-обновлении

dev_dependencies:
  test: ^1.24.0
```

```dart
// lib/my_cool_package.dart (намеренно испорченная версия)

// bug 7: экспортируем ВСЁ включая внутренние детали реализации
export 'src/internal_helper.dart';   // вспомогательный код, не часть публичного API
export 'src/public_api.dart';
export 'src/another_internal.dart';  // ещё внутренние детали

// bug 8: dart:io в основном файле — пакет не работает на Flutter Web
import 'dart:io';

class PackageLogger {
  void log(String message) {
    // bug 8: Platform.isAndroid недоступен в browser/WASM
    if (Platform.isAndroid) {
      print('[Android] $message');
    } else {
      print(message);
    }
  }
}
```

```dart
// lib/src/public_api.dart (намеренно испорченная версия)

<summary>Решение</summary>

```yaml
# pubspec.yaml — исправленная версия

# fix 1: только строчные буквы, цифры и подчёркивания — dart pub naming convention
name: my_cool_package

# fix 2: описание 60–180 символов (pub.dev score за description)
description: >-
  A versatile utility package for Dart providing common data structures,
  helpers, and functional programming primitives for everyday development.

# fix 3: полный semver — major.minor.patch
version: 1.0.0

# fix 4: добавляем ссылки für pub points
homepage: https://github.com/example/my_cool_package
repository: https://github.com/example/my_cool_package
issue_tracker: https://github.com/example/my_cool_package/issues

environment:
  # fix 5: верхняя граница обязательна — иначе пакет заявляет совместимость с любым SDK
  sdk: '>=3.0.0 <4.0.0'

dependencies:
  # fix 6: конкретные ограничения версии
  http: ^1.2.0

dev_dependencies:
  lints: ^3.0.0
  test: ^1.24.0
```

```dart
// lib/my_cool_package.dart — исправленная версия

// fix 7: экспортируем ТОЛЬКО публичный API
export 'src/public_api.dart';
// internal_helper.dart и another_internal.dart НЕ экспортируются

// fix 8: убираем dart:io из основного файла
// Для кросс-платформенных пакетов используем платформо-нейтральные API
// Если нужно логирование — используем dart:developer или пакет logging

import 'dart:developer' as dev;

class PackageLogger {
  void log(String message) {
    // fix 8: dart:developer работает везде: VM, web, Flutter
    dev.log(message, name: 'my_cool_package');
  }
}
```

```dart
// lib/src/public_api.dart — исправленная версия

/// A utility class that performs double computation on an integer [value].
///
/// Example:
/// ```dart
/// final obj = MyClass(5);
/// print(obj.compute()); // 10
/// ```
// fix 9: добавляем dartdoc-комментарии (pub.dev требует документацию для pub points)
class MyClass {
  /// The base integer value used in computations.
  final int value;

  /// Creates a [MyClass] instance with the given [value].
  const MyClass(this.value);

  /// Returns [value] multiplied by 2.
  // fix 10: добавляем документацию к методу
  int compute() => value * 2;

  // fix 11: убираем @override — метод не переопределяет ничего из базового класса
  int extraMethod() => value + 1;
}

// fix 12: конкретные типы вместо dynamic
String processData(String input) {
  return input;
}

// Общая версия с generics вместо dynamic:
T processTyped<T>(T input) => input;

// fix 13: breaking change требует major version bump (1.0.0 → 2.0.0)
// Для обратной совместимости добавляем опциональный параметр:
void doSomething(String name, {int count = 1}) {  // fix: named optional с default
  print('$name: $count');
}
// Или создаём новый метод и deprecate старый:
@Deprecated('Use doSomethingWithCount instead. Will be removed in 2.0.0.')
void doSomethingLegacy(String name) => doSomething(name);
void doSomethingWithCount(String name, int count) => doSomething(name, count: count);
```

```dart
// test/my_class_test.dart — исправленная версия

import 'package:my_cool_package/my_cool_package.dart';
import 'package:test/test.dart';

void main() {
  group('MyClass', () {
    // fix 14: описательное имя теста — что тестируем и какой ожидаем результат
    test('compute returns value multiplied by two', () {
      final obj = MyClass(5);
      expect(obj.compute(), equals(10));
    });

    test('compute with zero returns zero', () {
      expect(MyClass(0).compute(), equals(0));
    });

    test('compute with negative value returns doubled negative', () {
      expect(MyClass(-3).compute(), equals(-6));
    });

    // fix 15: тестируем публичный контракт (поведение), а не внутреннее состояние
    // Если нужно проверить начальное состояние — делаем это через публичный API:
    test('extraMethod returns value incremented by one', () {
      expect(MyClass(5).extraMethod(), equals(6));
    });
    // Прямое тестирование .value уместно только если это часть публичного контракта
  });

  group('processData', () {
    test('returns input unchanged', () {
      expect(processData('hello'), equals('hello'));
    });
  });
}

**15 проблем:** 1) Имя с пробелами и заглавными — pub.dev отклонит: только `[a-z0-9_]`. 2) Описание короче 60 символов — теряем pub.dev score. 3) `version: 1.0` — нет patch-сегмента, нарушает semver. 4) Нет `repository`/`homepage` — теряем pub points за документацию. 5) `sdk: '3.0.0'` без верхней границы — pub.dev предупреждает о потенциальной несовместимости с Dart 4. 6) `http: any` — нет контроля над версией зависимости. 7) Экспорт internal-файлов ломает инкапсуляцию: внутренние типы становятся частью публичного API. 8) `dart:io` с `Platform.*` в кросс-платформенном пакете — не работает на Flutter Web. 9) Нет dartdoc — pub.dev score за документацию снижается. 10) Метод без документации. 11) `@override` на метод без базового класса/интерфейса — линтер ошибается, можно ввести в заблуждение. 12) `dynamic` в публичном API — теряем типобезопасность. 13) Добавление обязательного параметра — breaking change, требует major version bump по semver. 14) Имена тестов `test1` — нечитаемо в отчётах. 15) Тестирование внутреннего состояния вместо поведения — хрупкие тесты, привязанные к реализации.
