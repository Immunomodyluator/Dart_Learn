<summary>Решение</summary>

```dart
class Animal {
  final String name;
  Animal(this.name);
  @override String toString() => '$runtimeType($name)';
}
class Dog extends Animal { Dog(super.name); }
class Cat extends Animal { Cat(super.name); }

// List<Dog> — подтип List<Animal> в Dart (ковариантный по умолчанию)
// Безопасно для чтения, но опасно для записи!
void printAll(List<Animal> animals) {
  for (final a in animals) print(a);
}

// Безопасная копия только на чтение — через Iterable (ковариантен)
void addAll<T extends Animal>(List<T> dest, Iterable<T> src) {
  dest.addAll(src);
}

void dangerousExample() {
  final dogs = <Dog>[Dog('Rex'), Dog('Buddy')];

  // Dart позволяет это (ковариантность List):
  final List<Animal> animals = dogs;

  // НО: это бросит CastError в рантайме, т.к. dogs — List<Dog>
  try {
    animals.add(Cat('Whiskers')); // рантайм: type 'Cat' is not a subtype of 'Dog'
  } catch (e) {
    print('Runtime error: $e');
  }
}

void main() {
  printAll([Dog('Rex'), Cat('Luna'), Dog('Buddy')]);
  // Dog(Rex)  Cat(Luna)  Dog(Buddy)

  dangerousExample();
}
```

Dart-списки **ковариантны** — `List<Dog>` присваивается `List<Animal>`. Это удобно, но добавление объекта неправильного типа обнаруживается только в **рантайме**. Для безопасной передачи только на чтение используй `Iterable<T>` или `covariant`. Для полностью type-safe кода — generic с bound `<T extends Animal>`.
