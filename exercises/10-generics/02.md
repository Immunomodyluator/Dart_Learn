# Задание 02 (10.1): Обобщённые классы — 🐛 Найти баг

**Тема:** 10. Дженерики | **Подтема:** 10.1 Обобщённые классы

---

Обобщённый класс `Pair` ведёт себя неожиданно. Найди и исправь все ошибки.

```dart
class Pair {               // нет параметра типа
  final first;
  final second;

  Pair(this.first, this.second);

  Pair swap() => Pair(second, first);

  @override
  String toString() => '($first, $second)';
}

void main() {
  final p = Pair(1, 'hello');
  print(p.first + 1);     // dynamic — нет проверки типа
  print(p.swap());        // (hello, 1) — ok, но типы потеряны

  // Попытка использовать как Pair<int, String>
  final q = Pair<int, String>(2, 'world'); // ошибка — нет параметров типа
}
```

<details>
<summary>Решение</summary>

```dart
class Pair<A, B> {
  final A first;
  final B second;

  const Pair(this.first, this.second);

  Pair<B, A> swap() => Pair(second, first); // типы переставляются корректно

  @override
  String toString() => '($first, $second)';
}

void main() {
  final p = Pair<int, String>(1, 'hello');
  print(p.first + 1);     // 2 — компилятор знает, что first — int
  print(p.swap());        // (hello, 1) — тип Pair<String, int>

  final q = Pair<int, String>(2, 'world');
  print(q);               // (2, world)
}
```

**Три бага:** 1) `class Pair` без параметров типа — поля `dynamic`, теряется typesafety. 2) `swap()` возвращает `Pair` без параметров — тип результата `Pair<dynamic, dynamic>`; правильно `Pair<B, A>`. 3) `Pair<int, String>(...)` не компилировался — класс не принимал параметры типа.

</details>
