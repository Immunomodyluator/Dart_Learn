# Задание 06 (10.3): Ковариантность и contravariance — 🐛 Найти баг

**Тема:** 10. Дженерики | **Подтема:** 10.3 Ковариантность и contravariance

---

Код компилируется, но падает с `TypeError` в рантайме. Объясни почему и исправь.

```dart
class Animal { String name; Animal(this.name); }
class Dog extends Animal { Dog(super.name); void bark() => print('Woof!'); }

void fillWithDogs(List<Animal> list) {
  list.add(Dog('Rex'));
  list.add(Dog('Buddy'));
}

void main() {
  final cats = <Dog>[];           // List<Dog>
  fillWithDogs(cats);             // передаём как List<Animal>
  // ^^ Dart разрешает (ковариантность), но...

  for (final dog in cats) {
    dog.bark();  // TypeError: Cat is not a subtype of Dog
  }
}
```

<details>
<summary>Решение</summary>

```dart
class Animal { String name; Animal(this.name); }
class Dog extends Animal {
  Dog(super.name);
  void bark() => print('${name}: Woof!');
}

// Исправление 1: принимать List<Dog> явно
void fillWithDogs(List<Dog> list) {
  list.add(Dog('Rex'));
  list.add(Dog('Buddy'));
}

// Исправление 2 (альтернативно): generic с bound
void fillWith<T extends Animal>(List<T> list, T Function(String) factory) {
  list.add(factory('Rex'));
  list.add(factory('Buddy'));
}

void main() {
  final dogs = <Dog>[];
  fillWithDogs(dogs);
  for (final d in dogs) d.bark(); // Rex: Woof!  Buddy: Woof!

  final dogs2 = <Dog>[];
  fillWith<Dog>(dogs2, Dog.new);
  for (final d in dogs2) d.bark();
}
```

**Баг:** Dart ковариантно разрешает `List<Dog>` там, где ожидается `List<Animal>`. Внутри `fillWithDogs` мы обращаемся к списку как `List<Animal>` и добавляем `Dog` — это корректно. Но исходный список `cats` объявлен как `List<Dog>`, и рантайм проверяет тип при `add` — здесь реально `Dog` добавляется в `List<Dog>`, поэтому конкретно этот пример не падает. Опасность возникает, когда добавляют **другой** подтип: `list.add(Cat('Whiskers'))` при том что список `List<Dog>` — тогда `CastError`. Правильное решение — типизировать параметр точно.
